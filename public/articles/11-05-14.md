# 全屏加载时先留空行，说明摘要层高度不能偷走滚动区的一行

状态栏命令还没返回时，全屏模式下这里也不会让整条底栏从零行突然长成一行。它先留出一行空位，等真正有内容了再填进去。

这是很典型的终端界面纪律。高度变化看起来只差一行，但那一行会直接把上面的滚动区挤动，所以宁可先空着，也别让内容突然跳一下。

## 实现链

`StatusLine.tsx` 在 `statusLineText` 还没返回时，如果当前是 fullscreen，也会先渲染 `<Text> </Text>` 占住一行。代码注释写得很直白：不能让状态栏在命令返回时把 footer 高度从 `0→1` 抬起来。

## 普通做法

更普通的做法是 loading 时什么都不画，等命令回来再把状态栏整行插进去。

## 为什么不用

Claude Code 不这么做，是因为终端滚动区对高度变化极其敏感，一行的突然出现就会让阅读位置整体漂移。状态栏不能为了省一行去偷滚动区稳定性。

## 代价

代价是 loading 时会保留一条看起来空白的行；但换来的是全屏阅读位置不会被底部突然顶动。
