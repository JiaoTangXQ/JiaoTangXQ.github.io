# 窄权限不是小气，而是在先封死自动化风险

## 实现链

这一页重点看“窄权限不是小气，而是在先封死自动化风险”。对应源码主要是 `src/services/SessionMemory/sessionMemory.ts`。
这一章真正落在 `sessionMemory.ts` 为记忆代理准备的窄权限工具上下文上。Claude Code 让 session memory 几乎只准编辑那一个记忆文件，说明它把自动化记忆首先视为风险边界问题，而不是写作能力问题。

## 普通做法

更轻松的做法，是给记忆代理更宽的文件权限，让它根据需要自由整理、归档甚至改别的上下文文件。

## 为什么不用

Claude Code 没这么做，因为自动记忆一旦拿到宽权限，最危险的不是写错一句总结，而是开始默默改写更多现实。它宁愿让代理笨一点，也要先把自动化写入封死在最窄的边界里。

## 代价

这样做更安全，也更符合“先控风险再谈智能”。代价是记忆代理的动作范围非常有限，很多看起来方便的自动整理都必须放弃。
