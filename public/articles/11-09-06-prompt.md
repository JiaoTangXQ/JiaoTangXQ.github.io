# 点悬浮 prompt 后先隐去表头再回原文，说明阅读修正必须分两段

## 实现链
StickyPromptHeader 的 onClick 先把 header 置成 clicked，让 FullscreenLayout 把 sticky 区域收回去，再执行回跳。这样真正落到原 prompt 前，先清掉了顶部的重复线索。

## 普通做法
普通实现会让浮层和原文一起保留，或者只做一次滚动跳转。那样用户会同时看到摘要和原句，修正动作看上去也不干净。

## 为什么不用
这里需要的是阅读修正，不是额外信息层。先隐 header 再回原文，能保证视野里只有一个主线索，用户不会在两个等价入口之间来回抖。

## 代价
代价是交互变成两段式，还得靠 sentinel 控制 header 的生命周期。换来的好处是点击 sticky prompt 的动作更像回到正文，而不是在两个标题之间切换。
