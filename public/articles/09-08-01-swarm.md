# 任务系统解决劳动闭环，swarm 解决组织秩序

任务系统和 swarm 都能“让 agent 工作”，但它们解决的根本不是同一类问题。

## 实现链

任务系统的核心在 `Task.ts`、`framework.ts`、`LocalAgentTask.tsx`，重点是登记、进度、通知、回收。swarm 的核心则在 `spawnInProcess.ts`、`inProcessRunner.ts`、`teammateMailbox.ts`、`permissionSync.ts` 和 backend registry，重点是身份、消息、审批、空间和模式选择。

## 普通做法

普通做法会把 swarm 理解成“任务系统的多实例版”，无非是一次起多个任务。

## 为什么不用

Claude Code 认为这不够，因为一旦进入团队，新增的不只是任务数，而是制度层。任务系统回答“怎样把劳动管住”，swarm 回答“怎样让多个劳动者像一个团队那样有秩序地存在”。

## 代价

代价是 swarm 不可能只是任务系统上的薄皮，必须有自己整套配套设施。
