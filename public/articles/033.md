# 第十三卷回收总导航判断

## 实现链
这页承担的是总导航判断职责。它对应的真实实现链不是单一模块，而是多条已经拆开的代码主线怎样重新形成可读入口，再把读者分送到更细的附录。

## 普通做法
更普通的做法，是只留目录树或只留一张总表，要求读者自己在大纲里跳。

## 为什么不用
Claude Code 没满足于那种“自己找”的导航方式，因为回收树已经足够深，不显式指路会让结构优势重新变成迷宫。

## 代价
代价是会多出一层导航页，而且不同父页之间会出现少量重复解释。
