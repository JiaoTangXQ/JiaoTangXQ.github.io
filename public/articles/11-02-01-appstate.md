# AppState 装的不是页面数据，而是当前世界事实

`AppState` 最容易被低估的地方，是它看起来像普通前端 state，实际上装进去的是整台工作台当前成立的事实：权限模式、任务、MCP、远端连接、团队上下文、通知、footer 选择、插件状态。

也就是说，它不是“这个组件现在怎么画”的小数据，而是“用户此刻身处怎样一个工作世界”的总快照。这样的状态才配支撑 Claude Code 这种终端应用。

## 实现链

`AppStateStore.ts` 里的 `AppState` 装的不是某个页面要不要展开，而是任务、MCP 客户端、远端会话 URL、通知、权限、模型、面板状态、工具上下文这些“当前世界事实”。它描述的是这台工作台现在活在什么现实里，不只是页面怎么画。

## 普通做法

更常见的前端 state 会优先存页面局部数据，比如 tab 索引、按钮开关、弹窗显隐，把跨系统事实留给别的模块各自管理。

## 为什么不用

Claude Code 不这么分，是因为这里的 UI 本身就在消费运行时现实。要是任务、远端、权限和通知不住进共同状态，界面层就只能到处向不同模块打听真相。

## 代价

代价是 state 类型会很胖，读起来不像传统“页面 store”；但换来的是界面和运行时说的是同一份现实，而不是几份拼接出来的近似值。
