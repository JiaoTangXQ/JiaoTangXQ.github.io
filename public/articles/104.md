# microcompact 和 autocompact 解决的是同一个问题吗？

短答案：不是。它们是连续性系统的两个不同层，解决的压力来自不同时间尺度，触发条件也不同。

## 先说为什么"一直堆消息"不行

最简单的上下文连续性方案是：把所有消息都保留在 context window 里，让模型看到完整的对话历史。这在 context window 足够大时确实能用，但 Claude Code 不能这么做，原因不只是 token 预算。

更根本的问题是：一个持续工作的 session 里，很多早期工具调用的输出（比如一个 bash 命令的 3000 行日志）到后期已经完全失去了参考价值。把它们留在 context 里，既占预算，又在"当前现场"里引入噪音。

所以 Claude Code 的连续性处理目标是：**不同时间尺度的问题，用不同粒度的机制处理。**

## microcompact：修复当前窗口里的碎接缝

`microcompact` 处理的是在当前 context window 内部发生的问题。

典型场景：LLM 进行了一轮复杂的工具调用序列，产生了大量中间结果，这些中间结果现在占据了 context 的很大比例，但它们的实际意义已经被后续的处理步骤覆盖了。这些是"接缝"——消息序列中的碎片，它们存在于 context 里但已经不是有效现场的一部分。

microcompact 的触发时机是在 query 循环的内部，当检测到 context 里存在可以折叠的接缝时，对那段消息序列做局部压缩。注意：这个操作不需要等到整个 session 达到某个阈值，它随时可以发生。

关键点：microcompact 不碰 session memory，也不重写整个对话历史，它只修复局部的碎片。

## autocompact：处理整段会话的预算压力

`autocompact` 是更大尺度的机制，它在整个会话的 token 使用量接近阈值时触发。

autocompact 会对会话的历史做一次较大范围的压缩，生成一个摘要替换掉压缩范围内的原始消息序列。这个摘要需要足够高质量，因为它会成为 session 继续运行的"历史记忆"的一部分。

autocompact 的一个关键问题是**触发时机**：它必须在当前主回合落稳之后才能运行，而不是在 LLM 正在推理的过程中突然截断上下文。这是连续性系统里时序很敏感的地方——提前触发会打断推理，延迟触发可能来不及。

## SessionMemory：第三个层

除了 microcompact 和 autocompact，Claude Code 还维护 SessionMemory，这是比 autocompact 更长期的存储层。

SessionMemory 不是每次压缩时都更新的，它只在特定阈值和自然停顿点更新。"自然停顿点"是关键词——它对应的是用户明显在做阶段性收尾的时刻，比如完成了一个功能模块、提交了一次 commit。在这种时刻更新 session memory，能保证记忆内容对应的是有意义的里程碑，而不是随机的压缩点。

## 三个层的关系

```
当前 context window
  └── microcompact（局部碎片修复，随时触发）
  
会话整体 token 预算
  └── autocompact（整段历史压缩，阈值触发）
  
长期连续性
  └── SessionMemory（阶段记忆更新，停顿点触发）
```

三个层的设计各自服从不同的触发条件和压缩粒度，互相不干扰。把任何两个层合并，都会让触发条件变得难以控制：microcompact 的局部性会被破坏，或者 session memory 的有意义更新会变成频繁的自动触发。

## 自动化克制的意义

`services/compact/` 目录的分层实现体现了一个更深的设计原则：**连续性处理不等于"随时整理"。** 一个随时都在整理的系统，会让每一步的"当前现场"都变得不稳定，因为用户不知道下一次整理会在什么时候发生、会改变哪些东西。

显式的分层和各自明确的触发条件，让连续性处理变得可预期。这是"自动化克制"——不是不整理，而是只在对的时机做对的事。

## 面试答题要点

被问到"你们的上下文管理机制是什么"时，区分层次是关键：

- 如果只能说"token 够了就压缩"，说明只理解了 autocompact 这一层
- 能区分"局部碎片修复"和"整段历史摘要"，说明理解了两层的不同时间尺度
- 能说清楚"为什么 session memory 不随 autocompact 一起更新"，说明理解了连续性的完整设计意图

关键对比：microcompact 是维护"当前现场清洁度"的，autocompact 是解决"长期预算压力"的，SessionMemory 是保存"有意义里程碑"的。三件事不同，机制分开是正确的。
