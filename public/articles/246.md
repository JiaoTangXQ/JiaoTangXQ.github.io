# 第十三卷回收索引：前四卷里的模式、目录和连接方式决定这一次我到底是谁

## 实现链
前四卷里后面逼出来的，不只是“世界会不会被改写”，  
而是“这一次我到底是谁、能不能进场”。

这条线真正逼出来的，是第十三卷里关于真实边界和进场身份先决问题的判断。

这页实际回收的实现链是：
- `02-02-03` 把模式、目录和连接方式写成“这一次我到底是谁”的先决问题，这会继续支撑 `13-01-02`，说明真实边界不是出事后再补，而是进场前就得认清。

真实边界不是先谈能力，再补身份。  
它先决定谁能进场，能力才有资格开始说话。

## 普通做法
更普通的做法，是把启动、输入、主循环里的这些小判断留在顺手逻辑里，不单独立规矩。

## 为什么不用
Claude Code 不这么做，是因为工作台一旦要长期运行，这些看似前期的小处理其实都是后面秩序能否站稳的根，不能等出事了再补。

## 代价
代价是前期代码和解释都会显得更重，也更早暴露出长期工程的野心。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
