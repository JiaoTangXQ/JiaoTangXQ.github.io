# Claude Code 条件能力地图

Claude Code 里很多能力不是永远在线，而是按 feature gate、模式和环境条件显式打开。

## 实现链

你会在 `commands.ts`、`tasks.ts`、`registry.ts` 这类文件里反复看到 `feature(...)`、动态 import、mode snapshot 和环境检测。命令、任务、队友后端、语音、远端、workflow 等能力，都可能根据构建开关、会话模式或宿主环境决定是否出现。

## 普通做法

普通做法要么把所有能力都硬塞进二进制里，要么只在配置层做一点开关控制。

## 为什么不用

Claude Code 不这么粗放，是因为它的能力差异太大了。有的只在某些环境可用，有的会引入重依赖，有的会明显改变权限与 UI 行为。条件能力如果不前置表达，系统会在运行期充满“为什么这个按钮这次有、下次没”的惊喜。

## 代价

代价是代码里会出现很多 feature gate 和条件 import，读起来不像一条纯粹主线。但这比把所有能力混成一锅更可控。
