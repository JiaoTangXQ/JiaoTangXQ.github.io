# hook 的五种类型有什么区别，什么场景用哪种？

## 从 schema 开始

`schemas/hooks.ts` 定义了所有 hook 类型的 Zod schema。四种可以持久化到 settings.json 的类型：

- `BashCommandHookSchema`（type: 'command'）
- `PromptHookSchema`（type: 'prompt'）
- `AgentHookSchema`（type: 'agent'）
- `HttpHookSchema`（type: 'http'）

还有第五种 `function` 类型，只能在代码里直接注册，不能写进配置文件（因为函数无法序列化）。

## 逐类分析

### command hook
```json
{
  "type": "command",
  "command": "prettier --check $TOOL_INPUT_PATH",
  "shell": "bash",
  "timeout": 30
}
```
最直接。触发时用配置的 shell 执行一段命令。适合"在工具执行前后跑一个脚本"——比如代码格式检查、日志记录、通知推送。

`async` 字段允许后台非阻塞执行，`asyncRewake` 更特殊：后台执行，但如果退出码是 2，会"唤醒"主 agent 继续工作。这是给"异步检查，有问题再介入"设计的。

`once` 字段：执行一次后自动移除。适合"初始化后不再需要"的 hook。

### prompt hook
```json
{
  "type": "prompt",
  "prompt": "判断这个 bash 命令是否安全：$ARGUMENTS",
  "model": "claude-haiku-3-5",
  "timeout": 10
}
```
用小模型评估输入。当判断逻辑太复杂、不适合写成脚本时使用。典型场景：语义理解、分类判断、生成结构化反馈。

### agent hook
```json
{
  "type": "agent",
  "prompt": "验证单元测试已经运行并通过",
  "timeout": 60
}
```

schema 注释里有一段很有价值的说明：
```
// DO NOT add .transform() here. This schema is used by parseSettingsFile,
// and updateSettingsForSource round-trips the parsed result through
// JSON.stringify — a transformed function value is silently dropped,
// deleting the user's prompt from settings.json (gh-24920, CC-79).
```

这是一个真实 bug 的防御注释。之前有人加了 `.transform()` 把 prompt 字符串转成函数，结果 JSON 序列化时函数被丢掉，用户配置消失。现在这里有注释明确禁止再犯这个错。

agent hook 会启动一个受限子 agent，给它特定的 prompt、有限的工具集、有限的回合数，让它做检验。它比 prompt hook 更强大——可以真正执行操作（比如跑测试、检查文件状态），而不只是做语言判断。默认用 Haiku 模型以控制成本。

### http hook
```json
{
  "type": "http",
  "url": "https://internal-policy.company.com/check",
  "headers": { "Authorization": "Bearer $MY_TOKEN" },
  "allowedEnvVars": ["MY_TOKEN"]
}
```

HTTP 请求，POST hook 输入数据到外部 URL。关键安全设计：`allowedEnvVars` 白名单。只有明确列出的环境变量才会被插入 header 值。其他 `$VAR` 引用会被解析为空字符串。这防止了意外的凭证泄漏。

## 共同字段的设计哲学

所有 hook 类型都有 `if` 字段：
```typescript
const IfConditionSchema = z.string().optional().describe(
  'Permission rule syntax to filter when this hook runs (e.g., "Bash(git *)"). ' +
  'Only runs if the tool call matches the pattern.'
)
```

`if` 用权限规则语法来过滤——同样的语法用于 alwaysAllow/alwaysDeny 规则。这不是巧合，而是统一语言设计：整套系统里描述"工具+参数模式"只需要学一种语法。

## 面试指导

五种 hook 类型的区分在于"执行后端"：脚本、小模型、子 agent、HTTP 服务、内存函数。面试时着重说 agent hook 和 http hook 的特殊之处——agent hook 启动真正的子 agent、http hook 的 allowedEnvVars 安全设计。agent hook 注释里的 bug 历史（.transform() 导致配置消失）是很有说服力的"真实踩坑"例子。
