# 第十三卷回收索引：任务世界里的劳动连续性与长期工作台

## 实现链
这页对应的实现链，主要落在 `Task.ts`、`tasks/*`、`AgentTool`、background/foreground 转场、通知协议、outputFile 和回收逻辑上。它真正回收的是：劳动为什么要先被做成正式任务对象，才能跨会话、跨前后台和跨协作继续成立。

## 普通做法
更普通的做法，是把后台工作留成零散异步句柄，或者把子 agent 当成框架外面的特殊分支。

## 为什么不用
Claude Code 没这么做，因为它要处理的不只是“能不能跑起来”，而是“谁能接手、怎么恢复、怎样通知用户、怎样体面退场”。这些都需要任务对象先成立。

## 代价
代价是任务模型更重、状态更多，前后台切换和通知协议都要额外维护。
