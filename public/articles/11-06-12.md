# 活动摘要只给进行中且未阻塞的任务，说明热度应该留给还能推进的事

任务面板不会把每个任务最近干了什么都摊开。只有它还在进行、而且没有被依赖关系卡住时，那条活动摘要才值得出现。

## 实现链

在 `TaskListV2.tsx` 里，`showActivity` 的条件是 `isInProgress && !isBlocked && activity`。`activity` 来自运行中 teammate 的活动摘要，`isBlocked` 则来自 `openBlockers.length > 0`。一旦任务被阻塞，面板就不再给它加活动热度，而是优先显示阻塞信息和更淡的任务主体。

## 普通做法

更普通的列表会把“最近动作”无差别挂在所有任务上，不管它是不是已经卡住了，也不管它还能不能继续推进。

## 为什么不用

Claude Code 想把热度留给真正还能向前走的事。被卡住的任务已经有阻塞语义了，再继续给它加活动感，只会让注意力误以为它还在顺利推进。

## 代价

代价是有些任务看起来会很安静，即使它们之前也忙过一阵。这个安静不是缺信息，而是界面故意把目光让给更值得推进的那一条。
