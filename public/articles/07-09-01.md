# 小桥接层不是业务功能，而是结构保险丝

## 实现链
- `mcpSkillBuilders.ts` 自己几乎不做业务，只保存 `createSkillCommand` 和 `parseSkillFrontmatterFields` 这两个构建器引用。它像一个很薄的登记簿，专门用来隔开 `client.ts`、`mcpSkills.ts` 和 `loadSkillsDir.ts` 之间的依赖图。
- 这层小桥接之所以重要，不是因为它功能多，而是因为它让复杂模块之间不用直接互相咬住。

## 普通做法
- 更普通的做法是需要什么就直接 import 什么，少一层中转就少一层理解成本。
- 对纯业务代码来说，这通常是合理直觉。

## 为什么不用
- Claude Code 没这么直连，是因为这里碰到的是依赖环和打包现实，而不是简单的“少写一个文件”。
- 如果没有这根保险丝，`client.ts` 和 `loadSkillsDir.ts` 一类的大模块会通过 MCP skills 重新缠成一团。

## 代价
- 代价是你要额外理解一个“看起来什么都没干”的模块。
- 但结构保险丝往往就该这样：平时不起眼，出问题时才知道它在挡灾。
