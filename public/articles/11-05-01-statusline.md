# StatusLine 汇总的是系统现实，不是装饰信息

状态栏最优雅的地方，是它不报“心情”，只报现实。模型、权限模式、工作目录、worktree、session 名称、上下文占用、费用、速率限制、`vim` 状态，这些都是当前这台工作台真正处在什么状态的摘要。

所以它不是装饰横条，而是一层持续更新的系统现实。普通读者不必钻进设置和日志，也能随时知道这台 agent 现在在哪、能做什么、已经花了多少、离上限还有多远。

## 实现链

`StatusLine.tsx` 先从 `buildStatusLineCommandInput()` 把模型、工作目录、费用、上下文、速率、worktree、agent、remote 等事实打包，再决定要不要执行状态栏命令。它汇总的是系统现实，不是一串装饰性标签。

## 普通做法

更普通的状态栏会手写几个固定字段，能显示点模式或目录就算完成任务。

## 为什么不用

Claude Code 不满足于固定标签，是因为底部这一行要承担“快速说明当前世界”的职责。要是它只会拼几块装饰信息，很多真正会改写操作判断的事实就进不来。

## 代价

代价是状态栏不再是纯展示层，而要跟着系统事实更新，还得处理命令执行、失败和节流；但摘要的质量更高。
