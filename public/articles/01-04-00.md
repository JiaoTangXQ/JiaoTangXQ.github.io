# 流程也是制度导读：系统不只靠主流程活着

这一章真正要补的，不是再解释一遍主流程怎么走，而是先提醒读者：Claude Code 之所以没把自己跑坏，靠的不只是“下一步是什么”，还靠“什么能做、什么不能做、什么要换一种方式做”。

所以这一章的五片叶子，不是在主流程旁边零散补丁。它们合起来讲的是同一件事：主流程负责推进，制度层负责让推进不失控。先看 [01-04-01-主流程不能解释为何不失控.md](/Users/xxx/tmp/cc/docs/claude-code-book/01-认识这台终端里的工作台/01-04-流程也是制度/01-04-01-主流程不能解释为何不失控.md)，再顺着权限与设置、hooks、连续性和条件能力往下读，这一章才会从“流程补充说明”变成“工作台为什么能在现实里活下去”的制度入口。

## 实现链
这一章的实现链横跨主流程之外的制度线：权限与设置先决定工具能不能动，hooks 允许外部规则插手，连续性机制决定跨回合如何延续，条件能力又会改写用户所在的能力世界。代码上这些东西分散在 `permissionSetup`、`Tool.ts`、`hooks`、`query.ts` 和 compact/记忆服务里，但它们共同做的是让主流程不会一脚踩进失控现场。

## 普通做法
更普通的系统会把主流程当绝对主角，把权限、设置、hook、记忆都当旁边的配套件。这样写起来简单，流程图也漂亮。

## 为什么不用那种做法
Claude Code 不能这么理解，因为这些制度线不是事后补丁，而是主流程能在真实世界持续工作的前提。主流程只负责推进，真正保证它不越权、不失忆、不撞外部制度的，是这些额外机制。

## 代价与回报
回报是系统为什么稳这件事能被说清。代价是读者必须接受：入口不只是一条流程线，还要看见分散在多处代码里的制度世界。
