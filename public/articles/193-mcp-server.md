## 普通做法
更普通的做法，是每种来源保留自己的装配方式和运行时语言，等真的冲突了再临时做桥接。 对父级入口页来说，更常见的写法就是继续把两三条不同链挤在一页里，读起来省页数，但判断会糊。

## 为什么不用
Claude Code 不这么做，是因为来源越多，越要先学旧骨架的语言；否则系统会被多套协议、多套入口和多套制度动作拖散。 这些父级页现在存在，就是为了先把分支关系讲清，再让更细页分别承担细节。

## 代价
代价是注册、翻译和回收线都会变多，很多地方看起来像是在重复做统一。 作为父级入口，它还额外带来导航变深、读者要多跳几页的成本。
