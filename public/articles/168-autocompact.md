## 实现链
连续性世界这一支现在已经自然分成两条更窄的线：

- 附件、计划和技能线索被重新带回，桌面才有可能继续工作
- `autocompact` 保会话，不做漂亮整理，才是真正的连续性保命线

所以这页现在只负责导流，不再负责把两条线混成一条。

## 普通做法
更普通的做法，是一发现上下文变长就立刻整理，或者把当前状态和长期记忆统统塞回主会话，省掉额外的插槽和外置存储。 对父级入口页来说，更常见的写法就是继续把两三条不同链挤在一页里，读起来省页数，但判断会糊。

## 为什么不用
Claude Code 不这么做，是因为整理太早会打断主回合，记忆不分层会把短期噪音和长期知识混在一起，最后让连续性系统同时失去可读性和可恢复性。 这些父级页现在存在，就是为了先把分支关系讲清，再让更细页分别承担细节。

## 代价
代价是需要额外维护 compact boundary、回合后插槽、外置 memory 和记忆回灌链，系统会更厚。 作为父级入口，它还额外带来导航变深、读者要多跳几页的成本。
