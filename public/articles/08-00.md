# 第八卷导读：远端不是多一个入口，而是工作台边界被整体改写

## 实现链
- `src/bridge/*` 负责把本地 REPL 会话整体桥到远端，`src/remote/*` 负责把远端消息、权限和重连翻译回本地世界，`src/assistant/*` 负责 viewerOnly 历史阅读，`src/utils/teleport/*` 则负责组织、环境、会话 API 和 seed bundle。
- 所以这一卷的主线不是“怎么联网”，而是代码怎样在跨边界之后重建身份、状态、权限、历史和恢复逻辑。你看到的每一页，背后都对应一个“工作台边界被改写后，哪条制度还得继续成立”的问题。

## 普通做法
- 普通讲法通常会把远端功能写成“多一个远程终端”或“把命令发到云端执行”。
- 这种讲法很容易懂，但它默认本地工作台本身没有被改造。

## 为什么不用
- Claude Code 这套代码明显不是“远程终端外挂”。一旦启用 bridge、remote session 或 assistant view，标题、审批、断线、历史拉取、环境选择和消息翻译都要重算。
- 如果还按“多一个入口”去理解，后面很多看似绕的设计都会被误读成过度工程。

## 代价
- 代价是这卷比普通远程执行教程更重，读者要同时面对桥接层、会话层、权限层和阅读层。
- 但这是必要代价，因为源码里真的就是这么多层在一起改写边界。
