# 第十三卷回收索引：团队世界里比任务世界多出身份和秩序说明团队问题首先是组织问题不是并发问题

## 实现链
团队世界比任务世界多出来的，不只是更多工作量，  
而是身份和秩序本身开始变成主要问题。

这条线真正逼出来的，是第十三卷里关于组织问题先于并发问题的判断。

这页实际回收的实现链是：
- `09-01-02` 把团队世界讲成比任务世界多出身份和秩序，这会在 `13-02-01` 里再次出现，说明团队问题首先不是并发数量，而是组织结构。

团队世界不是比任务世界多一点热闹。  
它先多出来的是身份关系和组织秩序，所以问题也先是组织问题。

## 普通做法
更普通的做法，是把后台动作当成松散的异步句柄，跑起来就算开始，结束了再临时补状态和通知。

## 为什么不用
Claude Code 不这么做，是因为没有统一任务对象，就没有稳定的生命周期、输出、通知和回看链，前后台切换也会变成一串彼此不认账的临时处理。

## 代价
代价是任务状态字段、输出文件、通知协议和前后台转场逻辑都会变厚，维护者要记住更多接缝。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
