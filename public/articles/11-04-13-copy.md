# 选中后的 copy 反馈只占一条旁白，说明复制动作不该抢驾驶舱的主视线

当输入区里有选中内容时，底部偶尔会给一条很轻的 copy 旁白：要么提示 `Ctrl+C`，要么在 xterm.js 环境里提示 native select 的做法。它不是弹窗，也不是一段说明书，只是在最小成本地告诉你当前可用的复制路径。

这类反馈的职责只是把动作收住，不是把界面讲满。驾驶舱里，复制可以被提醒，但不能被抬升成另一条主叙事；一旦提示开始抢戏，输入区就会从操纵台退化成教学页。

## 实现链

`PromptInputFooterLeftSide.tsx` 会根据 `copyOnSelect`、`isXtermJs()` 和当前是否有选区，决定只在底部补一条很轻的 `copy` / `native-select` 提示，而不是弹更重的反馈。复制动作被当成旁白，不是主剧情。

## 普通做法

更普通的实现会在复制后弹 toast，或者长期把复制说明挂在显眼位置，确保用户“看见了系统在说话”。

## 为什么不用

Claude Code 不这么高调，是因为复制不是会改写运行时现实的大动作，不值得抢走驾驶舱主视线。它只在需要时给一条最小提示，提醒你当前该按哪组键。

## 代价

代价是反馈更克制，没看到的人可能会错过；但整体上它把稀缺空间留给了更重要的状态。
