# tabs_context_mcp先看当前世界，再决定要不要新开一页

这条线最有产品味的一点，是它要求先看当前标签页上下文，再决定是否新开页面。换句话说，Claude Code 进入浏览器世界时，不是假设自己从零开始，而是先承认用户已经有一个正在发生的现场。

这种克制很重要。好的浏览器能力不是一上来就乱开新页，而是先理解用户此刻的浏览器到底是什么状态，再顺着那个现场工作。

## 实现链
`BASE_CHROME_PROMPT` 明确要求每次浏览器会话先调 `mcp__claude-in-chrome__tabs_context_mcp`，再决定复用还是新开 tab。`toolRendering.tsx` 又会追踪 `tabId`，并在支持超链接时渲染 `[View Tab]`，把“当前现场”这件事真正落到工具和 UI 上。

## 普通做法
普通自动化常常直接 `newPage()`，或者凭上次记住的 tab id 继续干。

## 为什么不用
Claude Code 不这么做，因为用户真实浏览器不是模型独占的实验场。tab 可能被用户关闭、切换或污染，先读当前上下文比盲开盲用更稳。

## 代价
代价是每次会话都多了一步上下文探测，工具链会比纯脚本化自动化更啰嗦。
