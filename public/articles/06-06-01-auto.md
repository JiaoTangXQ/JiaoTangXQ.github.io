# 进入 auto 前先剥离危险放行规则

## 实现链

这一页重点看“进入 auto 前先剥离危险放行规则”。对应源码主要是 `src/utils/permissions/permissionSetup.ts`。
这一章真正落在 `permissionSetup.ts` 对 auto mode 的进入、剥离和恢复逻辑上。Claude Code 进入 auto 前会先把危险放行规则从上下文里剥掉，离开 auto 时又把旧规则按程序还回去，说明模式切换不是 UI 换色，而是真正重写当前制度。

## 普通做法

更普通的做法，是沿用用户现有 allow 规则，进入自动模式后继续照单全收。这样模式切换很平滑，也不用维护额外的恢复逻辑。

## 为什么不用

Claude Code 没这么做，因为 auto mode 的核心不是“让老权限照常生效”，而是“在自动批准场景下重新定义什么叫安全”。如果危险放行规则不先剥掉，auto 就会继承本来只适合人工盯着看的高风险特权。

## 代价

这种做法更谨慎，能防止自动模式带着旧特权一路扩大风险。代价是模式切换明显更重，规则要暂存、恢复、对账，工程上比一个简单布尔开关复杂得多。
