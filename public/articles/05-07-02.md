# 自动后台化为什么也走同一条转场协议

自动后台化不是“另一套特殊逻辑”，Claude Code 让它和手动后台化走同一条协议，这很重要。

## 实现链

`registerAgentForeground()` 如果收到 `autoBackgroundMs`，会起一个 `setTimeout()`。时间到了以后，它做的事情和手动后台化是同一类动作：先把 `isBackgrounded` 设为 `true`，再 resolve 同一个 `backgroundSignal`。也就是说，自动后台化只是“谁来按下交棒按钮”不同，协议本身没变。

这种写法保证了前后台转场只有一套真逻辑。

## 普通做法

普通做法很容易分叉。手动后台化一套代码，超时自动后台化另一套代码，看起来都能工作，但细节经常不一致。

## 为什么不用

Claude Code 没让自动路径自立门户，是因为转场这种事最怕双轨。只要两条路径稍有差别，用户就会遇到“手动切可以，自动切不行”或者“自动切后丢消息”的诡异问题。

它把自动后台化也塞回同一条协议，本质上是在用结构逼一致性。

## 代价

代价是自动后台化也得遵守整套任务世界约束，写法没法特别省。即使只是一个超时器，也得接进正式生命周期。
