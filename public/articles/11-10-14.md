# BUBBLE_SHOW = 20，FADE_WINDOW = 6：气泡的完整生命周期

`CompanionSprite.tsx` 里控制气泡行为的两个常量：

```typescript
const BUBBLE_SHOW = 20;    // 完整显示的帧数
const FADE_WINDOW = 6;     // 淡出阶段的帧数
```

加上一个用于清除 `companionReaction` 的 `setTimeout`，这三个机制合在一起定义了气泡的完整生命周期：

1. `companionReaction` 被设置 → 气泡出现
2. 计数到 `BUBBLE_SHOW` 帧 → 进入淡出阶段
3. 淡出阶段持续 `FADE_WINDOW` 帧
4. `setTimeout` 到期 → 清除 `companionReaction`，气泡完全消失

## 为什么需要淡出

气泡突然消失（闪一下就没了）会产生视觉上的「跳跃感」——用户视线还停留在那里，内容突然不在了，会有轻微的视觉不适。

淡出让消失变成一个过渡过程：内容逐渐变透明，视线可以平滑地移开，不会产生突然丢失焦点的感觉。

这是终端 UI 里少数可以做动画效果的地方。终端大部分内容是静态文本，没有过渡；气泡的淡出给了一点「这个东西有生命，会优雅离开」的感觉。

## 完整生命周期的设计价值

很多界面里的提示或气泡只考虑「什么时候出现」，不考虑「怎么离开」。结果是：提示突然出现，又突然消失，或者需要用户主动关闭。

`buddy` 的气泡有清晰的生命周期：出现时机、显示时长、淡出过渡、最终消失。整个过程用户不需要干预，气泡自己管理自己的生死。

这是一种「懂礼貌」的体验层设计：出现时有存在感，离开时不打扰。体验层的退出，和它的登场一样，需要被认真设计。
