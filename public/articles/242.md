# 第十三卷回收索引：前四卷里的能提前的提前，必须收口的收口

## 实现链
前四卷里还有一条线，专门回答：能提前的事情可以提前，但该收口的地方不能放开。

这条线真正逼出来的，是第十三卷里关于制度先行的后半段判断。

这页实际回收的实现链是：
- `02-01-03` 把抢跑式预取写成“能提前的提前，必须收口的收口”，这会继续支撑 `13-04-01`，说明制度先行，而不是盲目求快。

能提前的可以提前。  
必须收口的，还是得先收口，再把后面的动作打开。

## 普通做法
更普通的做法，是把启动、输入、主循环里的这些小判断留在顺手逻辑里，不单独立规矩。

## 为什么不用
Claude Code 不这么做，是因为工作台一旦要长期运行，这些看似前期的小处理其实都是后面秩序能否站稳的根，不能等出事了再补。

## 代价
代价是前期代码和解释都会显得更重，也更早暴露出长期工程的野心。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
