# Claude Code 启动链路地图

启动链不是“把界面点亮”这么简单，而是先决定这次会话要按什么制度活。

## 实现链

启动主线从 `main.tsx` 开始，先处理运行模式、环境和初始状态，再进入 `interactiveHelpers.tsx` 这类 setup 和交互准备层，随后由 `commands.ts` 装配命令入口，同时把技能、插件命令和 feature gate 接上。也就是说，Claude Code 在真正进入主循环前，就已经决定了命令空间、权限姿态和部分运行环境。

## 普通做法

普通 CLI 的启动一般是：解析参数、创建进程上下文、进入主函数，剩下的都等用到再说。

## 为什么不用

Claude Code 不敢这么晚才做准备，因为它不是一次性脚本，而是长期会话工作台。权限模式、命令入口、动态能力和 setup 页面如果都拖到后面，用户会在第一轮交互里遇到大量“临时补装”的感觉。

所以它把很多决定前置到启动期，让后续主循环在一个已经成型的制度里运行。

## 代价

代价是启动层显得很重，很多慢操作和特性判断都要尽早发生，入口代码比普通 CLI 更复杂。
