# socket 目录先清陈旧再收权限，说明浏览器桥也要先做卫生

浏览器桥启动时，不是先忙着接消息，而是先把 socket 目录的老垃圾清掉，把目录权限收紧，再给 socket 文件本身设更严的权限。这个顺序很关键，因为桥一旦通向真实浏览器，卫生和边界就不再是附属工作，而是正式底座。

这种细节最能区分“能跑”和“能长期跑”。Claude Code 在这里先做清理、权限和陈旧连接回收，说明它对浏览器桥的期待不是临时玩具，而是长期设施。

## 实现链
`chromeNativeHost.ts` 在启动时会先检查 socket 目录是不是目录、清理死掉 PID 留下的 `.sock`、再把目录权限收成 `0700`、socket 权限收成 `0600`。接入链先做卫生，再开门。

## 普通做法
普通做法是直接监听一个 tmp 路径，旧文件留着也无所谓。

## 为什么不用
Claude Code 不这么做，因为浏览器桥是长期设施。陈旧 socket 和松散权限不只是脏，还会让连接错路或泄露到其他进程。

## 代价
代价是启动时要多做一次扫描和清理，代码也必须理解 PID 存活这种系统细节。
