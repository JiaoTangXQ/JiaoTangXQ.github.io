# 跨子系统事实必须住进同一个状态世界

一个权限请求会改消息区、输入区和模式提示，一个任务变化会改任务面板、通知和主对话流，一个远端或 bridge 变化又会改边界和 footer。Claude Code 很清楚，这些都不是局部事实。

所以它把跨子系统联动的东西集中进同一个 `AppState` 世界里，再让各组件只订阅自己关心的切片。这样界面才不会互相打架。

## 实现链

`AppState` 同时收任务、插件、MCP、远端、通知、输入模式、队列提示和面板状态，说明这些事实被要求同住一个 store。Claude Code 没把它们拆成互不相认的子 store，因为这些子系统经常一起决定用户眼前到底该看到什么、能做什么。

## 普通做法

更容易想到的写法，是每个子系统维护自己的 store，再靠 props、context 或事件总线互相同步。

## 为什么不用

Claude Code 不愿意让这些事实散住，是因为它们的边界不是静态的。比如权限模式会改写可用工具，远端状态会改写底部提示，任务状态会改写输入区和通知；分散存储只会把同步问题往后推。

## 代价

代价是共同状态世界明显更重，也更容易让人担心“是不是太大了”；但至少这里的大，是为了保同一个真相。
