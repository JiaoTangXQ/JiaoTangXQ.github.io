# 第十三卷回收索引：扩展世界里接入链处理真实安装、桥接和现场形状

## 实现链
扩展世界里更硬的现实，不在制度口号，  
而在接入链到底要处理多少真实安装、桥接和现场形状。

这条线真正逼出来的，是第十三卷里关于工程现实的判断。  
它说明接入链真正要处理的不是纯理论，而是真实用户面对的安装和现场。

这页实际回收的实现链是：
- `13-06-01` 继续把这条线钉牢：接入链真正要处理的不是纯理论，而是真实用户面对的安装、桥接和现场形状。

成熟的扩展不是假装接入过程永远漂亮。  
它是承认现实接入链本来就会长出一些不太优雅、但必须存在的形状。

## 普通做法
更普通的做法，是每种来源保留自己的装配方式和运行时语言，等真的冲突了再临时做桥接。

## 为什么不用
Claude Code 不这么做，是因为来源越多，越要先学旧骨架的语言；否则系统会被多套协议、多套入口和多套制度动作拖散。

## 代价
代价是注册、翻译和回收线都会变多，很多地方看起来像是在重复做统一。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
