# 粘顶点击后的抑制状态机会先挡住自我刷新，说明修正动作也要防止互相打架

## 实现链
StickyTracker 用 suppress = armed/force/none 这套小状态机挡住点击后的自我刷新。onClick 触发后，先抑制下一帧的 sticky 计算，再让 correction effect 接管。

## 普通做法
普通实现会点击后直接让滚动和 sticky 计算继续自由跑。那样 header 很容易在跳转中途又冒出来，自己和自己打架。

## 为什么不用
这里修正动作也要防止互相打架，因为 clicked 不是终点，只是过渡态。没有抑制位，header 就会在跳转时反复重建。

## 代价
代价是多了一层顺序敏感的状态机，读代码时得同时看 effect 的先后顺序。好处是点击 sticky prompt 的回跳不会被自己的更新反咬一口。
