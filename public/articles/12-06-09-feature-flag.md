# 自动启用要同时满足交互、已装扩展和 feature flag，说明便利不能越过现实闸门

Claude Code 并没有因为“自动启用”听起来方便，就把浏览器能力悄悄全开。它要同时满足正在交互、扩展已经装好，而且 feature flag 也同意，才会把这条线自动点亮。也就是说，便利在这里从来不是最高原则，现实闸门才是。

这很能说明它的产品气质。成熟系统会尽量少打扰人，但不会为了少打扰就越过真实前提。自动化只在条件都成立时出现，才叫体面。

## 实现链
`shouldAutoEnableClaudeInChrome()` 只有在交互 session、扩展已检测到、且满足 `ant` 或对应 feature flag 时才会返回真。自动启用不是“用户装了就强行开启”，而是经过多道现实闸门。

## 普通做法
偷懒做法是只要发现扩展就默认启用。

## 为什么不用
Claude Code 不这么干，因为自动启用一旦跑到不该跑的环境里，最先受伤的是冷启动、用户预期和权限边界。

## 代价
代价是用户会看到“同样装了扩展，为什么这台机器自动开、那台没自动开”的差异。
