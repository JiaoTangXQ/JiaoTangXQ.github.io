# wrapper 脚本固定落在 `~/.claude/chrome/` 且内容不变就不重写，说明长期设施讲固定落点和少惊动

wrapper script 不只是“能跑起来”就行。Claude Code 给它安排了固定落点 `~/.claude/chrome/`，而且如果内容没变，就直接复用现有文件，不再重写。这种设计背后传达的是同一种长期设施观：落点要稳定，环境要少惊动。

越是会被系统长期依赖的小设施，越需要这种克制。否则今天重写一次、明天抖一下路径，最后苦的都是排查问题的人。

## 实现链
`createWrapperScript()` 固定把脚本放在 `~/.claude/chrome/`，文件内容相同就直接复用旧文件。这说明它把 wrapper 当长期设施，而不是一次性临时文件。

## 普通做法
普通做法是每次启动在临时目录生成一个随机脚本。

## 为什么不用
Claude Code 不用临时脚本，因为 manifest 一旦指向某个路径，这个路径就应该尽量稳定。路径漂移只会增加浏览器侧不确定性。

## 代价
代价是用户目录里会长期存在这份设施文件，而且代码要额外维护“稳定路径 + 少惊动”的原则。
