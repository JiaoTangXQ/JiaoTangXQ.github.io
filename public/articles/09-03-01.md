# 同进程不是不隔离，而是换一种隔离手法

很多人一看到 in-process teammate，就会以为 Claude Code 放弃隔离了。实际上它只是把隔离从“进程边界”换成了“上下文边界”。

## 实现链

`spawnInProcessTeammate()` 会先创建 `TeammateContext` 和独立 `AbortController`。之后 `inProcessRunner.ts` 用 `runWithTeammateContext()` 包住执行，再在更里层用 `runWithAgentContext()` 做 agent 身份归因。也就是说，同进程 teammate 不是裸跑，而是套着专门的 AsyncLocalStorage 上下文壳。

## 普通做法

普通做法一旦走同进程，常常默认大家共享全局状态，然后靠命名约定别互相踩。

## 为什么不用

Claude Code 不敢这么放飞，是因为它要让同进程 teammate 尽量保留“像独立成员一样工作”的感觉。没有上下文隔离，日志、身份、权限归因和消息路由都会搅在一起。

所以它承认：不用进程也可以隔离，但要用更严格的上下文协议来换。

## 代价

代价是实现难度更高。AsyncLocalStorage、上下文传播和回调边界都比单纯开新进程更容易出错。
