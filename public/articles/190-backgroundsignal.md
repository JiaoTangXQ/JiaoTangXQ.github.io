# 第十三卷回收索引：任务世界里 `backgroundSignal` 是交棒信号，不是硬打断

## 实现链
任务世界里前台和后台之间最容易被误会的一点，是把切换看成硬切断。  
但系统真正要做的，不是断裂，而是接力。

这条线真正逼出来的，是第十三卷里关于前后台关系和统一制度动作的判断。  
`backgroundSignal` 的关键，不在打断，而在交棒。

这页实际回收的实现链是：
- `05-07-01` 把 `backgroundSignal` 写成交棒信号而不是硬打断，这会在 `13-04-02` 里回收到“前台和后台之间不是断裂关系，而是接力关系”。

很多系统会把前台切后台理解成一刀切。  
Claude Code 更克制，它把 `backgroundSignal` 做成交棒信号，让前后台维持接力而不是断裂。

## 普通做法
更普通的做法，是把后台动作当成松散的异步句柄，跑起来就算开始，结束了再临时补状态和通知。

## 为什么不用
Claude Code 不这么做，是因为没有统一任务对象，就没有稳定的生命周期、输出、通知和回看链，前后台切换也会变成一串彼此不认账的临时处理。

## 代价
代价是任务状态字段、输出文件、通知协议和前后台转场逻辑都会变厚，维护者要记住更多接缝。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
