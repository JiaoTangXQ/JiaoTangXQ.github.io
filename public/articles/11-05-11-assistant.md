# assistant 模式直接隐藏整条状态栏，说明错现实比少信息更糟

有些模式下，状态栏里那些模型、权限、目录字段反映的是外层进程，而不是眼前真正干活的那个 agent。这里没有硬着头皮继续显示，而是干脆整条隐藏。

这说明摘要层首先要忠于现实。少说一点还能忍，拿错世界的事实来安慰人，反而更糟。

## 实现链

`statusLineShouldDisplay()` 里直接规定 assistant/kairos 模式隐藏整条状态栏，因为这时本地进程的 model、cwd、permission mode 并不代表真正执行中的 agent。这里不是“暂时没内容”，而是主动承认这条摘要已经不再忠于现场。

## 普通做法

更普通的产品会觉得能显示总比不显示强，哪怕字段只代表外层壳子，也先亮给用户看。

## 为什么不用

Claude Code 不接受半真半假的摘要，因为用户会拿这些字段做操作判断。错现实比少信息更糟，所以它宁可整条隐藏。

## 代价

代价是 assistant 模式下底部一整层信息直接消失，用户得换用别的线索；但系统至少没有继续说错话。
