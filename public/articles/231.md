# 第十三卷回收索引：任务世界里任务完成要重新编码成结构化通知

## 实现链
任务完成如果只停在后台对象里，主会话并不会自动知道发生了什么。  
这里真正关键的，不是“做完”这个词本身，而是结果必须重新变成主会话可消费的结构化通知。

这页实际回收的实现链是：
- `05-08-01` 把任务完成重新编码成结构化通知，再送回消息队列，这会在 `13-02-02` 里回收到“后台劳动最终还得重新说回主会话语言”。

很多系统把“完成”当作终点。  
Claude Code 更像一套会话系统，它把完成重新编码成通知，让主会话能接住结果，而不是只在后台留下一个结束标志。

## 普通做法
更普通的做法，是把后台动作当成松散的异步句柄，跑起来就算开始，结束了再临时补状态和通知。

## 为什么不用
Claude Code 不这么做，是因为没有统一任务对象，就没有稳定的生命周期、输出、通知和回看链，前后台切换也会变成一串彼此不认账的临时处理。

## 代价
代价是任务状态字段、输出文件、通知协议和前后台转场逻辑都会变厚，维护者要记住更多接缝。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
