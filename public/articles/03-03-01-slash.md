# 面试题：用户输入 /commit，系统第一步做什么？不是执行命令。

## 场景推演

用户在 Claude Code 终端里输入 `/commit`，按 Enter。

假设你是面试官，你会怎么追问「系统第一步做什么」？

大多数候选人的回答：「检查 `/commit` 是不是一个注册的命令，如果是就执行它。」

这个答案在普通 CLI 工具里是对的。在 Claude Code 里，它发生在第五步。

## 实际的第一步到第五步

### Step 1：`setUserInputOnProcessing`

```typescript
if (mode === 'prompt' && inputString !== null && !isMeta) {
  setUserInputOnProcessing?.(inputString)
}
```

把用户输入立刻显示在 UI 上（「显示正在处理的提示词」）。这一步在所有处理之前完成，所以界面感觉是即时响应的，即使后面有 resize 和 I/O 操作。

### Step 2：图片处理

对于 `/commit`，`input` 是字符串，不是 ContentBlockParam 数组，所以这一步快速跳过。但如果用户同时粘贴了截图再输入 `/commit`，这里就会真实执行图片 resize。

### Step 3：bridge 检查 → effectiveSkipSlash

```typescript
let effectiveSkipSlash = skipSlashCommands
if (bridgeOrigin && inputString !== null && inputString.startsWith('/')) {
  const parsed = parseSlashCommand(inputString)
  const cmd = parsed ? findCommand(parsed.commandName, ...) : undefined
  if (cmd) {
    if (isBridgeSafeCommand(cmd)) {
      effectiveSkipSlash = false
    } else {
      const msg = `/${getCommandName(cmd)} isn't available over Remote Control.`
      return { messages: [...], shouldQuery: false }  // 直接拦截
    }
  }
}
```

如果这条 `/commit` 来自 bridge（手机端发来的），系统先检查 `/commit` 是不是 bridge 安全命令。`BRIDGE_SAFE_COMMANDS` 里没有 `commit`，所以如果是 bridge 来源，这里会直接返回拦截消息，命令根本不会执行。

如果是本地输入，`bridgeOrigin` 为 false，这一步快速跳过，`effectiveSkipSlash` 保持 `skipSlashCommands` 的值（通常是 false）。

### Step 4：ultraplan 检测

`/commit` 以 `/` 开头，不满足 `!inputString.startsWith('/')` 条件，ultraplan 检测跳过。

### Step 5：附件提取

```typescript
const shouldExtractAttachments =
  !skipAttachments &&
  inputString !== null &&
  (mode !== 'prompt' || effectiveSkipSlash || !inputString.startsWith('/'))
```

slash 命令的附件提取逻辑在 `getMessagesForSlashCommand` 内部处理，所以这里 `shouldExtractAttachments` 为 false，外层跳过附件提取。

### Step 6：终于到了 slash 分支

```typescript
if (inputString !== null && !effectiveSkipSlash && inputString.startsWith('/')) {
  const { processSlashCommand } = await import('./processSlashCommand.js')
  const slashResult = await processSlashCommand(inputString, ...)
  return addImageMetadataMessage(slashResult, imageMetadataTexts)
}
```

注意这里是动态 import（`await import('./processSlashCommand.js')`）——slash 处理逻辑是懒加载的，只在真正需要时才引入这个模块。这是一个启动性能优化。

## 这套顺序保护了什么

`/commit` 在本地终端输入，会顺利执行。同样的 `/commit` 从手机端 bridge 发来，会在 Step 3 被拦截，用户看到「/commit isn't available over Remote Control」。

同一个命令字符串，走不同入口，在 Step 3 就产生了分歧。这说明「命令名」和「命令执行」之间有一层入口语境的判断——slash 分支不是第一道门，而是经过入口检查后才能进入的第二个世界。

---

*面试指导：被问到「slash 命令是怎么触发的」时，不要从「检查 / 前缀」开始说。从「bridge 安全检查决定是否允许执行」这个更早发生的判断切入，能体现对系统安全边界的真实理解。*
