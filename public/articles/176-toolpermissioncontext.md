# 第十三卷回收索引：治理世界里的 `ToolPermissionContext` 随身携带制度快照说明能力一直活在制度里

## 实现链
治理世界另一半更细的判断，不是能力进不进系统，  
而是能力在运行时是不是一直带着制度。

这条线真正逼出来的，是第十三卷里关于制度先行和持续约束的判断。

这页实际回收的实现链是：
- `06-03-02` 把 `ToolPermissionContext` 讲成随身携带的制度快照，这正是 `13-04-01` 能说“制度先行”的直接基础。

能力不是先脱离制度裸奔，再靠事后补丁收回去。  
它从一开始就应该带着制度快照一起运行。

## 普通做法
更普通的做法，是每个能力各带一点本地校验，或者干脆给用户一个粗糙的总开关，把细节风险留给人自己判断。

## 为什么不用
Claude Code 不这么做，是因为强能力一旦缺少统一制度快照和变更流程，就会在最危险的地方出现口径不一致，最后连改规则本身都没有共同程序。

## 代价
代价是 permission context、settings 扇出、hooks 和模式契约都更重，读起来也更像制度系统。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
