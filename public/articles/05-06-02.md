# 先有同一任务对象，后有前台后台之分

Claude Code 不是先决定“这是前台还是后台”，而是先决定“这是不是同一个任务对象”。

## 实现链

`registerAsyncAgent()` 和 `registerAgentForeground()` 最终都产出 `LocalAgentTaskState`。差别主要只是 `isBackgrounded` 初值不同，以及前台版本额外返回 `backgroundSignal`。任务的 `id`、`outputFile`、`progress`、`pendingMessages`、`retain`、`evictAfter` 这些关键字段并没有换一套模型。

这说明前台和后台在 Claude Code 里更像同一任务的两种运行姿态，而不是两类不同东西。

## 普通做法

更容易想到的做法，是前台任务一套模型，后台任务另一套模型。前台专注交互，后台专注异步，两边各管各的。

## 为什么不用

Claude Code 不想这么分，是因为一旦模型不同，前后台切换就会变成迁移。你要复制字段、映射状态、转运消息，越写越像两套系统之间做同步。

它宁可一开始就承认前台后台只是同一任务的不同姿态，这样转场时改几个状态位就够了。

## 代价

代价是任务对象要同时照顾前台和后台需求，字段会看起来有点臃肿。有些字段在某一姿态下暂时用不上，但系统接受了这种冗余来换转场平滑。
