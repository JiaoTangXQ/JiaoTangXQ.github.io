# 第十三卷回收索引：界面世界里跨子系统事实进入同一个 `AppState` 世界，状态才真正有地方落屏

## 实现链
界面世界的另一半，不是界面本身怎么被承认，  
而是事实最终往哪里安放。

这条线真正逼出来的，是跨子系统事实必须先进入同一个状态世界，显示层才不会漂着。

这页实际回收的实现链是：
- `11-02-02` 把跨子系统事实收进同一个 `AppState` 世界，这会在 `13-05-02` 里回收到“只有先承认界面是运行时一部分，状态世界才真的有地方落屏”。

`AppState` 不是为了好看，而是为了让事实有一个共同落点。  
没有这个共同世界，界面就只能展示碎片，不能承接状态。

## 普通做法
更普通的做法，是把界面当展示壳，状态散落在各个组件或服务里，最后再临时拼起来给人看。

## 为什么不用
Claude Code 不这么做，是因为跨子系统事实如果没有同一个落屏世界，界面很快就会出现各说各话的假真相，长会话尤其容易失控。

## 代价
代价是 AppState 会更大，显示层也得承担一部分制度、接缝和注意力管理逻辑。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
