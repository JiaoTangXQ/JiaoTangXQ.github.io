# 任务和 team 提示会按有无团队切换，说明摘要层也在做条件分发

底部区不是固定地把所有提示排成一串。有没有团队、现在是否还在 loading、是否还有 agent 在跑，都会改写 `esc`、`stop agents`、`show tasks` 这些提示该怎么出现。也就是说，摘要层不仅在显示状态，还在做一层条件分发。

这很像一个成熟的前台调度台。它不会把不合时宜的动作塞给你，而是让此刻真正有意义的入口浮上来。

## 实现链

footer 左侧会根据 `hasTeams`、`hasBackgroundTasks`、`isViewingTeammate`、`hasTeammatePills` 等条件动态拼 `parts` 和 `tasksPart`。任务提示和团队提示不是固定模板，而是在底部实时做条件分发。

## 普通做法

更简单的摘要栏会把所有可能项都堆上去，或者用一套固定模板兼容所有场景。

## 为什么不用

Claude Code 不这么堆，是因为单 agent 场景和团队场景需要的提示根本不同。要是还用同一套底栏，用户不是被噪声淹没，就是缺关键入口。

## 代价

代价是 footer 不再只是展示层，而像一台小型编排器；但它因此更贴近当前协作场景。
