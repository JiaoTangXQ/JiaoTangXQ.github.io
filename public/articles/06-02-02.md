# 模式名称背后其实是不同的人机契约

## 实现链

这一页重点看“模式名称背后其实是不同的人机契约”。对应源码主要是 `src/utils/permissions/PermissionMode.ts`、`src/utils/permissions/permissions.ts`。
这一章真正落在 `PermissionMode.ts` 里那套统一模式定义上：模式不只是一串字符串，还带着标题、符号、颜色和外部映射关系。Claude Code 用它来声明“当前会话究竟按哪种契约运行”，所以模式本身就是运行时状态，而不是 UI 装饰。

## 普通做法

更直觉的做法，是给不同危险等级加几个零散开关，或者让每个工具自己解释要不要弹窗。这样很灵活，也容易局部加功能。

## 为什么不用

Claude Code 没这么做，因为零散开关表达不了完整的人机契约。它需要的是一个全局姿态，好让状态栏、工具审批、桥接层和 SDK 接口都能说同一种语言。

## 代价

这样做的好处是系统口径一致。代价是模式名不再只是“开或关”，而是变成一套需要学习的制度词汇，理解门槛会明显高于普通开关。
