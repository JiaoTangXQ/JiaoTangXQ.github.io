# Claude Code 主干调用链索引

这页只做一件事：把最重要的几条调用主干钉出来，告诉你看代码时先顺哪几根线。

## 实现链

最重要的主干有四条：

- 启动链：`main.tsx` -> 设置/入口判断 -> `interactiveHelpers.tsx` -> `commands.ts`
- 查询链：`QueryEngine.submitMessage()` -> `processUserInput()` -> `query()` -> 模型/工具/压缩
- 工具链：`Tool.ts` 协议 -> `tools.ts` 装配 -> `services/tools/toolExecution.ts`
- 任务链：`AgentTool.tsx` -> `Task.ts`/`framework.ts` -> `LocalAgentTask` 或 teammate runner

这四条线比目录索引更重要，因为它们把“程序怎么启动”“一轮怎么跑”“工具怎么做事”“后台和团队怎么成立”分别串起来了。

## 普通做法

更普通的索引会按文件名字母序或目录树顺序列文件，让读者自己拼调用关系。

## 为什么不用

Claude Code 不适合那种读法，因为主干跨目录太多。只看目录顺序，你会在 `components`、`utils`、`services` 之间来回跳，根本不知道哪些是核心链，哪些只是支撑层。

所以这里直接给调用主干，而不是让读者自己在文件海里摸索。

## 代价

代价是这张索引不会覆盖每个文件，甚至会故意忽略很多外围代码。它的目标不是完整，而是优先级正确。
