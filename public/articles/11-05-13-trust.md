# trust 挡住状态栏命令时也要发提醒，说明静默失败会伪装成一切正常

状态栏命令如果因为 trust 没通过而根本没跑成，界面上最危险的不是报错太吵，而是什么都没有，看起来像只是“暂时没内容”。所以这里会补一条低优先级提醒，明确告诉你它被跳过了。

这说明摘要层不仅要汇报成功，也要汇报“本来该有、但现在没有”的那种缺席。

## 实现链

`StatusLine.tsx` 发现 workspace trust 没通过时，不会静默跳过状态栏命令，而是主动发一条 `statusline-trust-blocked` 通知，并写 debug log。这里把“本来该有摘要，但现在被 trust 拦住了”也当成应被看见的事实。

## 普通做法

更普通的 hook 失败方式是吞掉错误，让界面默默空着，默认用户不会太在意。

## 为什么不用

Claude Code 不接受这种静默失败，因为用户会误以为只是暂时没信息，而不是有一道制度在挡。既然 trust 会直接改写摘要结果，就必须把这个挡板说明白。

## 代价

代价是状态栏逻辑要自己处理失败通知，不再只是消费命令输出；但可诊断性明显更高。
