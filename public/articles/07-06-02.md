# 变量替换把静态分发和运行时现场接了起来

## 实现链
- `loadPluginCommands.ts` 会在解析 `allowed-tools` 和正文时调用 `substitutePluginVariables`、`substituteUserConfigInContent`，把 `${CLAUDE_PLUGIN_ROOT}` 之类变量替换成当前安装路径和用户配置。
- 这样插件作者可以写相对稳定的模板，系统在实际装配时再把它翻译成当前机器、当前安装位置上的真实命令。

## 普通做法
- 普通做法是要求插件内容完全静态，或者干脆让作者自己写绝对路径。
- 这在单机自用场景里问题不大，因为环境不会经常变。

## 为什么不用
- Claude Code 不敢假设插件装在哪都一样，因为插件可能来自缓存、seed 目录、不同 marketplace，路径和选项都可能变化。
- 如果不做变量替换，插件要么写死本地现实，要么逼用户手动改文案，二者都不适合正式生态。

## 代价
- 代价是调试难度变高，因为命令真正执行时看到的内容已经不是磁盘上那份原文。
- 但这笔代价换来了可移植性，插件至少不用为每一种安装位置重写一遍。
