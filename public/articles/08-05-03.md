# 远端翻译层还得接住压缩和工具闭环，说明它在护会话连续性

## 实现链
- `useRemoteSession.ts` 会把远端 SDK 消息交给 `sdkMessageAdapter.ts` 转成本地消息，同时还会处理 `compact_boundary`、tool results、task_started/task_notification 和 init 消息里的 slash commands。
- 这说明远端翻译层不是单纯的“格式转换器”，它还在护住会话连续性：压缩发生了要标边界，工具结束了要清掉进度，后台任务变化了要改本地计数。

## 普通做法
- 普通做法是把远端消息原样显示出来，顶多做一点 JSON 到文本的转化。
- 那样协议层和界面层的耦合最少。

## 为什么不用
- Claude Code 没停在原样透传，是因为本地 REPL 不是日志窗口，而是一套会根据消息类型更新状态的工作台。
- 如果翻译层不接住 compaction、tool_result 和 task 生命周期，本地 UI 看到的就只是碎片，不是连续工作现场。

## 代价
- 代价是 adapter 和 hook 都得理解更多协议细节，维护成本高于简单透传。
- 但这笔成本直接换来“远端看起来仍像本地工作台在继续工作”。
