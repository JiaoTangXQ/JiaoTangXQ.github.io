# 第十三卷回收索引：前四卷里的技能正文、工具许可和 `hooks` 一起制度化装配进当前会话

## 实现链
前四卷里还有一条线，专门回答“工作方式为什么要先制度化装配”。

这页专门收后半段：技能正文、工具许可和 `hooks` 一起制度化装配进当前会话。

这页实际回收的实现链是：
- `03-06-02` 把技能正文、工具许可和 hooks 一起送进当前会话，这会继续支撑 `13-04-01`，说明工作方式本身也要被制度化装配。

工作方式不是先随手跑起来，再慢慢修。  
它要把技能正文、工具许可和 `hooks` 一起装进当前会话，系统才知道该怎么守住层次。

## 普通做法
更普通的做法，是把启动、输入、主循环里的这些小判断留在顺手逻辑里，不单独立规矩。

## 为什么不用
Claude Code 不这么做，是因为工作台一旦要长期运行，这些看似前期的小处理其实都是后面秩序能否站稳的根，不能等出事了再补。

## 代价
代价是前期代码和解释都会显得更重，也更早暴露出长期工程的野心。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
