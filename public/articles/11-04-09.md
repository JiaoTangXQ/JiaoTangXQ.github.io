# 任务通知最多只露三行，说明捷报也不能淹没主操纵台

后台任务连续完成时，排队区不会把每一条完成通知毫无节制地往上堆。它最多只露出三行，再把后面的结果折成一条“还有多少任务完成了”的摘要。这个决定很成熟，因为它承认好消息同样会形成噪音。

驾驶舱的中心任务不是庆功，而是继续操纵。Claude Code 愿意牺牲一点“全量展示”的痛快，换来主操纵台不被完成捷报冲垮。真正稳的界面，连成功都要学会克制。

## 实现链

`PromptInputQueuedCommands.tsx` 把任务通知上限定成 `MAX_VISIBLE_NOTIFICATIONS = 3`，超出的部分会被 `createOverflowNotificationMessage()` 汇总成一条“还有多少条”的摘要。即便是好消息，也得服从主操纵台的屏幕预算。

## 普通做法

更普通的做法，是来几条通知就渲染几条，顶多靠滚动把它们塞进去。

## 为什么不用

Claude Code 不愿意让捷报把输入区淹掉，是因为底部这块区域首先要保证用户还能继续操作。通知的职责是提醒，不是接管驾驶舱。

## 代价

代价是细节会被折成一条汇总，用户得再下钻才能看全；但至少输入区不会因为后台太热闹而失去可操作性。
