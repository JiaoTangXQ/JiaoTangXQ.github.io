# 第十三卷回收索引：扩展世界里再多来源也先汇回同一种运行时语言

## 实现链
扩展世界里最容易散的，不是来源多本身，  
而是来源一多，系统就开始偷偷让不同东西说不同话。

这条线真正逼出来的，是第十三卷里关于层级秩序的判断。  
新世界要站得住，先得学会旧骨架的语言。

这页实际回收的实现链是：
- `07-01-02` 把扩展解释成“再多也得汇回同一种运行时语言”，这会在 `13-02-01` 里回收到“层级秩序之所以站得住，是因为新世界先学会说旧骨架的语言”。

再多来源进来，也不能让系统各说各话。  
它们得先学会同一种运行时语言，后面的统一才有底。

## 普通做法
更普通的做法，是每种来源保留自己的装配方式和运行时语言，等真的冲突了再临时做桥接。

## 为什么不用
Claude Code 不这么做，是因为来源越多，越要先学旧骨架的语言；否则系统会被多套协议、多套入口和多套制度动作拖散。

## 代价
代价是注册、翻译和回收线都会变多，很多地方看起来像是在重复做统一。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
