# 第十三卷回收索引：前四卷里的连续性本身就是时间制度

## 实现链
连续性不是后来补上的工作习惯。  
它本身就是时间制度的一部分。

这条线真正逼出来的，是第十三卷里关于长期工作台的判断。

这页实际回收的实现链是：
- `01-04-04` 把连续性讲成时间制度，这会在 `13-07-02` 里回收到“把 agent 做成长久工作台”的总观念。

连续性不是“还能继续”，而是“时间本来就该这样被组织”。  
一旦这么看，长期工作台就不是额外加上的能力，而是时间制度本身的自然结果。

## 普通做法
更普通的做法，是把启动、输入、主循环里的这些小判断留在顺手逻辑里，不单独立规矩。

## 为什么不用
Claude Code 不这么做，是因为工作台一旦要长期运行，这些看似前期的小处理其实都是后面秩序能否站稳的根，不能等出事了再补。

## 代价
代价是前期代码和解释都会显得更重，也更早暴露出长期工程的野心。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
