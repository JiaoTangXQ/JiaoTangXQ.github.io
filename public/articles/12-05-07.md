# 打开链接前先探测可用浏览器并按平台礼法落地，说明产品动作也服从宿主习惯

Claude Code 在打开 Chrome 相关页面时，并不是不分平台地硬调一个命令，而是先探测当前最合适的浏览器，再按 macOS、Windows、Linux 各自的礼法去打开。也就是说，连“帮你开个页面”这种小动作，它都先尊重宿主世界的习惯。

这很能说明它不是把浏览器当玩具。真正的产品级接入，连最后这一步动作都要讲究体面，不给宿主环境添多余的别扭。

## 实现链
`detectAvailableBrowser()` 会按平台和优先级检查 Chrome、Brave、Arc、Edge 等浏览器；`openInChrome()` 再分别走 macOS `open -a`、Windows `rundll32`、Linux/WSL 的浏览器 binary。这不是简单打开 URL，而是按宿主操作系统的礼法落地。

## 普通做法
普通做法是永远调用同一个 `xdg-open` 或硬编码一个浏览器二进制。

## 为什么不用
Claude Code 不这么做，因为它不是在自己的沙盒里运行，用户机器上装的浏览器和系统习惯都不一样。

## 代价
代价是浏览器探测矩阵和平台分支会变多，维护成本不小。
