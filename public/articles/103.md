# 第十三卷回收索引：治理世界里的归还用户习惯与历史特权

## 实现链

这页具体回收的判断是“治理世界—归还用户习惯与历史特权”。
这页回收的实现链，真正落在 `ToolPermissionContext`、`tools.ts` 里的筛选逻辑、`query.ts` 获取权限上下文、以及 permissions/rules/hooks/settings 这些制度对象上。Claude Code 不是等能力已经跑出去了再补一道治理，而是先给能力套上制度语境，再决定它此刻能不能出现、能不能继续、变更规则又该走什么程序。

## 普通做法

更常见的做法，是把权限写成零散的 if/else，把配置视为静态选项，运行时只在敏感动作前做一次简单检查。这样足够支撑中小型工具系统，而且代码更短。

## 为什么不用

Claude Code 没采用这种轻量路线，是因为它管理的不是一两个危险按钮，而是一整套会持续调用工具、还会修改自身制度的运行时。要是权限、规则、hooks、settings 不是常驻语境，系统就会出现“能力先发生、制度再追认”的漏洞。

## 代价

这种做法确实更稳，因为制度不是贴边提醒，而是运行时骨架的一部分。代价是概念层比普通 CLI 厚很多，`ToolPermissionContext` 这类对象也会让初学者觉得系统过于郑重。
