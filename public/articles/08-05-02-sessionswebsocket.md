# SessionsWebSocket把断线重连当成日常，不当成事故

## 实现链
- `SessionsWebSocket.ts` 明确区分永久关闭码和可重试关闭码：`4003` 直接停，`4001` 只给有限次重试窗口，其他情况按最大重试次数和固定延迟重连。每次连接还都会重新取 token，并维持 ping。
- 这套代码默认网络抖动和 session 短暂不可见是日常，而不是每次都该当成致命事故。

## 普通做法
- 普通实现要么一掉线就彻底断开，要么无脑无限重连。
- 两种写法都很常见，因为状态机最少。

## 为什么不用
- Claude Code 没走这两端，是因为远端会话有些错误确实不可恢复，但像 compaction 窗口里的 `session not found` 又可能只是暂态。
- 如果全部当事故，就会过于脆弱；如果全部无限重连，又会在永久错误上耗死。

## 代价
- 代价是 close code 语义和重试预算都要维护，代码比普通 WebSocket 包装更复杂。
- 但对远端工作台来说，这种细致状态机比“一根线断了就全完”成熟得多。
