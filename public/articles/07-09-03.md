# 先注册再取用说明小桥接层在替依赖图挡灾

## 实现链
- `registerMCPSkillBuilders()` 在启动时先把构建器写进去，后面任何地方需要时再 `getMCPSkillBuilders()` 取出。如果没注册成功，它宁可明确抛错，也不假装系统还能继续。
- 这说明小桥接层真正守护的是初始化顺序和依赖图安全，而不是业务逻辑本身。

## 普通做法
- 普通做法是第一次用到时再 import，或者缺了就默默返回空值。
- 这样调用方代码会更短，也更像典型工具函数。

## 为什么不用
- Claude Code 没用“缺了就算了”的态度，因为这里缺的不是某个可选装饰，而是 MCP skill 整条翻译链的入口。
- 一旦桥接器没准备好却继续往下跑，后面看到的就不是正常降级，而是一串更难查的结构性错误。

## 代价
- 代价是初始化顺序变成一条硬约束，错误也会更早暴露。
- 不过这种早暴露通常是好事，因为它把问题拦在边界，而不是让错误一路渗到业务层。
