# 第十三卷回收索引：远端边界世界里宁可接受翻译损耗也要保住统一消息世界

## 实现链
远端世界里真正难的，不是完全没有损耗，而是决定哪些统一性值得留下。

这条线真正逼出来的，是第十三卷里关于翻译损耗和统一消息世界的判断。  
系统愿意付出一点翻译成本，但不会把消息世界拆成好几套口音各说各话。

这页实际回收的实现链是：
- `08-06-02` 把 `tool progress` 压成 `system message`，说明它宁可接受翻译损耗，也要保住统一消息世界，这会在 `13-06-02` 里回收到“长期产品一直在和现实与历史谈判”。

远端系统不是追求零损耗。  
它更像是在翻译以后，仍然把统一消息世界留住，让系统和人都还能在同一层里读懂彼此。

## 普通做法
更普通的做法，是让远端层直接把原始参数、原始消息或原始权限请求一路透传，剩下的界面、主循环和审批链各自再想办法接。

## 为什么不用
Claude Code 不这么做，是因为一旦 bundle 边界、消息边界和本地审批链被远端层直接穿透，统一体验会先破，后面再补只会越来越乱。

## 代价
代价是翻译层、注入点和兼容逻辑会明显增多，代码阅读链也更长。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
