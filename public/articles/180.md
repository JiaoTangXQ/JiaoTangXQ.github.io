# 第十三卷回收索引：治理世界里的自动模式先剥离危险放行说明优雅也得先面对真实风险

## 实现链
治理世界里真正麻烦的一半，不是自动模式能不能开，而是开之前要先把哪些危险放掉。

这条线真正逼出来的，是第十三卷里关于真实风险必须先被正视的判断。

这页实际回收的实现链是：
- `06-06-01` 说明自动模式前要先剥离危险放行，这会在 `13-01-02` 被回收到“真正的优雅要面对真实风险”。

自动模式不是把风险先按下去再说。  
它先承认哪些放行会带来真实危险，再决定哪些东西必须先剥离。

## 普通做法
更普通的做法，是每个能力各带一点本地校验，或者干脆给用户一个粗糙的总开关，把细节风险留给人自己判断。

## 为什么不用
Claude Code 不这么做，是因为强能力一旦缺少统一制度快照和变更流程，就会在最危险的地方出现口径不一致，最后连改规则本身都没有共同程序。

## 代价
代价是 permission context、settings 扇出、hooks 和模式契约都更重，读起来也更像制度系统。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
