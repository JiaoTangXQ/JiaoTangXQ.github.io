# Claude Code 命令系统地图

命令系统回答的是“用户怎样进入某种工作方式”，而不是“模型怎样直接动手”。

## 实现链

`commands.ts` 把内置命令、条件命令、技能命令、插件命令一起装进命令空间。命令对象本身属于 `Command` 协议，有的只是生成 prompt，有的是本地逻辑，有的还会拉起本地 JSX 交互。也就是说，命令层负责的是入口分流，不是工具执行本身。

## 普通做法

普通 CLI 工具里的命令通常是子命令路由，进入后直接执行具体逻辑。

## 为什么不用

Claude Code 没把命令写成纯 CLI 子命令，是因为它有两种世界要同时服务：用户输入入口和模型内部工具世界。命令要能切换工作模式、注入提示、打开本地流程、接入技能，而不只是调用一个 handler。

## 代价

代价是“命令”和“工具”的区别对新手不再天然直观，必须刻意学习：命令更像入口，工具更像手。
