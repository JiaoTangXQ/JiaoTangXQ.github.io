# 颜色会随会话保留又在清理时归零，说明身份也是生命周期

颜色不是随机动画，它跟着 teammate 身份活，也跟着团队清理一起死。

## 实现链

`teammateLayoutManager.ts` 用 `Map` 保存颜色分配，同一个 teammateId 再次请求时会拿回原色；`clearTeammateColors()` 则在团队清理时统一清空。也就是说，颜色的生死跟着团队生命周期走。

## 普通做法

普通做法可能每次渲染随机给颜色，或者用一次性数组分配，重开就变。

## 为什么不用

Claude Code 不想让颜色漂移，因为颜色本身承担身份记忆。颜色今天是 researcher、明天变 reviewer，会让人脑瞬间掉线。

## 代价

代价是还要专门管理颜色状态，并在清理时记得重置，否则会产生旧身份残留。
