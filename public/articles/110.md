# 面试题：Claude Code 把浏览器能力接成 MCP 服务而不是专用协议，有什么实际好处？

## 问题情境

Claude Code 支持 "Claude in Chrome"——让 AI 操作用户真实浏览器里的标签页、截图、填表单。

这类功能通常的实现思路是：写一个浏览器扩展，扩展暴露 WebSocket 接口，Claude Code 直连过来，用一套专用命令协议通信。

Claude Code 没有走这条路。它把 Chrome 扩展实现成一个 MCP 服务——每个浏览器操作都是一个 MCP Tool，权限请求走 `request_access` 协议，发现机制走 MCP 的工具搜索路径。

**面试题：MCP 化浏览器能力的具体好处是什么？说"统一架构"不算，说出三条有实际代码支撑的理由。**

## 理由一：权限管理不需要另起一套

如果浏览器能力用专用协议，权限管理要单独实现：
- 用户首次使用时，要告知风险
- 每次使用时，要检查是否已授权
- 撤销权限时，要清除相关状态

这些逻辑在 Claude Code 的权限系统里已经有了：`ToolPermissionContext` 随每次工具调用携带当前的权限快照，`request_access` 是整场会话级许可的标准入口，hooks 系统可以拦截任何工具的调用。

把浏览器工具接成 MCP Tool，意味着这一套权限基础设施可以直接复用。`tabs_context_mcp`、`screenshot_mcp` 等浏览器工具和 `BashTool`、`EditTool` 走完全一样的权限路径——用户看到的权限询问界面是一样的，审批逻辑是一样的，审计日志的格式是一样的。

如果用专用协议，这些要全部重新造。造了之后，两套权限系统要一起维护，出了安全问题要分别修复。

## 理由二：工具发现不需要专门处理

MCP 协议里有工具发现机制：客户端通过 `list_tools` 请求获取服务提供的工具列表。Claude Code 的提示词生成、`ToolSearch` 功能和命令补全都依赖这个机制。

浏览器扩展提供的 `tabs_context`、`navigate`、`click`、`screenshot` 等工具，接成 MCP 服务后，自动出现在工具列表里。AI 生成提示词时，这些工具和其他工具一起被考虑；用户用 `@` 提及工具时，浏览器工具也能被搜索到。

用专用协议的话，发现机制要单独实现：浏览器工具不在 MCP 工具列表里，提示词生成不知道有这些工具，用户也找不到它们，除非单独做一个浏览器工具面板。

## 理由三：重连和会话恢复有现成的逻辑

MCP 服务的连接断开和重连是 Claude Code 已经处理过的问题。`RemoteSessionManager` 和 `SessionsWebSocket` 里有断线检测、重连策略和会话恢复逻辑。

Chrome 扩展的连接偶尔会断（浏览器刷新、扩展重启、网络问题）。如果是专用协议，断线重连要自己实现，包括：何时重试、重试几次、恢复后如何同步状态、失败了怎么告知用户。

接成 MCP 服务后，这些都走已有路径。当然 MCP 重连逻辑里可能有些不完全适合浏览器场景的地方，但出发点是复用，而不是重造。

## 命名的统一性

还有一个不那么技术但很重要的好处：命名。

用专用协议时，浏览器工具可能叫 `chrome.tab.screenshot`、`browser.navigate` 或者各种临时命名。

接成 MCP 服务后，工具命名遵循 MCP 约定：`tabs_context_mcp`、`screenshot_mcp`。这个 `_mcp` 后缀不是多余的——它告诉所有层（后端、提示词、前端渲染）这是一个 MCP 工具，走 MCP 路径处理。

这种命名统一性在大型系统里非常重要：代码审查时能一眼看出工具类型，调试时能快速找到对应的处理器，文档里能统一描述"所有 MCP 工具的行为"。

## 代价是存在的

MCP 化不是没有代价：

1. MCP 协议增加了一层序列化/反序列化，延迟比直连略高。对"点击按钮"这种操作来说几乎不感知，但如果需要高频截图，可能有影响。
2. MCP 工具的 schema 定义比直连协议更啰嗦，扩展端的实现代码量更大。
3. MCP 协议有自己的版本演进，浏览器扩展和主程序需要保持协议版本兼容。

但这些代价在"重新造一套权限系统"和"重新造工具发现机制"面前很小。

## 面试要点

1. **权限系统复用**：MCP 工具自动进入已有的权限管理流程，不需要单独的授权 UI 和审批逻辑。
2. **工具发现机制复用**：浏览器工具自动出现在工具列表，提示词生成和工具搜索无需特殊处理。
3. **重连逻辑复用**：断线重连走已有 MCP 重连路径，不需要为浏览器连接单独实现恢复策略。
4. **命名统一性**：`_mcp` 后缀是跨层的类型标记，代码可读性和可维护性都有提升。
