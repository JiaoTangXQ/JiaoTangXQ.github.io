# 只认启动时 TMUX 环境，说明团队空间宁可保守也不误判

Claude Code 对团队模式采用快照式判断，而不是一边运行一边漂。

## 实现链

`teammateModeSnapshot.ts` 会在启动时捕获 `teammateMode`，后续 `getTeammateModeFromSnapshot()` 只认这份快照。`registry.ts` 再根据这份快照和当前环境解析出最终模式。意思很清楚：团队空间的判定以启动时为准，中途不随意漂移。

## 普通做法

普通做法更像实时自适应，环境变量一变、配置一改，下一次操作就按新条件走。

## 为什么不用

Claude Code 不想让团队空间在会话中途变卦，因为那会带来更大的解释成本。用户更需要稳定，而不是每次都追求最新探测结果。

## 代价

代价是中途改配置不一定立刻生效，看起来不够灵活。但这是刻意换来的稳态。
