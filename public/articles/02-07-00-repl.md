# REPL 交棒导读：首屏变轻是为了把重活放到后台

这一章真正要补的，不是再夸一次 REPL 入口很轻，而是先提醒读者：Claude Code 的成熟，在于它知道首屏只是交棒点，真正的重活可以也应该后移。

所以这一章的三片叶子，要连起来读成“前台变轻，后台变重”。先看 [02-07-01-为什么launchRepl更像交棒而不是高潮.md](/Users/xxx/tmp/cc/docs/claude-code-book/02-程序怎样启动成一个agent/02-07-REPL薄而后台预热重/02-07-01-为什么launchRepl更像交棒而不是高潮.md)，再顺着首屏前/后双节奏和 deferred prefetch 往下读，这一章才会从“REPL 启动细节”变成“系统怎样把等待重新分配”的入口。

## 实现链
这一章主要看 `replLauncher.tsx`、`main.tsx` 和相关交互辅助。代码把 REPL 首屏刻意做薄，让用户先看到可交互现场，再把更多预热、提示和补充工作放到首屏后后台继续做。

## 普通做法
很多程序会坚持把所有准备工作做完，再让用户看到界面，这样逻辑最完整。

## 为什么不用那种做法
Claude Code 不这么等，是因为终端里的第一感觉很重要，用户宁可先拿到可工作的界面，再让后台慢慢把重活做完。

## 代价与回报
回报是首屏更快。代价是系统节奏被切成前台轻、后台重两段。
