# 只有启用且通过装配的插件才有资格进场

## 实现链
- `pluginLoader.ts` 会先做 manifest 校验、来源合法性检查、重复名检测、启用状态判断，再把通过的插件交给 `loadPluginCommands.ts` 去解析命令和 skills。
- 也就是说，插件不是“被发现就算存在”，而是“通过一整套装配和治理流程后才算正式进场”。

## 普通做法
- 普通系统更常见的做法是只要目录存在就加载，最多在运行时报错。
- 用户视角看这种方式更直接，因为“我放进去了，为什么不马上生效”这个问题最少。

## 为什么不用
- Claude Code 不愿意把坏插件、禁用插件或来源不合规的插件半加载进来，因为半加载比完全不加载更危险。它会让帮助、hooks、命令和权限层看到一套不完整事实。
- 从代码结构看，它更重视“先确认能进入制度，再允许你出现在系统里”。

## 代价
- 代价是插件作者在接入早期会觉得门槛更高，一点 manifest 或来源问题就可能整包进不来。
- 但这换来的运行时一致性是值得的，因为系统不会被一半成功、一半失败的插件状态拖坏。
