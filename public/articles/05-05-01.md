# 进度为什么必须住在任务对象里

Claude Code 把进度放进任务对象，而不是放进某个临时 UI 状态，是因为进度在这里是任务治理的一部分。

## 实现链

`LocalAgentTask.tsx` 先定义 `AgentProgress`、`ProgressTracker`，再用 `updateProgressFromMessage()` 从 assistant 消息里累计工具次数、token 数和最近活动。真正落地时，`updateAgentProgress()` 会把这些内容写回任务的 `progress` 字段，而不是只喂给界面组件。

这样做的结果是：无论面板、通知、SDK 还是后续摘要服务，只要拿到任务对象，就能看到统一进度。

## 普通做法

普通做法会把进度当临时显示数据。界面需要就单独算，界面销毁就丢掉；最多再把最终摘要单独存下来。

## 为什么不用

Claude Code 不愿意让进度只活在 UI 层，因为子 agent 的进度不是“动画效果”，而是管理现场。后台面板要看，任务通知要看，摘要更新也要看。如果每个消费者自己算一遍，很快就会出现口径不一致。

把进度住进任务对象，本质上是在说：进度是任务状态，不是组件私房钱。

## 代价

代价是任务对象会变胖，更新频率也会上升。每次消息到来都可能引发任务状态更新，对性能和状态管理都是压力。
