# 第十三卷回收索引：连续性世界里系统必须分层留存，历史才留得有层次

## 实现链
连续性不是把历史压成一团以后继续往前跑。  
它要先让历史留得有层次，后面的人才能接得上。

这条线真正逼出来的，是第十三卷里关于系统必须分层留存的判断。

这页实际回收的实现链是：
- `10-01-02` 先承认系统必须分层留存，这会在 `13-07-02` 里继续成立，说明长期可用先靠历史留得有层次。

连续性系统不是把历史做平，而是把历史分层留住。  
这样后面的人接手时，才知道哪些是旧层，哪些是当前层，哪些是还能继续往下走的层。

## 普通做法
更普通的做法，是一发现上下文变长就立刻整理，或者把当前状态和长期记忆统统塞回主会话，省掉额外的插槽和外置存储。

## 为什么不用
Claude Code 不这么做，是因为整理太早会打断主回合，记忆不分层会把短期噪音和长期知识混在一起，最后让连续性系统同时失去可读性和可恢复性。

## 代价
代价是需要额外维护 compact boundary、回合后插槽、外置 memory 和记忆回灌链，系统会更厚。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
