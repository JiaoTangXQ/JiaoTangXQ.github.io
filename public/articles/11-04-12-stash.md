# stash 提示只占一行且自动回填，说明中断保护应该轻而不丢

输入区里有暂存内容时，系统只用一条很轻的 dim 提示告诉你“已经 stash，提交后会自动恢复”。它没有开大弹窗，也没有逼你额外确认。这个决定很有分寸感，因为中断保护的目标是防丢，不是再制造一层流程负担。

真正好的保护应该像安全带，而不是新的驾驶步骤。Claude Code 在这里做得很克制：提醒足够明确，但只占一行；恢复足够可靠，但不要求你额外记账。轻，是为了不抢主路；不丢，才是底线。

## 实现链

`PromptInput.tsx` 用 `key: 'stash-hint'` 和 `FOOTER_TEMPORARY_STATUS_TIMEOUT` 发一条很轻的 stash 提示；真正的 stash 则会把文本、光标位置和粘贴内容一起保存，恢复时再完整回填。中断保护要轻，但不能丢现场。

## 普通做法

更普通的做法，要么不提供 stash，只能自己复制；要么弹一个很重的确认框，把中断保护做成主流程。

## 为什么不用

Claude Code 不想让 stash 抢走操纵面，因为它解决的是“临时中断一下也别丢现场”，不是一项需要用户停下来认真阅读的大动作。提示轻，数据却要真保住。

## 代价

代价是输入区要维护额外的 stash 状态和恢复逻辑；但换来的是中断以后能更体面地回到原位置。
