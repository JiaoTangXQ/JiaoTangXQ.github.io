# Claude Code 工具系统地图

Claude Code 的工具系统不是“给模型一张函数表”，而是一套受协议、权限和 UI 共同约束的行动接口。

## 实现链

`Tool.ts` 先定义工具协议，包括输入 schema、描述、上下文、权限上下文和各种渲染钩子。`tools.ts` 负责把当前会话真正能用的工具池装出来。真正执行时，还要经过 `services/tools/toolExecution.ts` 一类执行层，那里会处理权限检查、进度、结果写回、兄弟工具协调和错误路径。

## 普通做法

普通做法里，一个工具往往就是名字、schema 和 handler 三件套，再加一点简单权限判断。

## 为什么不用

Claude Code 不满足于这种极简工具协议，因为工具一旦进入工作台，就不只是调用 API。它还必须被权限系统理解、被 UI 渲染、被主循环中断、被任务系统追踪，甚至被 team lead 代为审批。

所以这里的工具不是函数，而是正式运行时接口。

## 代价

代价是工具定义明显更重，开发者新增工具时要理解的不只是 handler，还要理解工具如何进入整套制度。
