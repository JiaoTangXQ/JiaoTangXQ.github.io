# 等待审批时先停loading说明制度可以暂时压过流动感

## 实现链
- `useRemoteSession.ts` 在权限请求进入队列时会调整 `setIsLoading(false)`，等用户给出 allow/deny 结果后再恢复相应状态。
- 这意味着一旦远端工作被制度暂停，本地 UI 也要老老实实承认“现在不是在继续跑”，而不是继续假装流动。

## 普通做法
- 普通做法是 spinner 继续转，审批弹窗只作为一个叠层，底下仍保持“工作进行中”的感觉。
- 从体验上看会更连续。

## 为什么不用
- Claude Code 没把流动感放在第一位，因为等待审批本质上就是停住。继续 loading 只会让用户误以为后台还在前进，模糊了制度边界。
- 从代码结构看，它把审批视为真实暂停点，而不是 UI 小插曲。

## 代价
- 代价是体验会显得更“卡顿”，不像某些产品那样无缝。
- 但这类卡顿是诚实的，它在告诉用户：现在真正决定事情走向的是你的裁决。
