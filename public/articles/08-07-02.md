# 本地不认识远端工具，也要先给它一个可裁决外壳

## 实现链
- `createToolStub()` 给未知远端工具补出 `userFacingName()`、`renderToolUseMessage()`、`needsPermissions()` 等最小行为，让审批 UI 至少能拿到名字和输入摘要。
- 这层外壳很薄，但已经足够把“陌生远端动作”拉进本地裁决空间。

## 普通做法
- 更普通的选择是要求本地和远端必须共享一份工具注册表，否则审批系统就拒绝处理。
- 这样规则更硬，也更容易验证。

## 为什么不用
- Claude Code 没把两边强绑死，因为远端常常会有本地未加载的 MCP 工具。要求完全同构会严重限制远端扩展能力。
- 它选择先做一个可裁决外壳，保证制度不断裂，而不是要求世界先完全一致。

## 代价
- 代价是本地 UI 只能展示有限信息，很多专用描述和 rich render 都没有。
- 不过审批最关键的“名字、输入、允许还是拒绝”仍然保住了。
