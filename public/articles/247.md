# 第十三卷回收索引：前四卷里的同一命令在不同入口不是同一能力

## 实现链
`processSlashCommand()` 之外，这条线先走 `REMOTE_SAFE_COMMANDS`，bridge 再走 `BRIDGE_SAFE_COMMANDS` 和 `isBridgeSafeCommand()`；`prompt`、`local`、`local-jsx` 还会按 `effectiveSkipSlash`、`bridgeOrigin` 和各自的放行表被拆开处理。

## 普通做法
更普通的做法，是把同名 `/command` 当成同一能力，不管入口来自本地 REPL、远端现场还是 bridge，都只看名字。

## 为什么不用
Claude Code 不这么做，因为远端现场、本地 UI、文件系统和手机桥接的约束并不相同，入口一换，能力边界就必须重新判定。

## 代价
代价是用户得理解入口差异，不能只记一个命令名；但这样能力与宿主现实至少是对齐的。
