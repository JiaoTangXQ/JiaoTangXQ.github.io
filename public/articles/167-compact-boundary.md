# 第十三卷回收索引：连续性世界里的历史接缝与长期工作台

## 实现链
连续性世界里先要想清楚的，不是整理之后还能不能继续，  
而是历史被整理过以后，凭什么还能继续接上。

这条线真正逼出来的，是第十三卷里关于历史接缝和长期工作台的判断。

这页实际回收的实现链是：
- [209-第十三卷回收索引-连续性世界-系统必须分层留存历史才留得有层次.md](/Users/xxx/tmp/cc/docs/claude-code-book/appendices/209-第十三卷回收索引-连续性世界-系统必须分层留存历史才留得有层次.md)
- [210-第十三卷回收索引-连续性世界-compact-boundary先承认历史已经被整理过.md](/Users/xxx/tmp/cc/docs/claude-code-book/appendices/210-第十三卷回收索引-连续性世界-compact-boundary先承认历史已经被整理过.md)

这一页只负责把这条线收成父级入口。  
真正的展开分别放在 `209` 和 `210`，一个讲历史怎样被分层留住，一个讲 `compact boundary` 怎样先承认历史已经被整理过。

## 普通做法
更普通的做法，是一发现上下文变长就立刻整理，或者把当前状态和长期记忆统统塞回主会话，省掉额外的插槽和外置存储。

## 为什么不用
Claude Code 不这么做，是因为整理太早会打断主回合，记忆不分层会把短期噪音和长期知识混在一起，最后让连续性系统同时失去可读性和可恢复性。

## 代价
代价是需要额外维护 compact boundary、回合后插槽、外置 memory 和记忆回灌链，系统会更厚。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
