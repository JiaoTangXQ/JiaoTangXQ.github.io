# 第十三卷回收索引：前四卷里的主流程解释不了它为何不失控，所以先会收口

## 实现链
前四卷里有一条线，专门回答：主流程如果解释不了它为何不失控，那系统就得先学会收口。

这条线真正逼出来的，是第十三卷里关于制度先行的前半段判断。

这页实际回收的实现链是：
- `01-04-01` 把“主流程解释不了它为何不失控”讲清，这会在 `13-04-01` 里回收到“制度和主流程一样都是主结构”，说明真正的优雅从来不是只会跑，而是先会收口。

真正稳的系统，不是先冲出去再回头补洞。  
它先把该收的门收住，主流程才不会一路滑成失控。

## 普通做法
更普通的做法，是把启动、输入、主循环里的这些小判断留在顺手逻辑里，不单独立规矩。

## 为什么不用
Claude Code 不这么做，是因为工作台一旦要长期运行，这些看似前期的小处理其实都是后面秩序能否站稳的根，不能等出事了再补。

## 代价
代价是前期代码和解释都会显得更重，也更早暴露出长期工程的野心。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
