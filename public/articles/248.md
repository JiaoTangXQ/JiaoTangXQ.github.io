# 第十三卷回收索引：前四卷里的副作用型工具不能被当纯函数

## 实现链
`Tool` 定义里不是只有输入和输出，还带着 `isConcurrencySafe()`、`interruptBehavior()` 和 `contextModifier`；这说明工具结果可能改文件、改状态、改权限，甚至改下一轮能看到什么。

## 普通做法
更普通的做法，是把工具当成纯 RPC：输入进去，字符串结果出来，主循环只管拼接返回值。

## 为什么不用
Claude Code 不这么做，因为这里的工具早就不是纯查询了，而是会编辑文件、开子 agent、调整权限、接真实设备的动作。

## 代价
代价是工具协议更重，连“工具做完了”都可能意味着上下文要重写，测试也不再只是看一个返回字符串。
