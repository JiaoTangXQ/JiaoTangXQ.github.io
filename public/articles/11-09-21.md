# 零命中时滚回锚点：试探失败不丢来处

增量搜索的过程是一种试探：用户输入一个字，系统看看能不能找到；再输入一个字，范围收窄；再输入，可能变成零命中。

零命中时系统该怎么响应？

## 三种可能的处理

**停在当前位置**：视口不动，只是没有高亮显示。用户可能不确定现在在哪里——是搜索把自己带到这里的，还是本来就在这里？

**跳到列表头部**：有些搜索实现在没有命中时会回到开头。这可预测，但很突兀，打断了用户的阅读流。

**滚回锚点**：回到用户按下 `/` 时所在的位置。这是 Claude Code 的选择。

## 实现细节

当 `setSearchQuery` 重新计算后 `matches.length === 0`：

```typescript
scrollRef.current?.scrollTo(searchAnchor.current);
setPositions?.(null);  // 清掉黄色 current 高亮
// query 不清空
```

视口滚回锚点位置，current 高亮消失（因为没有命中），但搜索词保留在输入框里——用户可以继续修改，不需要重新按 `/` 开始。

## 锚点是「出发地」的记忆

`searchAnchor` 在用户按下 `/` 时记录。无论后续搜索把视口带到哪里，只要命中消失了，就回到这里。

这个设计的底层逻辑是：**搜索是可撤销的试探**。用户开始搜索，是因为想在当前上下文附近找点东西。如果搜不到，那就回到出发点，重新调整搜索词。

视口回到锚点，用户面对的是他们熟悉的上下文，可以立刻继续工作——要么换词继续搜，要么放弃搜索，一切都从熟悉的位置开始，而不是从一个被搜索带飞到的陌生位置。
