# swarm 连运行后端和空间组织都要一起决定，说明它是团队运行时

只要一个系统同时决定“在哪里跑”和“怎么摆出来”，它就已经不只是任务调度器了，更像完整运行时。

## 实现链

`registry.ts` 决定使用 in-process 还是 pane backend，`teammateLayoutManager.ts` 决定 pane 创建和颜色分配，`PaneBackendExecutor` 负责把具体 teammate 启进对应空间。swarm 在这里既管执行后端，也管团队空间。

## 普通做法

普通做法会把后端选择交给底层，把界面布局交给上层，自己只负责调度。

## 为什么不用

Claude Code 不这么切，是因为团队的执行方式和团队的展示方式密切相关。lead 在左、队友在右，或者同进程但仍走 mailbox，这些都不是单独一层能说清的。

## 代价

代价是 swarm 的边界很宽，读起来像在同时管很多事。但这正是“团队运行时”的含义。
