# bridge 只注册两个 setter，说明团队中心对外只开窄口

`leaderPermissionBridge` 很克制，只开放了两个 setter，这不是简陋，而是故意收窄接口。

## 实现链

桥里没有暴露整棵组件树，没有暴露整个 AppState，只开放“往确认队列里塞东西”和“更新权限上下文”这两个必要能力。执行层因此只能做最少的事：请求展示、同步规则，而不能随意操纵 UI。

## 普通做法

普通做法一旦开桥，容易一路开大，最后把一整个 UI store 都暴露给运行时。

## 为什么不用

Claude Code 不愿意这么开口，是因为团队中心一旦暴露太宽，后续任何 worker 逻辑都可能顺手依赖它，桥就会迅速变成后门。窄口虽然不舒服，但能防止制度中心被用烂。

## 代价

代价是某些需求不能一步到位，执行层想要更多 UI 能力时必须重新设计桥，而不是顺手拿现成对象。
