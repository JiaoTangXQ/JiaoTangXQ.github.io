# 用户主要面对 lead，说明团队不是平权群聊

Claude Code 的团队不是微信群。用户默认面对的是 lead，不是每个 worker 都能轮流跳出来。

## 实现链

`TEAMMATE_SYSTEM_PROMPT_ADDENDUM` 明说 “The user interacts primarily with the team lead”。`inProcessRunner.ts` 和 `leaderPermissionBridge.ts` 又把权限确认和工具确认统一挂到 leader 的 UI 队列上。这说明 lead 不只是“资深一点的 worker”，而是整个团队对用户的正式接口。

## 普通做法

更直觉的做法，是让每个 worker 都可以直接回用户，谁有结果谁说，形成一种平权群聊式体验。

## 为什么不用

Claude Code 不接受这种平权群聊，因为那会迅速把用户淹没。用户不是来管理五个 agent 的，他们来的是一个工作台。lead 的职责，就是把团队内部的并发，翻译成外部可理解的单一接口。

## 代价

代价是 lead 会成为制度中心，很多流程都要绕它走，速度看起来没那么“去中心化”。但这恰恰是为了减少混乱。
