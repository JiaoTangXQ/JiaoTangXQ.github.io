# 第十三卷回收索引：前四卷里的真实边界会改写能力和副作用抽象

## 实现链
真实边界在前四卷里更后面逼出来的，不只是“我是谁”，  
而是“能力本身和副作用抽象到底还能不能装作不受边界影响”。

这条线真正逼出来的，是第十三卷里关于能力和副作用抽象被边界改写的判断。

它不再负责把两条回收线塞在一起讲完。  
它只负责保住一个入口判断：真实边界会改写能力，也会改写副作用抽象。

## 普通做法
更普通的做法，是把启动、输入、主循环里的这些小判断留在顺手逻辑里，不单独立规矩。 对父级入口页来说，更常见的写法就是继续把两三条不同链挤在一页里，读起来省页数，但判断会糊。

## 为什么不用
Claude Code 不这么做，是因为工作台一旦要长期运行，这些看似前期的小处理其实都是后面秩序能否站稳的根，不能等出事了再补。 这些父级页现在存在，就是为了先把分支关系讲清，再让更细页分别承担细节。

## 代价
代价是前期代码和解释都会显得更重，也更早暴露出长期工程的野心。 作为父级入口，它还额外带来导航变深、读者要多跳几页的成本。
