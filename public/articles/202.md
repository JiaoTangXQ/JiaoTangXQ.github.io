# 第十三卷回收索引：外延执行里连远了以后系统还像一个整体

## 实现链
外延执行里另一半更容易被忽略的，不是入口是否足够多，  
而是系统拉远以后还像不像一个整体。

这条线真正逼出来的，是第十三卷里关于整体体验也是语言秩序证据的判断。  
如果用户连远了以后感到系统散开了，说明层级语言已经开始失真。

这页实际回收的实现链是：
- `12-08-02` 把“连远了以后系统还像一个整体”讲成外延执行卷的结论，这会在 `13-02-01` 里继续回收到“整体体验是层级语言是否仍然统一的证据”。

成熟的外延执行，不会只在局部入口上像一个系统。  
它连远了以后，整体体验也还是同一层级语言的延伸。

## 普通做法
更普通的做法，是把桌面和浏览器能力做成一堆散工具，或者另起一套专用 runtime，让它们按自己的语言各跑各的。

## 为什么不用
Claude Code 不这么做，是因为命名、接口和权限一旦分裂，外部能力就会把原工作台撕成第二套系统，越往后越难收编回来。

## 代价
代价是 wrapper、manifest、bridge、统一命名和适配层都会增加，实现明显更笨重。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
