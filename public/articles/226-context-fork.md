# 第十三卷回收索引：前四卷里的 `context: fork` 是命令世界通往任务世界的桥

## 实现链
前四卷里还有一条线，专门回答“命令世界怎样把工作交给任务世界”。

这条线真正逼出来的，是第十三卷里关于层间交棒的判断。

这页实际回收的实现链是：
- `03-07-01` 把 `context: fork` 写成命令世界通往任务世界的桥，这会在 `13-02-02` 里回收到层间交棒。

桥不是多一层装饰。  
它是把命令世界的入口，转成任务世界可继续执行的入口。

## 普通做法
更普通的做法，是把后台动作当成松散的异步句柄，跑起来就算开始，结束了再临时补状态和通知。

## 为什么不用
Claude Code 不这么做，是因为没有统一任务对象，就没有稳定的生命周期、输出、通知和回看链，前后台切换也会变成一串彼此不认账的临时处理。

## 代价
代价是任务状态字段、输出文件、通知协议和前后台转场逻辑都会变厚，维护者要记住更多接缝。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
