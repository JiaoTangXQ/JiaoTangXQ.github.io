# 第十三卷回收索引：扩展世界里 `MCP server` 进来的不只是工具

## 实现链
扩展世界里真正被接进来的，从来不只是一个工具对象。  
`MCP server` 一旦进来，带来的也是一整套会改写系统边界的东西。

这条线真正逼出来的，是第十三卷里关于“接进来的不只是工具”的判断。

这页实际回收的实现链是：
- `07-08-01` 把 `MCP server` 讲成“进来的不只是工具”，这会在 `13-04-02` 里回收到“新世界进来时也先过原制度，而不是偷偷平行生长”。

`MCP server` 不是一件插上就完的附件。  
它代表的是新世界进门时，系统必须先认出这不是单个工具，而是一整套要被制度接住的能力。

## 普通做法
更普通的做法，是每种来源保留自己的装配方式和运行时语言，等真的冲突了再临时做桥接。

## 为什么不用
Claude Code 不这么做，是因为来源越多，越要先学旧骨架的语言；否则系统会被多套协议、多套入口和多套制度动作拖散。

## 代价
代价是注册、翻译和回收线都会变多，很多地方看起来像是在重复做统一。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
