# 第十三卷回收索引：连续性世界里的整理时机与回合后插槽

## 实现链

这页具体回收的判断是“连续性世界—整理时机与回合后插槽”。
这页真正回收的是 `query.ts` 里的 microcompact/autocompact 接缝、`services/compact/*`、SessionMemory 和 memory extraction 这条连续性链。Claude Code 不把“连续性”理解成一直堆消息，而是把压缩、边界消息、session memory、自动抽取记忆和回合后处理时机都写成不同层级的制度。

## 普通做法

更直觉的做法，要么是尽量保留所有上下文，等撑不住了再粗暴摘要；要么定时做一次统一压缩，把连续性问题当成单一的 token 管理问题。

## 为什么不用

Claude Code 没这么做，是因为它同时要顾 token 预算、现场可读性、长期记忆和风险边界。microcompact 针对的是当前窗口里的碎接缝，autocompact 处理的是整段会话的预算压力，SessionMemory 和自动记忆又是另一层长期连续性，几件事混成一层就会互相打架。

## 代价

这样分层之后，连续性处理更细，也更不容易一刀切地伤到现场。代价是这块代码很难“一眼懂”，因为整理时机、整理粒度和长期记忆边界都被拆散在不同模块里。
