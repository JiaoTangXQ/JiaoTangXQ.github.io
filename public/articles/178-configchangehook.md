# 第十三卷回收索引：治理世界里的 ConfigChange 先过 hook

## 实现链
治理世界更高阶的一层，不是只约束怎么用系统，  
而是连改系统规则这件事本身也要先纳入制度。

这条线真正逼出来的，是第十三卷里关于强而不乱的更深一层判断。

这页实际回收的实现链是：
- `06-09-01` 让 `ConfigChange` 先过 hook，这会在 `13-04-01` 里继续支撑“连改规则也要先过程序”。

一个成熟系统不会把改制度当成旁路动作。  
它会先让 `ConfigChange` 经过 hook，再决定这次变化是否真的进入当前会话。

## 普通做法
更普通的做法，是每个能力各带一点本地校验，或者干脆给用户一个粗糙的总开关，把细节风险留给人自己判断。

## 为什么不用
Claude Code 不这么做，是因为强能力一旦缺少统一制度快照和变更流程，就会在最危险的地方出现口径不一致，最后连改规则本身都没有共同程序。

## 代价
代价是 permission context、settings 扇出、hooks 和模式契约都更重，读起来也更像制度系统。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
