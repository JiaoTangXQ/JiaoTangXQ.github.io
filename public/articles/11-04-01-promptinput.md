# PromptInput 接住的不是文本，而是整片控制面

如果只看名字，会以为 `PromptInput` 只是输入框。但真实情况正相反。它周围挂着模式切换、补全、历史、建议、图片粘贴、权限模式、模型与思考强度、后台任务、团队视图，甚至 bridge、全局搜索和临时对话框。

这说明 Claude Code 把“准备说什么”和“准备怎样工作”故意放在同一处。用户每次敲字，其实都站在整台工作台的主操纵台上。优点是所有决定都在一个手边，缺点是这里天然容易变重，所以它才要靠大量小 hook 和状态切片把复杂度拆散。

## 实现链

`PromptInput.tsx` 不只管输入字符串，它同时编排历史搜索、高亮、队友视图、提示词建议、粘贴处理、stash、底部 footer、排队通知和多个对话框。代码里它更像驾驶舱总装层，接住的是一整片控制面，不是一只文本框。

## 普通做法

更普通的输入组件会只负责值和光标，别的提示、通知和模式切换都散到周围小组件里。

## 为什么不用

Claude Code 不把这些控制拆散，是因为这里的输入区就是主操纵台。模式、队友关系、通知和粘贴策略都会直接改写“现在能怎么输入、该提示什么”。

## 代价

代价是 `PromptInput.tsx` 体量明显更大，组件边界不再像普通文本框那样轻；但它也因此真的能承担控制面职责。
