# 第十三卷回收索引：界面世界里的注意力压缩与接缝锚点

## 实现链

这页具体回收的判断是“界面世界—注意力压缩与接缝锚点”。
这页真正回收的是 `AppStateStore`、`PromptInput`、`StatusLine`、`TaskListV2`、`Messages` 这些状态与显示簇如何把运行时事实翻译到屏幕上。Claude Code 没把终端界面当成纯回放器，而是让界面承认任务状态、权限制度、注意力资格和接缝锚点已经进入显示层，所以“显示什么”其实是运行时判断的一部分。

## 普通做法

更直觉的终端 UI 写法，是后端产出什么，界面就直接 print 什么，最多做一点颜色和布局整理。那样界面很薄，状态复杂度主要留在后端。

## 为什么不用

Claude Code 没这么写，因为它要面对长 transcript、小屏幕、后台任务、权限提示和多来源消息竞争。要是界面只是薄包装，系统就没地方处理“什么该现在上屏、什么该压缩、什么只是接缝提示、什么需要保住用户当前视线”。它把这一层抬高，是为了让界面真正承担工作台职责。

## 代价

这样做之后，UI 不再只是展示层，而是带着更多派生状态、订阅纪律和布局规则。好处是长会话更能用；代价是界面层比普通 TUI 厚得多，也更难被第一次接触的人快速看懂。
