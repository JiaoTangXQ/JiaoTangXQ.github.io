# 后端一旦探明就固定到进程生命周期，说明团队运行时追求稳态

Claude Code 宁愿牺牲一点动态灵活，也要让团队运行时在一个会话里保持稳态。

## 实现链

`registry.ts` 会缓存 backend detection、in-process backend 实例和 pane executor 实例。注释写得很清楚：一旦探明，就固定到进程生命周期。

## 普通做法

普通做法会觉得环境变化时应该随时重探测，永远使用当前最优后端。

## 为什么不用

Claude Code 更在意团队世界的稳定口音。后端一旦漂移，权限、空间、消息和用户心智模型都会跟着漂。稳态比瞬时最优更重要。

## 代价

代价是会话中途无法轻易切换到更新或更强的后端，需要显式重启或重建。
