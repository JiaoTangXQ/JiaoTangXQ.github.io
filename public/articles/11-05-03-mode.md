# 远端时宁可不显示本地 mode，也不把错现实端给用户

在远端模式下，底部不会继续显示本地这边的 permission mode。原因很朴素：真正跑在远端容器里的 agent，不一定和本地这层外壳处在同一种制度状态里。界面如果硬把本地 mode 亮出来，只会把错的现实端给用户。

成熟的摘要层宁可少说，也不乱说。它首先忠于真实世界，而不是追求每一栏都看起来很完整。

## 实现链

`PromptInputFooterLeftSide.tsx` 构造 `modePart` 时专门加了 `!getIsRemoteMode()` 条件。只要进入 remote 之类“真正 agent 活在别处”的场景，本地 permission mode 就不再上屏，避免把错现实端给用户。

## 普通做法

更普通的做法，是本地知道当前 mode，就继续显示给用户，默认“有总比没有强”。

## 为什么不用

Claude Code 不接受这种半真半假的摘要，因为远端场景里本地 mode 不一定代表真正执行现场。少一条信息还能忍，错一条信息会直接误导操作判断。

## 代价

代价是远端模式下用户少了一个熟悉提示，需要从别的指示器理解当前环境；但至少底部没有继续说错话。
