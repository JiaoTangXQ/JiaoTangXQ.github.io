# 第十三卷回收索引：前四卷里的附件注入把文件记忆和技能发现重新带回下一轮

## 实现链
前四卷里还有一条线，讲的不是大架构，而是怎么把文件、记忆和技能发现重新带回下一轮。  
这类动作不显眼，但它决定系统是不是还能继续往下走。

这条线真正逼出来的，是第十三卷里关于“小连接件少失控一次”的判断。

这页实际回收的实现链是：
- `04-08-03` 把附件注入写成把文件、记忆和技能发现重新带回下一轮，这会在 `13-03-02` 里回收到“小连接件少失控一次”的缝合能力。

真正成熟的连续性设计，不是把桌面清空。  
它是把文件、记忆和技能发现重新带回下一轮，让下一次继续时还能认得出自己。

## 普通做法
更普通的做法，是把启动、输入、主循环里的这些小判断留在顺手逻辑里，不单独立规矩。

## 为什么不用
Claude Code 不这么做，是因为工作台一旦要长期运行，这些看似前期的小处理其实都是后面秩序能否站稳的根，不能等出事了再补。

## 代价
代价是前期代码和解释都会显得更重，也更早暴露出长期工程的野心。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
