# 第十三卷回收索引：界面世界细化

## 实现链
这页对应的实现链，主要落在 `AppStateStore`、`Messages`、`buildMessageLookups()`、`VirtualMessageList`、`PromptInput`、`StatusLine` 等显示与事实翻译层上。它真正回收的是：终端界面为什么不是薄皮，而是运行时对人公开事实的器官。

## 普通做法
更普通的 CLI 界面做法，是把终端当打印机：消息直接输出、输入直接回收、状态靠零散提示拼出来。

## 为什么不用
Claude Code 没这么组织，因为一旦任务、远端、权限和工具都并行存在，界面如果不先承认共同事实，就会越来越像会说话但会撒谎的壳。

## 代价
代价是状态层和编排层都会变厚，很多“小 UI 功能”都会长出缓存、查找表和布局纪律。
