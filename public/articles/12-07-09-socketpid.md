# socket 名字绑用户名而单进程再带 pid，说明共享桥也先分清地址层次

浏览器桥的 socket 命名不是随手拍的。先有按用户名划出的那一层公共名字，再在 Unix 下给单个进程挂上 `pid.sock`。这说明 Claude Code 很清楚，同一台机器上既有“这一位用户的桥”，也有“这一个进程的桥”，地址层次必须先分清。

这种层次感很重要，因为一旦桥开始共享给真实浏览器和真实用户，多进程、多窗口、多次重连都不再是意外，而是常态。

## 实现链
`getSocketDir()` 先把目录绑到用户名，`getSecureSocketPath()` 再在目录下生成 `pid.sock`。先分用户，再分进程，避免所有 session 抢同一个地址。

## 普通做法
普通做法是整个机器共用一个固定 socket 路径。

## 为什么不用
Claude Code 不用单一路径，因为用户可能并行开多个 session，甚至多个 Claude 进程。地址层次不分清，桥接就会串话。

## 代价
代价是连接管理要学会扫描目录、清理旧 socket 和兼容旧路径。
