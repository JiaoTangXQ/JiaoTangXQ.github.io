# 非虚拟路径也设消息上限，说明滚动存活比一次全看更重要

## 实现链
`Messages.tsx` 在非虚拟路径里有 `MAX_MESSAGES_WITHOUT_VIRTUALIZATION = 200` 的安全上限，`computeSliceStart()` 还会用 `sliceAnchorRef` 记住当前窗口从哪条消息开始。这样做不是为了省一点字节，而是让超长历史在主线程里还能活着。

## 普通做法
普通列表要么全量渲染，要么简单按尾部 `slice(-N)` 截断，不会额外考虑重排、折叠后还能不能稳定滚动。

## 为什么不用
Claude Code 的消息区既要展示历史，又要在长会话里保持交互可用。全量挂出几千条消息，用户拿到的只会是卡顿和乱跳；它宁愿保住当前这屏的可用性，也不去追求一次看完所有旧内容。

## 代价
代价是旧消息不会一直留在挂载树里，阅读和排查历史时要接受“只保留一段活窗口”。但换来的结果是会话还能继续滚，不会被历史拖死。
