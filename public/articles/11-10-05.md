# companionReservedColumns：精灵先报预算，再进输入区

终端 UI 的列宽是一种稀缺资源。输入框、状态指示器、模式提示……每个元素都在争用同一行的水平空间。

`buddy` 精灵出现在输入区右侧，它也需要空间。如果不提前声明自己的宽度需求，就会直接挤压输入框，让输入区变窄，甚至让内容溢出边界。

## companionReservedColumns 做什么

```typescript
function companionReservedColumns(
  terminalColumns: number, 
  speaking: boolean
): number
```

这个函数决定「给 buddy 预留多少列」，考虑因素包括：

- Feature flag 是否启用
- 当前是否有 companion（`getCompanion()` 是否返回值）
- 是否处于静音状态（`companionMuted`）
- 终端宽度（窄屏时精灵不显示完整形态，需要的列数不同）
- 是否正在「说话」（气泡出现时需要更多列）

如果 companion 不存在，或者被静音，或者 feature flag 关闭，返回 0——不占任何列。

## 布局协议

`PromptInput.tsx` 在计算输入框可用宽度时：

```typescript
const availableWidth = columns - companionReservedColumns(columns, speaking);
```

精灵在输入区里占有合法的位置，通过正式的「预算申报」而不是「强行插入」。

这个设计模式很重要：任何需要占用共享布局空间的组件，应该通过明确的协议告知宿主「我需要多少空间」，而不是自己渲染出来，让宿主事后发现布局坏了。

体验层功能也不例外。精灵可以出现，但它的出现需要先在布局系统里「报名」，而不是悄悄把输入框挤开。
