# 精灵会预留列宽，说明陪伴感不能偷吃主输入空间

## 实现链

`CompanionSprite.tsx` 里的 `companionReservedColumns(terminalColumns, speaking)` 会先看 feature flag、`getCompanion()`、`companionMuted` 和终端宽度，再决定要不要为精灵和气泡预留列宽。`PromptInput.tsx` 再把这部分宽度从输入区里减掉。

## 普通做法

普通做法会先把装饰层画上去，再看主输入被挤成什么样。那样虽然省脑子，但最后通常是主工作区替体验层买单。

## 为什么不用

Claude Code 先算预算再露面，是因为 buddy 再可爱也不能挤占主输入的可读性。它可以出现，但必须被纳入布局契约。

## 代价

代价是布局代码要显式知道 buddy 的存在，还得处理有无气泡、是否说话、是否全屏这些条件。换来的则是输入区不会因为陪伴感变形。
