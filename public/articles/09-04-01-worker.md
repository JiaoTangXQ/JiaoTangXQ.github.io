# 每个 worker 各批各的，用户会先崩溃

多 agent 团队里最不能分散的，往往不是计算，而是授权。

## 实现链

`inProcessRunner.ts` 给 worker 请求权限时，优先把请求送进 leader 的 `ToolUseConfirm` 队列；桥不可用时，才通过 `permissionSync.ts` 发 mailbox 请求。也就是说，最终批不批，默认都回到 leader 这一个界面中心。

## 普通做法

普通做法会让每个 worker 自己弹权限框。谁要读文件、谁要执行命令，就自己向用户申请。

## 为什么不用

Claude Code 不这么做，因为这会把用户炸掉。用户面对的不是一个任务，而是一群不同 worker 的并发审批。用户不仅会被打断，还会很快失去“我到底在批准谁做什么”的心智模型。

lead 收口权限，不是抢权，而是在替用户收走混乱。

## 代价

代价是 leader 会成为制度瓶颈，所有 ask 权限都可能排队。但这比让用户在多窗口里来回点确认更可控。
