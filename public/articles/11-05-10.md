# 远端和主动模式关掉终端进度条，说明摘要层不该越权汇报别处进度

终端进度条不是任何时候都能开。到了 remote 模式，或者进入 proactive 这类另有节奏的状态，它会选择闭嘴。因为这时本地终端并不真正拥有那段进度的解释权，硬报反而容易把人带偏。

这说明摘要层很克制。它只汇报自己确实有资格代表的现实，不把别处的节奏冒充成眼前的真相。

## 实现链

这一页对应的约束体现在底部摘要的显示闸门上：`statusLineShouldDisplay(settings)` 会在某些模式直接关掉整条状态栏，而 `PromptInputFooter.tsx` 也会优先丢掉可选 status line，避免本地终端替远端或别的主动执行现场越权代言进度。摘要层只在自己确实代表当前现场时才发声。

## 普通做法

更普通的实现是无论 agent 在哪，都让本地 UI 持续显示统一进度，默认“有反馈总比没反馈强”。

## 为什么不用

Claude Code 不接受这种统一汇报，是因为进度一旦脱离真实执行现场，就很容易变成误导。远端或主动模式下，本地终端并不总有资格解释那边到底进行到哪一步。

## 代价

代价是某些模式下本地反馈会显得更克制，用户需要借助别的线索理解系统状态；但至少底部没有假装自己知道别处的真实进度。
