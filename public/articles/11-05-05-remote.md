# remote 链接出现在底部区，说明远端不是背景条件而是当前世界事实

一旦当前会话真的连到了远端，底部区会直接出现一个 `remote` 链接。它没有被藏进设置页，也没有只留给调试面板，因为这里要表达的不是“有个高级功能开着”，而是“你现在就在另一个会话世界里工作”。

这正是好的摘要层该做的事。远端不是幕后条件，而是当前世界事实；既然事实已经改写了，就该被人第一眼看见。

## 实现链

`PromptInputFooterLeftSide.tsx` 会把 `remoteSessionUrl` 直接渲染成 `remote` 链接塞进底部 parts。远端连接不是藏在设置页的背景条件，而是当前世界事实，所以它被放到最容易被看见的位置。

## 普通做法

更普通的产品会把远端地址藏进详情页，底部最多显示一个模糊的“已连接”。

## 为什么不用

Claude Code 不愿意把 remote 降成背景信息，是因为远端会改写权限、执行位置和恢复语义。用户需要随时知道自己是不是在另一个会话世界里。

## 代价

代价是底部一行更紧，布局必须考虑本地与远端两种现实；但远端事实因此不会被 UI 偷偷藏起来。
