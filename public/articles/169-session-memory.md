# 第十三卷回收索引：连续性世界里 session memory 先外置当前工作状态别把一切塞回主会话

## 实现链
连续性世界里最容易弄混的，不是有没有记忆，  
而是当前工作状态到底该不该全回灌进主会话。

这条线真正逼出来的，是第十三卷里关于外置工作笔记的判断。

这页实际回收的实现链是：
- `10-06-01` 把 `session memory` 讲成外置工作笔记，这会继续支撑 `13-07-02`，说明当前工作状态不必全塞回主会话，而是可以单独托管。

好的连续性系统，不会把所有状态都往主会话里硬塞。  
它先把当前工作状态外置出去，主会话才不会被短期噪音挤满。

## 普通做法
更普通的做法，是一发现上下文变长就立刻整理，或者把当前状态和长期记忆统统塞回主会话，省掉额外的插槽和外置存储。

## 为什么不用
Claude Code 不这么做，是因为整理太早会打断主回合，记忆不分层会把短期噪音和长期知识混在一起，最后让连续性系统同时失去可读性和可恢复性。

## 代价
代价是需要额外维护 compact boundary、回合后插槽、外置 memory 和记忆回灌链，系统会更厚。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
