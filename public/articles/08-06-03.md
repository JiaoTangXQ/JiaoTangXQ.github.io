# 远端工具就算本地不认识，也要先变成可审批对象

## 实现链
- `remotePermissionBridge.ts` 的 `createToolStub()` 会为本地不认识的远端工具造一个最小 `Tool` 外壳，保证它仍能进入本地的权限判断和确认队列。
- 重点不是把远端工具完整本地化，而是先把它翻译成“本地制度能裁决的对象”。

## 普通做法
- 普通做法可能是对未知远端工具直接放行，或者直接拒绝，因为本地既然不认识它，就没法按本地规则处理。
- 这两种都很省事。

## 为什么不用
- Claude Code 没选极端放行或极端拒绝，因为远端工具是否被本地加载，不等于它应不应该经过本地审批。
- 从代码看，它更在意把审批制度维持住，哪怕展示层只能退化成一个简陋 stub。

## 代价
- 代价是未知工具的本地展示和解释能力比较弱，用户看到的只是最低限度外壳。
- 但这总比绕开审批更稳，因为制度边界至少没丢。
