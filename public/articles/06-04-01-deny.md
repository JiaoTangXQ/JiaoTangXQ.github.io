# deny规则为什么要在装配工具池时就先裁掉

## 实现链

这一页重点看“deny规则为什么要在装配工具池时就先裁掉”。对应源码主要是 `src/tools.ts`、`src/utils/permissions/permissions.ts`。
这一章真正落在 `tools.ts` 的工具池筛选和 `permissions.ts` 的运行时裁决双层结构上。Claude Code 先用 deny 规则把一部分工具直接从可见工具池里裁掉，再把真正的动作审批留到运行时结合现场继续判。

## 普通做法

更简单的做法，要么只在装配层裁工具，看不见就算安全；要么只在运行时审批，工具全展示，真正调用时再说。

## 为什么不用

Claude Code 不只选其中一层，是因为两层解决的问题不同。装配层负责不让明显不该出现的能力混进当前世界，运行时层负责处理“当前现场到底该不该做”这种上下文问题。只保留一层，都会把另一层的问题硬塞给错误的模块。

## 代价

双层治理更稳，也更贴近真实权限边界。代价是代码会看起来像“重复判断”，新手第一次看时容易误以为系统做了两遍同样的事。
