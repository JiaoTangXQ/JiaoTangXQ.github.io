# 第十三卷回收索引：任务世界里进度先住进任务对象，劳动进行中也持续被认领

## 实现链
任务世界把后台能力翻译成劳动以后，还要继续回答一个问题：  
这份劳动在进行中，怎么一直有人认领。

这条线真正逼出来的，是第十三卷里关于长期工作台观的进一步判断。  
系统不是只在完成时认领任务，而是在进行中也持续认领它。

这页实际回收的实现链是：
- `05-05-01` 把进度正式写进任务对象，这会在 `13-07-02` 里回收到“长期工作台要接住工作正在推进时的生命体征”，说明劳动还在走的时候，系统也没有把它从认领范围里放出去。

很多系统只在完成那一刻才算认领任务。  
Claude Code 更克制，它让进度先住进任务对象，这样劳动进行中也能一直被系统看见。

## 普通做法
更普通的做法，是把后台动作当成松散的异步句柄，跑起来就算开始，结束了再临时补状态和通知。

## 为什么不用
Claude Code 不这么做，是因为没有统一任务对象，就没有稳定的生命周期、输出、通知和回看链，前后台切换也会变成一串彼此不认账的临时处理。

## 代价
代价是任务状态字段、输出文件、通知协议和前后台转场逻辑都会变厚，维护者要记住更多接缝。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
