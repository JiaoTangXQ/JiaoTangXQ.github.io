# copy 提示按环境说不同的话，说明终端里的「复制」不是一件固定的事

在 web 浏览器里，Ctrl+C 就是复制，这件事很简单。在终端里，Ctrl+C 通常是中断信号。要复制文本，得用鼠标选中后右键，或者在 xterm.js 里用特定手势，或者配置 `copyOnSelect`。

不同终端环境，「如何复制」是不同的动作。Claude Code 的底部提示认真对待了这个差异。

## 三种情况，三种提示

`PromptInputFooterLeftSide.tsx` 里处理 copy 提示时有几个判断条件：

- `isXtermJs()`：是否在 xterm.js 嵌入环境里运行
- `copyOnSelect`：是否启用了选中即复制
- `useHasSelection()`：当前是否有选中内容

这三个条件组合后，底部提示会显示：
- 在有选区且启用了 copyOnSelect 时：文字已在剪贴板
- 在 xterm.js 环境且有选区时：提示用 native select 路径
- 在普通终端且有选区时：提示按 Ctrl+C

没有选区时：提示不出现。

## 提示的克制

这条提示不是弹窗，不是常驻状态栏，只在有选区时才出现，显示完提示目的就完成了。没有比这更必要的东西——用户选中了，系统就说一句「你现在可以这样复制」，然后不再多说。

## 为什么要区分环境

因为如果不区分，提示就只能是「可能可以用 Ctrl+C，也可能用其他方式，具体看你的环境」，这对用户毫无帮助。或者干脆永远只显示 Ctrl+C，在 xterm.js 里的用户就被误导了——他们按 Ctrl+C 只会发出中断信号，而不是复制。

终端环境的异质性是真实存在的，诚实的提示应该如实反映它，而不是假装所有情况都一样。
