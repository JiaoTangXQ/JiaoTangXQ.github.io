# Claude Code 查询主循环地图

这页讲的是 Claude Code 怎样把“一次提问”变成“一轮可持续推进的工作回合”。

## 实现链

`QueryEngine.ts` 负责长期会话状态，保存消息、权限拒绝、总 usage、文件缓存和 session 连续性。真正跑单轮的是 `query.ts`，它会处理用户输入、系统提示、压缩、模型调用、工具调度、错误恢复和停止条件。也就是说，`QueryEngine` 管“这段会话怎么活”，`query()` 管“这一回合怎么跑完”。

## 普通做法

普通做法常见一个大函数：组 prompt、调模型、执行工具、渲染结果，全塞一起。

## 为什么不用

Claude Code 不这么做，是因为长期会话和当前回合不是一回事。消息持久化、session 恢复、memory 注入、预算累计这些是会话级问题；工具执行、stop hooks、compact、重试是回合级问题。把两者揉一起，迟早会把恢复链和执行链搅烂。

## 代价

代价是你要理解两层状态：一个是跨回合活着的 QueryEngine，一个是每回合单独运行的 query 主循环。结构更复杂，但职责更清晰。
