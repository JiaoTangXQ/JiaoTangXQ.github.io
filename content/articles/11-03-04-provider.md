# 禁止嵌套 Provider 说明唯一现实不能套娃

`AppStateProvider` 明确禁止嵌套，这不是洁癖，而是在保护“唯一现实”。一旦一个终端里同时出现两层状态世界，组件很快就会搞不清自己究竟该信哪一层。

Claude Code 宁可直接报错，也不接受多套现实并存。对这种工作台来说，现实来源必须唯一，不能套娃。

## 实现链

`AppStateProvider` 一进来就读 `HasAppStateContext`，发现外面已经有一层就直接抛错。代码明确禁止嵌套 provider，因为这份共同现实被要求只有一个源头，不能同一棵树里再套出第二个世界。

## 普通做法

普通 context 容器通常默认允许嵌套，哪里想局部覆盖就再包一层，React 自己也不会阻止。

## 为什么不用

Claude Code 不允许 AppState 套娃，是因为任务、权限、通知和远端状态都被要求是全局共同事实。要是这里还能局部再造一个世界，很多组件会在不知情时读到两套真相。

## 代价

代价是灵活性变差，测试或局部演示时不能随手再包一层 provider；但这份约束正是为了守住“只有一个现实源头”。
