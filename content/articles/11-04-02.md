# 这么多对话框都围着输入区转，说明它是主操纵台

`PromptInput` 附近长出这么多弹层和辅助面板，并不是界面失控，而是在承认一个事实：终端里最频繁的动作不是看，而是下一步怎么下命令。于是历史搜索、任务面板、团队切换、快捷打开、自动模式确认，都围着输入区转。

这种设计很像飞机驾驶舱。真正重要的开关不会散在整架飞机各处，而会尽量收在飞行员手边。Claude Code 也是这样做的。它牺牲了一点局部组件的清爽，换来整台工作台操作路径的稳定。

## 实现链

`PromptInput.tsx` 里挂着模型选择、自动模式 opt-in、stash 提示、图片粘贴反馈等多种弹层和旁白，而且全都围着底部输入区组织。对话框不是随机长在页面别处，而是围着主操纵台转。

## 普通做法

普通 UI 往往会把这些弹层分散到页面各角落，谁触发谁自己弹，输入框本身只负责收文本。

## 为什么不用

Claude Code 不这么散，是因为用户下一步动作大多都要回到输入区继续发生。把弹层围住主操纵台，才能让模式切换、失败反馈和继续输入保持在同一条动作线上。

## 代价

代价是输入组件不得不理解更多外围行为，底部区域的布局和弹层逃逸都更复杂；但交互节奏更连贯。
