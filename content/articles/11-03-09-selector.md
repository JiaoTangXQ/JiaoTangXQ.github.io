# 派生 selector 留在纯函数里说明复杂路由也先做干净翻译

连派生判断也被收进 `selectors.ts` 这种纯函数文件里。像“输入此刻该发给 leader、viewed teammate 还是 named agent”这种复杂路由，不直接塞进组件里现场判断，而是先翻译成干净的派生结果。

这样界面负责显示，规则负责判断。复杂度没有消失，但边界被整理干净了。

## 实现链

`selectors.ts` 顶部直接写着 selector 要保持 pure and simple，只做数据提取，不做副作用。派生读取必须先在纯函数里把现实翻译干净，再交给组件使用，哪怕逻辑复杂也先守这个纪律。

## 普通做法

更常见的做法，是在组件里边读 state 边顺手算路由、顺手打日志、顺手塞一点副作用。

## 为什么不用

Claude Code 不让 selector 变脏，是因为订阅层一旦掺进副作用，重渲染次数和读取时机都会开始影响业务行为。它要的是“订阅只是读现实，不顺手改现实”。

## 代价

代价是派生逻辑不能偷懒塞进组件任意角落，得额外维护纯函数层；但这样至少能保住订阅语义干净。
