# Claude Code 源码导图：任务、外延、前段主干的代码落点

## 前段主干：src/entrypoints/ 和 src/bootstrap/

**核心问题**：Claude Code 启动时发生了什么？主循环是怎么运转的？会话恢复走哪条路？

`src/entrypoints/` 包含不同模式下的入口：CLI 交互模式、SDK 调用模式、远端模式。每种入口的启动逻辑不同，但最终都汇聚到同一套主循环。

`src/bootstrap/` 处理启动时的初始化：读取配置、设置权限上下文、初始化工具集、加载历史会话。`src/bootstrap/state.ts` 里有 `getSdkAgentProgressSummariesEnabled` 这样的开关——这是特性标志，控制某些实验性功能是否激活。

**值得重点阅读的文件**：`src/main.tsx`。这里有 teleport（跳到远端模式）和 remote（处理远端请求）的分流逻辑，是整个系统"入口分发"的总交换机。

前段主干的核心设计判断是：**启动时先把环境认清，再确定走哪条路**。不同的运行环境（本地 CLI、SDK 调用、远端容器）有不同的初始化路径，但它们都在启动时就确定，而不是在运行中动态切换。

## 任务世界：src/tasks/ 下的七种实现

每种 TaskType 都有对应的实现目录：

```
src/tasks/
  LocalShellTask/        # local_bash
  LocalAgentTask/        # local_agent
  RemoteAgentTask/       # remote_agent
  InProcessTeammateTask/ # in_process_teammate
  LocalWorkflowTask/     # local_workflow（推测）
  MonitorMcpTask/        # monitor_mcp（推测）
  DreamTask/             # dream
```

这些实现的组织方式值得注意：每个目录对应一种任务类型，但它们都通过 `src/tasks/types.ts` 里的 union 类型统一对外暴露。

**LocalAgentTask 是最复杂的实现**（也是最值得阅读的）。`LocalAgentTask.tsx` 里的 `ProgressTracker`、`AgentProgress`、`updateProgressFromMessage` 展示了任务如何把 AI 的 token 消耗和工具调用活动翻译成人可理解的进度信息。

核心设计判断是：**任务是可观察的劳动对象，不是匿名的异步过程**。`pendingMessages`、`backgroundSignal`、`retain`、`evictAfter` 等字段是这个判断的具体落地。

## 外延执行：src/tools/ 下的外延工具

外延执行指的是 Claude Code 把能力延伸到终端之外——浏览器、桌面、设备。

对应的源码分散在：
- `src/tools/computerUse/`（计算机操作工具）
- Chrome 扩展相关的 MCP 工具定义（可能在 `src/tools/` 的子目录）
- `src/native-ts/`（Native Host 相关代码）

外延执行的共同逻辑是：**把强能力翻译成 MCP 语言，让权限系统、工具发现、命名约定都能直接复用**。

`src/tools/AgentTool/` 里有 `loadAgentsDir.ts`——这是"从目录加载 agent 定义"的实现，展示了 Claude Code 如何让新的 agent 类型被自动发现，而不需要硬编码每种 agent 的配置。

## 三条线的汇聚点

任务世界和外延执行在 `src/utils/task/` 汇聚：

```
src/utils/task/
  framework.ts     # registerTask, updateTaskState, PANEL_GRACE_MS
  diskOutput.ts    # getTaskOutputPath, initTaskOutputAsSymlink, evictTaskOutput
  sdkProgress.ts   # emitTaskProgress
```

`framework.ts` 里的 `registerTask` 和 `updateTaskState` 是所有任务类型用来操作 AppState 的标准函数——无论是 LocalShellTask 还是 RemoteAgentTask，都通过同一套函数注册和更新自己的状态。

这就是前段主干、任务世界和外延执行能"说同一种语言"的基础：所有的任务状态变化都经过 `framework.ts` 里定义的函数。

## 面试要点

1. **前段主干的核心是入口分发**：`main.tsx` 是总交换机，不同运行环境的初始化路径在启动时就确定。
2. **任务世界的核心是 LocalAgentTask**：这里有最复杂的任务状态管理逻辑，包括进度追踪、token 计数、活动记录。
3. **外延执行的核心是 MCP 翻译**：强能力通过被翻译成 MCP 工具融入工作台，不另起运行时。
4. **三条线在 utils/task/ 汇聚**：registerTask 和 updateTaskState 是所有任务类型的公共操作接口，这是"说同一种语言"的工程基础。
