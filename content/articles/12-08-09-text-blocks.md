# 只收 text blocks 回来，说明桥上推理也要翻回宿主可执行语言

浏览器侧推理跑完以后，Claude Code 并没有把模型返回的所有内容原样搬回来，而是只抽取真正承载命令输出的 `text blocks`。这一步很关键，因为桥上的推理世界再复杂，回到宿主时仍然得翻成宿主真正能继续执行的语言。

这就是外延执行最成熟的一点：连借来的推理都不会在桥上越长越野，最后还是要回到原系统能接住的那种口音里。

## 实现链
`callAnthropicMessages` 在拿到 `sideQuery()` 返回后，会显式过滤 `response.content`，只保留 `b.type === 'text'` 的 block。也就是说，桥上推理最后必须翻回宿主能执行的文本命令，而不是把 thinking/tool_use 等原始块一路带下去。

## 普通做法
普通做法是把模型原样输出整包转发给下游，让下游自己判断哪些块有用。

## 为什么不用
Claude Code 不这么转发，因为浏览器 MCP 这一层真正需要的是可执行命令文本，不是把模型内部块类型继续扩散给更多层。

## 代价
代价是会丢掉部分模型内部结构信息，调试时看不到完整原始 block。
