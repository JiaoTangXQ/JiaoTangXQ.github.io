# 第十三卷回收索引：前四卷里的 local 命令直接调用本地模块直接写回输出

## 实现链
`getMessagesForSlashCommand()` 处理 `local` 时会懒加载本地模块，直接调用实现，再把结果作为本地输出写回当前会话；结果可以是普通文本、`compact`，也可以是 `skip`。

## 普通做法
更普通的做法，是把所有命令都转成 prompt，让模型决定后续动作。

## 为什么不用
Claude Code 不这么做，因为很多本地操作本来就应该由程序自己直接做完，再把结果写回去。

## 代价
代价是命令系统必须同时维护“模型型命令”和“程序型命令”两套心智模型。
