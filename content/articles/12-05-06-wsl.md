# WSL 和订阅门槛直接先摊牌，说明产品边界先说清再给入口

Claude Code 在这个菜单里没有把限制藏到失败以后才说。`WSL` 不支持就直接亮红字，没 `claude.ai` 订阅也直接亮红字。也就是说，它先把产品边界摊在台面上，再决定要不要继续给你操作入口。

这看上去不浪漫，却非常体面。因为真实产品不是只负责展示能力，也负责尽早告诉人“这里为什么还进不去”。边界说得越早，用户越不会把时间浪费在错误道路上。

## 实现链
`main.tsx` 在启用 Claude in Chrome 前先检查 `isClaudeAISubscriber()`；`useChromeExtensionNotification()` 也会直接给出“需要 claude.ai subscription”或“扩展未检测到”的启动提示。代码不是等用户操作失败后再解释，而是先把资格边界摊牌。

## 普通做法
更讨好的做法是先把入口都露出来，等用户点了再报“当前环境不支持”或“需要订阅”。

## 为什么不用
Claude Code 不这么做，因为浏览器接入链本来就重，再让用户走一半才知道没资格，只会放大挫败感。

## 代价
代价是入口判断会带上更多环境和商业分支，看起来没有“人人都能点”那么统一。
