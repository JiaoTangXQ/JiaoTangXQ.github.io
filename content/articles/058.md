# 第十三卷回收索引：远端边界世界里的风险与层级边界

## 实现链
这页对应的实现链，主要落在 `bridge`、`remote`、`assistant`、`teleport`、`RemoteSessionManager`、会话翻译与重连处理上。它真正解释的是：跨出本地机器以后，代码为什么先重算会话边界、权限口径和消息语言。

## 普通做法
更普通的做法，是把远端当成另一台终端，能连上就行，消息、工具和权限各自局部打补丁。

## 为什么不用
Claude Code 没走这条路，因为远端真正难的不是连线，而是让本地与远端继续活在同一个制度世界里。它宁可多写翻译层和重连逻辑，也不愿让远端自成一套。

## 代价
代价是消息转换、权限桥、viewerOnly 阅读姿态和恢复链都更复杂，边界问题也更难一眼看清。
