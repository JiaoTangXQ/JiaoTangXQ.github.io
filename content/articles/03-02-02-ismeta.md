# 图片元信息为什么会被单独写成 isMeta 消息

Claude Code 不只把图片块送进会话，还会专门补一条 `isMeta` 消息，告诉模型这张图片的尺寸、来源路径之类的元信息。它没有把这些说明硬塞进用户原文里，也没有指望模型自己从像素里“猜出一切”，而是把它们当成一类单独的系统事实来处理。

这一步很有分寸。图片本体属于这一轮要看的内容；图片元信息则更像系统对当前材料的登记说明。把两者分开，用户界面不会被技术说明刷屏，但模型又能准确知道自己眼前除了图像，还有哪些关于这张图的重要事实。尤其是源路径这类信息，一旦进入元消息，后面的工具和会话恢复都会更容易接住它。

这就是 `isMeta` 的价值。它不是偷偷塞私货，而是在替系统保存那些“对模型有用、但不该打扰用户阅读”的事实。Claude Code 在这里非常克制: 它既不把元信息完全藏掉，也不把它和正文搅在一起，而是给它安排了一个独立但正式的位置。

## 实现链
图片尺寸和来源路径不会混进普通用户文本，而是先积累在 `imageMetadataTexts` 里，最后由 `addImageMetadataMessage()` 生成一条 `isMeta: true` 的隐藏用户消息发给模型。

## 普通做法
最常见的做法是把这些元信息直接拼进可见提示词，或者干脆不告诉模型尺寸和来源。

## 为什么不用
前者会污染用户看到的 transcript，后者又会让模型不知道图像边界和来源。`isMeta` 正好把“模型需要知道”与“用户不必看到”拆开。

## 代价
会话里多了一层隐藏消息语义，读日志时不理解 `isMeta` 就容易看懵；但这是保持界面干净和模型信息完整之间的折中。
