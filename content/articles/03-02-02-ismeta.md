# 面试题：isMeta 是什么？为什么不直接把信息塞进用户消息？

## 面试官追问

「你知道 Claude Code 里有一类叫 `isMeta` 的消息吗？它的存在解决了什么问题？」

这是个很有辨别度的问题。只了解 API 调用的候选人会愣住；真正读过源码的人能说出它的设计意图。

## isMeta 是什么

定义在 `createUserMessage` 里，是一个标记字段：

```typescript
createUserMessage({
  content: imageMetadataTexts.map(text => ({ type: 'text', text })),
  isMeta: true,  // 这里
})
```

带着 `isMeta: true` 的用户消息，在 UI 渲染层会被过滤掉，用户看不见它出现在 transcript 里。但它仍然是合法的 `user` role 消息，会被发送给模型。

换句话说：**对用户透明，对模型可见**。

## 哪些信息被放进 isMeta 消息

**1. 图片元信息**

粘贴截图后，图片的尺寸和磁盘路径：
```
[Image: 1920×1080, source: /tmp/screenshot-1234.png]
```

这些信息告诉模型可以引用这个路径，或者知道这是 Retina 分辨率的图。用户不需要在 transcript 里看到这行技术说明。

**2. 系统生成的提示（scheduled tasks, isMeta 队列消息）**

在 `processUserInput` 里有这个参数：

```typescript
isMeta?: boolean
// When true, the resulting UserMessage gets `isMeta: true` (user-hidden,
// model-visible). Propagated from `QueuedCommand.isMeta` for queued
// system-generated prompts.
```

定时任务、后台 fork 子 agent 的结果回填，都用 `isMeta` 静默地进入会话，不打扰用户界面。

**3. local-jsx 命令的 metaMessages**

`LocalJSXCommandOnDone` 回调支持传入 `metaMessages`：

```typescript
const metaMessages = (options?.metaMessages ?? []).map((content: string) =>
  createUserMessage({ content, isMeta: true })
)
```

命令完成后需要给模型额外传递的信息（比如「用户在 config 里改了什么」），不用出现在 transcript，通过 isMeta 消息发给模型。

## 为什么不直接塞进用户消息

有三个替代方案，以及为什么都有问题：

**方案 A：把元信息直接追加到用户原文**

`「帮我看这个错误」` → `「帮我看这个错误 [Image: 1920×1080, source: /tmp/shot.png]」`

问题：用户会在 transcript 里看到这行奇怪的补充文本。每次粘贴图片，transcript 就多一行技术噪声。如果模型还把这个文本引用到回复里（「您提到的 1920×1080 图片……」），阅读体验会很糟糕。

**方案 B：用 system message**

system 消息有不同的语义——它代表配置层面的全局指令，不该被临时元信息污染。而且 system 消息的更新机制和 user/assistant 消息不同，用来传图片路径是滥用。

**方案 C：干脆不告诉模型**

那么模型就不知道图片文件在磁盘哪里，也不知道分辨率，很多后续操作的上下文就丢了。

**isMeta 的解法**：单独建一条用户角色的消息，打上不渲染标记。模型收到完整上下文，用户 transcript 保持干净。两边的需求都满足了，代价是日志里多了一类需要理解的消息类型。

## 边界条件

`isMeta` 消息仍然是 `user` role，所以它计入 token 使用，也参与对话压缩逻辑。在 `/compact` 或 autocompact 时，这些消息可能被裁掉——这是合理的，因为它们是系统生成的技术说明，而不是用户真实意图。

---

*面试指导：被问到「Claude Code 怎么处理不该让用户看见但模型需要知道的信息」时，`isMeta` 是一个精准的答案。重点说清楚它和 system message 的区别：isMeta 是每轮可变的技术注解，system 是全局配置。*
