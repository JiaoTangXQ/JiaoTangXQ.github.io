# setup 现场导读：抽象环境要切成真实工作现场

这一章真正要讲的，不是再多做一轮配置，而是 `init` 里那套抽象环境为什么还得经过 `setup`，才能真正落成用户此刻身处的工作现场。

所以这一章的三片叶子，要连起来读成“抽象世界切进真实现场”。先看 [02-05-01-为什么setup是理论世界切进真实现场的那一刀.md](/Users/xxx/tmp/cc/docs/claude-code-book/02-程序怎样启动成一个agent/02-05-setup怎样把抽象环境落成真实现场/02-05-01-为什么setup是理论世界切进真实现场的那一刀.md)，再顺着 cwd/worktree/session 重排依赖和并行优化边界往下读，这一章才会从“落地步骤”变成“工作台怎样真正长到用户现场”的入口。

## 实现链
这一章主要看 `setup.ts` 和它在 `main.tsx` 里的承接。`init()` 解决的是抽象环境能不能跑，`setup()` 解决的是现在具体在哪个 repo、哪个 worktree、哪个 session、该连哪种现场。

## 普通做法
普通程序往往把 init 和 setup 混成一个大初始化阶段，环境和现场不再区分。

## 为什么不用那种做法
Claude Code 单独留出 setup，是因为能运行和处在正确现场里运行是两件事。

## 代价与回报
回报是后面的会话真正绑定到用户现场。代价是启动阶段再多一层现场装配。
