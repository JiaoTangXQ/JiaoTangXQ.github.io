# assistant会补拉历史并维持阅读姿态，说明它更像远端会话视图

## 实现链
- `useAssistantHistory.ts` 会把拉到的历史页通过 `convertSDKMessage()` 转成本地消息，再以前插方式塞到当前列表，并通过 layout effect 保持视口位置。
- 这套处理明显是为阅读姿态服务的：用户是在一段已有远端会话里继续往上翻，而不是在一个新会话里重新播放。

## 普通做法
- 普通做法要么整页重刷，要么拉到旧消息后直接把滚动条顶到最上。
- 实现更简单，也不需要精细锚定。

## 为什么不用
- Claude Code 不愿让阅读姿态被打断，是因为 assistant 的价值本来就来自“远端会话视图”。如果每次翻页都把用户视线打飞，它就更像调试接口而不是阅读器。
- 从代码看，作者在很认真地保护“你正在读某处上下文”这个事实。

## 代价
- 代价是滚动锚定和前插补偿代码明显更绕，排错也不轻松。
- 但这能换来阅读稳定性，值得。
