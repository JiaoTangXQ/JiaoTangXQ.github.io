# 第十三卷回收索引：治理世界里的模式名先写成人机风险契约别把权限当方便滑杆

## 实现链
治理世界里最早被误读的，不是权限怎么开，而是模式名到底在说什么。

这条线真正逼出来的，是第十三卷里关于风险契约要先于方便性的判断。

这页实际回收的实现链是：
- `06-02-02` 把模式名称写成人机契约，而不是方便程度滑杆，这会在 `13-01-02` 里回收到“优雅也必须正视真实人、真实判断方式和真实责任分配”。

模式名不是给按钮换个更顺眼的说法。  
它先把系统和人的风险关系写清，后面权限怎么开才有资格继续谈。

## 普通做法
更普通的做法，是每个能力各带一点本地校验，或者干脆给用户一个粗糙的总开关，把细节风险留给人自己判断。

## 为什么不用
Claude Code 不这么做，是因为强能力一旦缺少统一制度快照和变更流程，就会在最危险的地方出现口径不一致，最后连改规则本身都没有共同程序。

## 代价
代价是 permission context、settings 扇出、hooks 和模式契约都更重，读起来也更像制度系统。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
