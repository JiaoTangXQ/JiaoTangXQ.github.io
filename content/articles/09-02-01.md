# 共享任务不等于共享沟通

哪怕几个 agent 在同一团队、同一任务列表里工作，它们也不会自动共享心智。

## 实现链

`teammatePromptAddendum.ts` 直接把规矩写进系统提示：如果要和队友沟通，必须使用 `SendMessage`；直接输出文本对别人不可见。`teammateMailbox.ts` 也说明了这一点，真正可被团队读取的是收件箱文件里的消息，不是某个 agent 自己对模型说了什么。

也就是说，团队共享的是制度，不是隐形心电感应。

## 普通做法

普通做法会让多个 agent 共用一段上下文，或者把彼此输出直接暴露给所有成员，默认大家都“看得到”。

## 为什么不用

Claude Code 不这样做，是因为默认共享最容易造成上下文污染。一个 worker 的探索过程不该自动进入所有人脑子里，尤其不该自动进入用户面前。它要求显式通信，是在强迫每条跨成员信息都经过一次“我到底要不要发出去”的判断。

## 代价

代价是沟通成本更高。很多本来可以靠共享内存偷偷完成的事情，现在都要正式发消息。但这正是它换来的边界清晰。
