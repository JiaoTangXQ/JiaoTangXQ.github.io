# Claude Code 扩展系统地图

Claude Code 的扩展世界不是“装几个外挂”，而是把外部能力正式收编进工作台。

## 实现链

扩展主干主要由 `services/mcp/*`、`skills/*`、`plugins/*`、`utils/plugins/*` 组成。命令层会把技能命令和插件命令装进去，工具层会把 MCP 工具暴露给模型，远端和桥接层则把外部会话纳入统一控制。也就是说，外部能力不是挂在外面，而是被翻译进命令、工具和状态系统。

## 普通做法

普通做法会把插件、协议接入、远端控制分散成彼此独立的小系统。

## 为什么不用

Claude Code 不这么分，是因为只要这些能力最终都要被模型使用、被权限系统理解、被 UI 展示，它们就必须被收进同一运行时语法，而不是保持各说各话。

## 代价

代价是扩展系统会显得很大，很多本来可以“各管各的”的外部能力都被迫遵守同一套制度。但这正是统一体验的代价。
