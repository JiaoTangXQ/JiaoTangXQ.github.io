# preAction 导读：真正闸门发生在完整初始化之前

这一章要先钉住一个判断：很多人以为初始化做完才算正式开工，Claude Code 更成熟的地方，是它把真正的启动闸门放在完整初始化之前。

所以这一章的三片叶子，不是在讲一个 CLI 小技巧。先看 [02-03-01-显示帮助不该付完整初始化成本.md](/Users/xxx/tmp/cc/docs/claude-code-book/02-程序怎样启动成一个agent/02-03-preAction是真正闸门/02-03-01-显示帮助不该付完整初始化成本.md)，再顺着 preAction 前后职责和副作用延后往下读，这一章才会从“命令行前置钩子”变成“真正闸门怎样替系统拦下不必要成本”的入口。

## 实现链
这一章围着 `main.tsx` 和 commander 的 `preAction` 闸门展开。代码先用它拦下帮助、版本和某些只需浅处理的路径，再决定哪些副作用要晚到这道闸门之后。

## 普通做法
更普通的 CLI 会把所有初始化都放在最前面，不管用户是要真正运行还是只看帮助，都先把完整环境搭起来。

## 为什么不用那种做法
Claude Code 不肯这样浪费，因为很多路径根本不值得付完整初始化成本。

## 代价与回报
回报是轻路径更轻。代价是入口多出一层真正开始之前的开始。
