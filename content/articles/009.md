# Claude Code 权限系统地图

Claude Code 的权限系统不是几条 if 判断，而是一整套模式、规则和 UI 协议。

## 实现链

`Tool.ts` 里的 `ToolPermissionContext` 先定义权限模式、工作目录、allow/deny/ask 规则和 plan mode 相关字段。工具执行时会经过 `hasPermissionsToUseTool()` 一类检查，再进入 `components/permissions/*` 的确认 UI，比如 `PermissionRequest.tsx`、文件写入确认、bash 确认等。团队场景下，这套 ask 流程还能被 lead UI 或 permissionSync 接管。

## 普通做法

普通做法通常是危险命令前弹个确认框，或者简单列一个 allowlist。

## 为什么不用

Claude Code 不够用这种“临门一脚式权限”，因为它支持本地命令、文件编辑、子 agent、团队协作、远端能力和 plan mode。权限如果不是运行时前提，而只是事后插一层确认框，很快就会处处漏风。

## 代价

代价是权限相关代码会又长又碎，读起来比普通 CLI 重很多。但这套重量正是它能在真实仓库里长期运行的前提。
