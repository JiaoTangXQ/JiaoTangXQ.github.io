# 远控真正难的是把标题权限和状态一起送过去

## 实现链
- `replBridge.ts` 维护的不只是消息流，还有 `onSetPermissionMode`、`onStateChange`、标题更新、session 创建和 archive。远端世界要知道本地 session 叫什么、当前权限模式是什么、桥接状态是已连接还是重连中。
- 所以真正难点不是“把内容送过去”，而是把内容之外的会话元信息也一起带过去，而且不能彼此漂移。

## 普通做法
- 普通远端终端往往只传输出文本，标题、权限和状态都留在本地 UI 自己猜。
- 这样通信协议最瘦，服务端也最省心。

## 为什么不用
- Claude Code 没这么做，是因为 claude.ai 侧看到的是一张真实 session 卡片，不是黑屏终端镜像。标题、权限和状态如果不同步，用户看到的远端会话就只是个壳。
- 从代码看，它更愿意把元信息链做厚，也不接受“只同步文本”的表面完成。

## 代价
- 代价是桥接协议会越来越像会话协议，而不是消息管道。
- 但这正符合产品目标：它要桥的是会话，不是管道。
