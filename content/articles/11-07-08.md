# 工具结果的搜索文本优先让工具自己定义，说明可搜语义属于工具不是消息壳

## 实现链
`Messages.tsx` 的搜索逻辑会优先看工具是否提供 `extractSearchText`，尤其是 `user` 里的 `tool_result`。如果工具没提供，才退回到通用的 `renderableSearchText()`，结果还会放进 `WeakMap` 缓存，避免每次按键都重算。

## 普通做法
普通列表通常直接搜显示出来的文字，或者用一套统一的字段猜“应该搜什么”，不会把语义权交回给具体工具。

## 为什么不用
Claude Code 里的工具结果有自己的结构，很多时候“用户真正关心的重点”不是屏幕上最显眼的字，而是工具作者定义的那一层语义。让工具自己说，搜索才不会只搜到壳子，漏掉真正有用的内容。

## 代价
代价是搜索链条更长，工具和 UI 要多一层约定。好处是搜索结果更贴近真实意图，不会被通用文本 heuristics 带偏。
