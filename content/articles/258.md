# 第十三卷回收索引：前四卷里的统一外观与内部机制分开

## 实现链
`processSlashCommand()` 先把入口统一成 `/command args`，再由 `getMessagesForSlashCommand()` 按 `command.type` 分流：`prompt` 改写会话并推进主循环，`local` 由本地程序直接写回，`local-jsx` 则拉起临时界面后再回到会话。

## 普通做法
更普通的做法，是把所有 slash 命令都强迫走同一种内部执行机制，只保留一种实现。

## 为什么不用
Claude Code 不这么做，因为同样是一个外观，并不代表背后的物理机制真的同质。

## 代价
代价是实现者视角会更不整齐，但这份不整齐正好保住了真实差异。
