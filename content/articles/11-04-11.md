# 超长粘贴先变引用，说明驾驶舱先保可驾驭性再保原样塞入

当用户一次粘进来一大段超长文本，输入区不会傻傻把整块内容原样塞满当前操纵面，而是只保留前后预览，把中间折成一个可追溯的引用。系统在这里承认：原样保留当然重要，但先让人还能继续看、继续改、继续驾驭更重要。

这不是偷懒，而是一种很成熟的界面伦理。驾驶舱不是仓库，不能为了忠实搬运就把自己的操纵面堵死。Claude Code 先保输入面的可操作性，再把完整内容挂在引用后面，顺序是对的。

## 实现链

`inputPaste.ts` 在 `maybeTruncateMessageForInput()` 里，会把超长粘贴折成 `start + [Pasted text #N lines] + end` 这种引用占位，同时把完整内容单独保存在 `placeholderContent`。输入框先保可驾驭性，再保原文不丢。

## 普通做法

普通文本框要么照单全贴进去，把操纵面直接堵满；要么粗暴截断，中间内容真的丢掉。

## 为什么不用

Claude Code 不选这两条路，是因为驾驶舱既不能被一大坨文本淹没，也不能偷偷丢内容。它宁可多做一层占位和回填，也要同时保住可操作性和可追溯性。

## 代价

代价是“你眼前看到的文本”和“系统实际保存的内容”被分成了两层，代码得负责占位、追踪和还原；但这是为了不让输入框变成仓库。
