# 早期分流导读：init 之前先决定自己是谁

这一章真正重要的，不是又读了一份配置，而是系统为什么必须在完整初始化前先认清自己正以什么身份、什么入口、什么连接方式进入这次运行。

所以这一章的三片叶子，要连起来读成“身份先于初始化”。先看 [02-02-01-为什么有些设置必须在init之前先读出来.md](/Users/xxx/tmp/cc/docs/claude-code-book/02-程序怎样启动成一个agent/02-02-在真正初始化前先决定自己是谁/02-02-01-为什么有些设置必须在init之前先读出来.md)，再顺着特殊入口分流和模式目录/连接方式前置往下读，这一章才会从“启动前配置”变成“程序怎样先决定自己是谁”的入口。

## 实现链
这一章的实现链主要在 `main.tsx` 里：某些设置、特殊入口、模式目录和连接方式都要在 `init()` 之前先分流，不然后面初始化出来的就是错世界。后面几页分别拆先读什么、先分什么、为什么不能晚一点再定。

## 普通做法
普通程序常把身份判断放到初始化之后，先把公共环境搭出来，再看是走哪条路径。

## 为什么不用那种做法
Claude Code 不能这么晚决定，因为模式、工作目录、连接方式和某些设置会反过来影响初始化本身。

## 代价与回报
回报是不会先搭错环境再返工。代价是入口阶段的条件分支明显变多。
