# 同一套 agent 组成团队后，新增的是制度层，不是更多线程

Claude Code 的 teammate 本体并没有变成另一种 AI，但一旦它们组成团队，外面就多了一层制度外壳。

## 实现链

同样是 `runAgent()`，普通子 agent 主要受任务系统管理；teammate 则多了 `TeammateIdentity`、mailbox、leader 权限桥、团队模式快照、pane backend 等额外设施。这些都不是“线程更多了”，而是制度更多了。

## 普通做法

普通做法会把团队理解成并发执行器叠加。

## 为什么不用

Claude Code 不把团队简化成更多线程，是因为线程并不解释谁能发话、谁能批权限、谁代表用户。那些真正让团队成立的东西，全是制度字段和通信协议。

## 代价

代价是 swarm 学习成本比普通任务系统高得多，因为你学的不再只是执行，而是组织。
