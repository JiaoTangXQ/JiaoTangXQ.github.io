# computer use 和 Claude in Chrome：两种外延能力，同一套接入姿势

桌面控制（computer use）和浏览器自动化（Claude in Chrome）是两种完全不同的技术能力：

- computer use 控制操作系统，截屏、点击、键盘输入
- Claude in Chrome 控制浏览器，读取页面、执行 JavaScript

但看 `main.tsx`，会发现它们进入 Claude Code 系统的方式几乎一样：都产出动态 MCP 配置，都通过标准的工具发现机制登记，都走同一个权限检查流程。

## 为什么要用同一扇门

Claude Code 的工具系统、权限系统、主循环是一套已经经过验证的结构。外延能力如果不通过这扇门而是「走后门」（为自己开专用通道、绕过权限检查），就会出现：

- 这个工具没有被问权限，那个工具被问了——用户体验不一致
- 主循环不知道某个工具的存在，无法正确处理它的调用
- 遥测和日志里看不到这个工具的行为，排障困难

统一入口意味着所有工具都「说同一种语言」，主循环可以一视同仁地调度它们，权限系统可以统一管理它们，日志可以统一记录它们。

## usePromptsFromClaudeInChrome 做的翻译

`usePromptsFromClaudeInChrome()` 把宿主的工具权限模式（`ask`/`auto`/其他）翻译成浏览器扩展能理解的标志：`ask` 或 `skip_all_permission_checks`。

这是接入协议的一部分：外延能力要接受宿主的权限制度，而不是自己另立一套。宿主说「这次要问用户」，扩展就进入询问模式；宿主说「全部跳过」，扩展才全部跳过。宿主权限模式是单一真相，不是多个并行管理的权限系统。
