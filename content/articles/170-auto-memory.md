# 第十三卷回收索引：连续性世界里 auto-memory 只沉淀以后还值得记得的长期知识

## 实现链
连续性世界里另一半容易弄混的，不是记不记得，  
而是哪些东西值得被记成长期知识。

这条线真正逼出来的，是第十三卷里关于长期沉淀边界的判断。

这页实际回收的实现链是：
- `10-08-01` 把 `auto-memory` 明确写成项目长期知识，而不是当前工作噪音，这会在 `13-07-02` 里重新出现，说明长期记忆只该沉淀以后还值得记得的东西。

好的连续性系统，不会把当前工作噪音也一并沉淀成长期记忆。  
它只保留以后还值得记得的知识，让长期记忆保持干净。

## 普通做法
更普通的做法，是一发现上下文变长就立刻整理，或者把当前状态和长期记忆统统塞回主会话，省掉额外的插槽和外置存储。

## 为什么不用
Claude Code 不这么做，是因为整理太早会打断主回合，记忆不分层会把短期噪音和长期知识混在一起，最后让连续性系统同时失去可读性和可恢复性。

## 代价
代价是需要额外维护 compact boundary、回合后插槽、外置 memory 和记忆回灌链，系统会更厚。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
