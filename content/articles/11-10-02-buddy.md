# buddy不是贴纸，它被认真接进了输入区、附件和提示词

## 实现链

`buddy` 不是单独挂个图标，而是被接进了三条线：`PromptInput.tsx` 负责给 `/buddy` 做彩虹高亮并预留列宽，`useBuddyNotification.tsx` 负责启动 teaser，`buddy/prompt.ts` 负责生成 `companion_intro`，再由 `utils/attachments.ts` 和 `utils/messages.ts` 把这份说明送进消息流。

## 普通做法

普通做法要么只做一块 UI 装饰，要么只在提示词里写一段人格说明。两种都省事，但都会把“陪伴感”做成单点功能，而不是系统里可控的一层。

## 为什么不用

Claude Code 没把 buddy 当贴纸，是因为它同时要服务 UI、提示词和上下文三处：既要被看见，也要告诉主模型角色边界，还不能抢走主输入区的注意力。它是晚到的体验层，不是旁白式注释。

## 代价

代价是这层体验需要跨 UI、配置和消息附件协同，装配比单纯贴图复杂得多。换来的则是结构分工清楚：看得见的是 sprite，进上下文的是附件，守边界的是提示词。
