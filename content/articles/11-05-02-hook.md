# hook 可生成状态栏文本，说明它是可编排摘要层

更见功力的是，`StatusLine` 连自己的文本都允许通过 hook 命令生成。这等于承认“摘要”本身也是一种能力，而不是写死在组件里的样式。

这样做的好处是，团队、企业或插件可以把自己最关心的现实塞进同一条摘要层，而不必重造第二套状态栏。代价是它必须非常克制，否则状态栏会从摘要退化成噪声条。

## 实现链

`StatusLine.tsx` 不是把摘要文本写死在 JSX 里，而是走 `executeStatusLineCommand(buildStatusLineCommandInput(...))` 这条 hook 链生成状态栏文本。摘要本身也被当成可编排能力，而不是硬编码字符串。

## 普通做法

更直觉的做法，是前端自己拼一条固定文案，遇到不同团队需求再加几个 if。

## 为什么不用

Claude Code 不把摘要焊死，是因为不同环境和组织关心的状态不同。既然状态栏承担的是“当前世界摘要”，它就必须允许外部规则参与生成，而不是永远只说内置那一句。

## 代价

代价是状态栏逻辑得处理 trust、命令失败和更新时机，复杂度比普通标签栏高很多；但可编排性也因此真正成立。
