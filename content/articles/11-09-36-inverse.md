# 手动滚动退出时只清当前高亮，inverse 痕迹留着

用户在搜索模式下手动滚动（不用 n/N 跳转，而是直接用鼠标滚轮或键盘翻页），`disarmSearch()` 会被触发。

`disarmSearch` 只做一件事：

```typescript
setPositions(null);  // 清掉黄色 current 高亮
// query 不清
// inverse 高亮不清
```

## 黄色 current 和 inverse 是两种不同的东西

**黄色 current**：标记「当前跳转到了哪个命中」的位置。它是搜索**导航**的状态——你在第几个命中，正在看哪里。

**inverse 样式**：标记「所有命中在哪里」的全局视图。它是搜索**结果**的痕迹——这个词在整个 transcript 里出现过哪些地方。

## 手动滚动的语义

手动滚动表达的是：「我想暂时离开当前高亮位置，去看看别的地方」。

这不等于：「我放弃了这次搜索，不想知道结果在哪里了。」

如果 `disarmSearch` 同时清掉 inverse 高亮，用户离开一下再回来，发现所有搜索痕迹都消失了，需要重新输入关键词才能找回上下文。这让搜索变成了「走一步消一步」的操作，失去了「做一次全局定位」的价值。

保留 inverse 高亮，用户在手动浏览后仍然能看到命中痕迹，知道「哦，那个词在这几个地方出现过，我可以回去继续用 n/N 跳转，或者直接看清楚了再说」。

搜索的两种状态分开清理，是在正确区分「我现在在哪」（可以放弃）和「搜索找到了什么」（不应该因为我移动了就丢失）。
