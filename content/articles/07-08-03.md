# 资源刷新会连命令和技能一起重算，说明它们共用一片现实

## 实现链
- `list_changed` 触发后，Claude Code 不只刷新资源清单，还会连依赖这些资源的命令、prompts 和 skills 一起失效重算。
- 代码在这里表达的是一件很朴素的事：这些对象虽然显示在不同入口里，但它们共享同一片后端现实。

## 普通做法
- 普通做法会给每类对象各自缓存、各自刷新，资源刷新只影响资源面板，命令和 skill 继续用旧快照。
- 这样职责边界看起来更独立。

## 为什么不用
- Claude Code 不敢让它们各刷各的，因为一旦后端世界已经变了，而命令和 skill 还拿着旧引用，用户看到的就是互相打架的事实。
- 它宁可多做一轮重算，也不接受“同一个 server 在不同入口里讲不同版本的真相”。

## 代价
- 代价是刷新影响面变大，一次资源变化就可能牵动多层缓存失效。
- 但这换来了状态一致性，对长期连接系统来说是值得的。
