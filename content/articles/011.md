# Claude Code 术语表

这份术语表不是百科全书，而是把整本书里最容易混淆的词按 Claude Code 的实际代码语义钉住。

## 实现链

下面这些词，最好直接按源码语义理解：

- `Command`：用户进入某种工作方式的入口，见 `commands.ts` 和 `types/command.*`。
- `Tool`：模型直接调用的行动接口，见 `Tool.ts`、`tools.ts`、`services/tools/*`。
- `Task`：被系统正式托管的工作对象，见 `Task.ts`、`framework.ts`、`tasks/*`。
- `Agent` / `subagent` / `teammate`：都是执行主体，但角色不同。子 agent 主要接在 `AgentTool` 和任务系统上；teammate 还要经过团队身份、mailbox 和审批制度。
- `Permission Mode`：系统当前的安全姿态，不只是一个弹窗设置，见 `ToolPermissionContext`。
- `MCP` / `Resource` / `Plugin` / `Skill`：都是外部能力接入词，但进入运行时的方式不同。

## 普通做法

普通术语表喜欢用业内通用定义，比如把 command 说成“CLI 子命令”，把 tool 说成“函数调用接口”，把 agent 说成“任何会调模型的程序”。

## 为什么不用

Claude Code 不能直接照搬这些宽泛定义，因为它的代码已经把这些词做得更窄、更具体了。比如 command 和 tool 在这里绝不是一回事，teammate 也不是普通 subagent 的别名。继续沿用宽泛说法，新手会很快读糊。

## 代价

代价是这份术语表会显得比业内通用定义更“挑剔”。但这是必要的，因为手册服务的是这份代码，不是抽象 AI 行业。
