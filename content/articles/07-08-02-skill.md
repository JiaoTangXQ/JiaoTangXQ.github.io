# skill、资源和资源工具一起出现，说明它在收编整片世界

## 实现链
- MCP skills 的发现要靠 `mcpSkillBuilders.ts` 和 `loadSkillsDir.ts` 提供的构建能力，而资源本身又会通过 `ListMcpResourcesTool`、`ReadMcpResourceTool` 暴露出来。
- 代码把 skill、资源、资源工具放在一起看，意思很清楚：这不是“顺手多挂几个按钮”，而是在把同一个 server 背后的整片现实一起收编。

## 普通做法
- 普通做法可能会把 skill 单独当成 prompt，把资源单独当成浏览列表，互相几乎不通信。
- 那样模块边界看起来更清爽。

## 为什么不用
- Claude Code 没完全切开，是因为很多 skill 之所以成立，本来就依赖那一批资源和资源工具。如果资源变了，相关 skill 的可用性和含义也可能跟着变。
- 从代码结构看，它更愿意承认这种血缘关系，而不是为了“模块漂亮”强行拆散。

## 代价
- 代价是 MCP 接入层的耦合度更高，读源码时会觉得工具、资源、skills 交错在一起。
- 但这比假装它们彼此无关更诚实。
