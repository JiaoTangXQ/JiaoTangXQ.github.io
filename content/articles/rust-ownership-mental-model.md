## 所有权不是 Rust 的发明

在 C 里，你 `malloc` 了一块内存，你就得记住 `free` 它。在 Python 里，垃圾回收器帮你管。在 Rust 里，编译器帮你管。

三种语言，三种策略，但底层问题是一样的：**谁负责这块内存的生命周期**。

Rust 的创新不是发明了所有权概念，而是**把它变成了类型系统的一部分**。这意味着所有权错误在编译时就能被发现，而不是在运行时崩溃。

## 心智模型：值是有主人的

把每个值想象成一个物理对象。每个物理对象在任意时刻只能有一个主人。

```rust
let s1 = String::from("hello");
let s2 = s1; // s1 把所有权转移给了 s2
// s1 不能再使用了
```

这就像你把一本书递给朋友 —— 书离开了你的手，你不能假装还持有它。

## 借用：看一眼但不拿走

有时候你不需要拥有一个值，你只需要看一眼。

```rust
fn print_length(s: &String) {
    println!("{}", s.len());
}

let s = String::from("hello");
print_length(&s); // 借用 s，但不拿走
println!("{}", s); // s 仍然可用
```

`&` 就是"我借你看一下"。不可变借用可以同时有多个，可变借用同一时间只能有一个。这消除了数据竞争。

## 生命周期：借多久

如果你借了一本书，总得还。生命周期标注告诉编译器"这个引用至少活多久"。

大多数时候编译器能自动推断。只有在函数签名中引用的来源不明确时，你才需要手动标注。
