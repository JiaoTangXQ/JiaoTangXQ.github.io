# 第十三卷回收索引：界面世界里的分组折叠先替人压缩注意力长会话才不至于失控

## 实现链
界面世界里有些设计，不是为了多展示一点信息，而是为了让人别在长会话里被淹没。

这条线真正逼出来的，是第十三卷里关于注意力压缩和小连接件的判断。

这页实际回收的实现链是：
- `11-07-04` 把分组和折叠解释成替人压缩注意力，这会在 `13-03-02` 里变成“小连接件的价值是少失控一次”。

好的界面不是把所有东西都摊开。  
它要先替人把信息压成还能处理的形状，长会话才不至于失控。

## 普通做法
更普通的做法，是把界面当展示壳，状态散落在各个组件或服务里，最后再临时拼起来给人看。

## 为什么不用
Claude Code 不这么做，是因为跨子系统事实如果没有同一个落屏世界，界面很快就会出现各说各话的假真相，长会话尤其容易失控。

## 代价
代价是 AppState 会更大，显示层也得承担一部分制度、接缝和注意力管理逻辑。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
