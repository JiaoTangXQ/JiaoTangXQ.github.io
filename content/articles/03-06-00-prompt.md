# prompt 命令导读：命令也能暂时重塑一段会话

这一章要先提醒读者：`prompt` 命令最关键的地方，不是又多了一种命令，而是命令第一次有能力去改写接下来这一小段会话该怎样继续。

所以这一章的三片叶子，要连起来读成“工作方式怎样被装进当前会话”。先看 [03-06-01-getPromptForCommand真正在生成什么.md](/Users/xxx/tmp/cc/docs/claude-code-book/03-用户一句话怎样进入系统/03-06-prompt命令装进当前会话/03-06-01-getPromptForCommand真正在生成什么.md)，再顺着技能正文/工具许可/hooks 一起装进去，以及最后这页“命令系统的灵魂”往下读，这一章才会从“prompt 细节”变成“命令怎样暂时重塑协作制度”的入口。

## 实现链
`prompt` 命令不是吐一段文本给用户，而是组装新的 system / user 消息、allowed tools、model、effort、hook 上下文，再把这组东西交回主循环，让会话暂时进入一套新的工作模式。

## 普通做法
很多产品把命令理解成“触发一个功能”，功能跑完就结束。

## 为什么不用
因为 skill/command 在这里经常要持续影响后续几轮推理，像临时换工作流，而不只是执行一次动作。

## 代价
命令结果不再一眼可见，往往要进入下一轮才能看出效果；但这让命令真正成为会话级工具，而不是菜单按钮。
