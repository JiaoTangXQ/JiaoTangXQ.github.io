# 默认值里连思考和提示都先带着产品立场

默认状态里连 `thinkingEnabled` 和 `promptSuggestionEnabled` 都是在出生时就算好的，这说明 Claude Code 不把产品立场散落到各个组件临时判断。哪些能力默认打开、哪些提示默认存在，先写进共同现实，再交给界面显示。

所以用户进入的不是一块空终端，而是一台已经有性格、有制度的工作台。它先替你把开局世界搭好。

## 实现链

`getDefaultAppState()` 里不只填技术字段，连 `thinkingEnabled` 和 `promptSuggestionEnabled` 这种体验选项也在默认值里先定好，分别走 `shouldEnableThinkingByDefault()` 和 `shouldEnablePromptSuggestion()`。也就是说，开局状态连产品立场都先写进去了。

## 普通做法

更普通的做法，是把这类开关延后到组件里判断，或者等用户点到相关区域时再按需初始化。

## 为什么不用

Claude Code 不愿意让这些体验开关散在组件层，是因为它们会影响输入、提示和主循环的默认行为。既然会改写用户的第一感受，就应该在共同状态里先讲清楚。

## 代价

代价是默认状态不再只是“技术初始化”，而带着明确的产品判断；好处则是全系统从一开始就站在同一种默认体验上。
