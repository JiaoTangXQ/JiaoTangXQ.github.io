# 技能正文、工具许可和 hooks 为什么要一起装配进会话

面试题版本：**"一个 Claude Code 技能（skill）通常包含哪些部分？如果只加载技能正文（系统提示词）而不加载对应的工具许可和 hooks，会出现什么问题？"**

## 技能的三个组成部分

一个完整的技能文件（`~/.claude/skills/code-review.md`）通常包含：

**技能正文**（系统提示词）：
```markdown
你是一个严格的代码审查专家。专注于：
1. 安全漏洞
2. 性能问题
3. 代码可读性

你可以使用 file_read 工具查看代码，但不应该直接修改代码。
```

**工具许可声明**：
```yaml
allowed_tools: [file_read, grep, glob]
forbidden_tools: [file_edit, bash]
```

**hooks 配置**：
```yaml
post_session_hook: generate_review_summary
pre_file_read_hook: check_file_scope
```

这三个部分在语义上互相依赖。

## 分开加载会出现什么问题

**只加载技能正文，不加载工具许可**：

系统提示词说"你可以使用 file_read 工具查看代码，但不应该直接修改代码"，但如果工具许可是默认的（允许所有工具），模型仍然可以调用 file_edit 工具。

结果：技能的意图（只读代码审查）和系统的能力（可以修改代码）之间有矛盾。如果模型出于"帮助用户"的理由尝试修改代码，这违反了技能的预期行为。

**只加载工具许可，不加载技能正文**：

工具许可限制了只能用 file_read，但没有告诉模型为什么、在什么场景下使用。模型没有上下文，可能用 file_read 做一些和代码审查无关的事情。

**只加载正文和许可，不加载 hooks**：

`post_session_hook: generate_review_summary` 这个 hook 在会话结束时自动生成审查摘要报告。如果这个 hook 没有被装配，审查完成后不会有自动摘要，用户需要手动要求生成。

这不是"可选功能没有"，而是技能承诺的行为没有兑现——技能描述里可能说"审查完成后会自动生成摘要"，但实际上没有。

## 一起装配的实现

`prompt` 命令在加载技能时：

```typescript
async function activateSkill(skillPath: string) {
  const skill = await parseSkillFile(skillPath)
  
  // 三者打包，一次性应用
  setAppState(s => ({
    ...s,
    systemPrompt: skill.systemPrompt,  // 技能正文
    toolPermissions: skill.toolPermissions,  // 工具许可
    activeHooks: {
      ...s.activeHooks,
      ...skill.hooks,  // hooks 配置
    },
  }))
  
  // 技能激活后，所有三部分同时生效
}
```

三者同时生效，不是先生效一部分再等待其他部分。这避免了中间状态：不存在"技能正文已生效但工具许可还是旧的"这种瞬间。

## 为什么需要整体装配

技能正文、工具许可和 hooks 共同定义了"这次会话的工作框架"。这个框架必须是一致的——三个部分要互相理解、互相支持。

**技能正文**告诉模型该做什么（代码审查）、能用什么（file_read）、不该做什么（不改代码）。
**工具许可**强制执行这些约束（禁止 file_edit 调用）。
**hooks**补充自动行为（完成后自动生成摘要）。

三者一起，才构成了一个完整的"代码审查工作框架"。缺少任何一部分，框架都是不完整的，技能的实际行为会偏离技能的设计意图。

## 与恢复的关联

技能被激活后，会话可能经历压缩（compact）和恢复（resume）。在这些过程中，Claude Code 需要知道"这次会话从哪个技能开始的，以便在恢复时重新应用正确的配置"。

如果三个部分分开装配、分开记录，恢复时可能只恢复了其中一部分（比如只恢复了技能正文，但工具许可和 hooks 的状态丢失了）。

整体打包装配意味着整体可以被序列化和恢复：一次性保存技能的完整引用，恢复时重新激活整个技能，三个部分一起恢复。

## 面试角度的总结

技能正文、工具许可和 hooks 三者必须一起装配进会话，是因为它们共同构成了一个语义上连贯的"工作框架"。分开加载会导致框架的内在矛盾（正文说一套、能力是另一套），让技能承诺的行为无法兑现。整体装配保证了一致性，也为压缩/恢复机制提供了原子性的会话配置单元。
