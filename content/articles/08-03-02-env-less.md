# env-less拿掉的不是步骤，而是一层世界

## 实现链
- `initReplBridge.ts` 同时保留 env-less 路径，核心思路不是“少跑几行代码”，而是把原来独立的环境分发层拿掉，让桥接用更直接的 bootstrap 上下文进入 `initBridgeCore()`。
- 也就是说，env-less 少掉的不是几个中间步骤，而是整整一层“先选环境、再开会话”的世界观。

## 普通做法
- 普通做法往往只保留一种远端启动方式，避免两套路径并存。
- 这样代码最整洁，也最容易解释。

## 为什么不用
- Claude Code 没强行只留一条，是因为它既要支持完整托管环境，也要支持更轻的 bridge 形态。两者共享核心，但前置世界不同。
- 从代码结构看，它是在复用核心会话桥接，而不是假装所有远端场景都属于同一制度层。

## 代价
- 代价是桥接层必须长期维护两条进入路径，阅读和测试都更累。
- 这不是最省心的架构，但在产品形态尚未完全收敛时通常更现实。
