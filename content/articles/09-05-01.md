# 文件邮箱不是落后，而是团队最低共同通信面

很多人会嫌 file-based mailbox 老派，但在 Claude Code 这里，它正好承担了“所有 teammate 都能理解的最低共同语言”。

## 实现链

`teammateMailbox.ts` 给每个 teammate 建一个 `~/.claude/teams/{team}/inboxes/{agent}.json`。pane teammate 和 in-process teammate 都能通过 `writeToMailbox()`、`readMailbox()` 收发消息。`PaneBackendExecutor` 的 `sendMessage()` 也明确说明：所有 teammate 最终共用同一套 mailbox 机制。

这说明 mailbox 不是补丁，而是被故意选成跨后端统一协议。

## 普通做法

普通做法会让同进程成员走内存队列，跨进程成员走 IPC 或 socket，不同后端各用各的最快办法。

## 为什么不用

Claude Code 没把通信拆成几套，是因为团队语义比局部速度更重要。只要通信底层不同，很多上层规则就会出现“这个后端能这样，那个后端不行”的分裂。

所以它宁可选一个朴素但统一的底层面，也不愿让团队系统碎成多种方言。

## 代价

代价当然是性能和优雅度看起来不如纯内存消息通道，尤其在同进程场景下显得笨重。但它换来了跨模式一致。
