# 同步子 agent 为什么一开始也先拿前台任务身份

同步子 agent 看起来“不需要任务系统”，因为它本来就在当前回合里跑完。但 Claude Code 还是先把它登记成前台任务。

## 实现链

在 `AgentTool.tsx` 的同步执行路径里，真正进入 `runAgent()` 之前，会调用 `registerAgentForeground()`。这一步给同步子 agent 建立任务对象，并返回 `backgroundSignal`。也就是说，哪怕它原本准备同步跑到底，系统也先把它放进一个可被背景化的前台任务框架里。

这不是为了装样子，而是为了让它随时可以从“当前回合内执行”切到“后台继续执行”。

## 普通做法

普通做法会直接执行。反正同步任务理论上应该很快跑完，没必要先登记，更没必要给它准备后台转场协议。

## 为什么不用

Claude Code 不走这条路，是因为“同步”只是当前选择，不是本体属性。一个看起来同步的子 agent，可能因为读文件太多、搜索太久、模型回合过长而需要让出前台。如果不先注册成任务，就只能中途硬切、重建或重新拉起。

它先给同步子 agent 一个前台任务身份，等于先给了它体面转场的资格。

## 代价

代价是即便很快完成的同步任务，也要经历一轮任务注册和清理，结构上比“直接跑完就返回”重得多。
