# 面试题：prompt 型命令执行完，会话发生了什么变化？

## 追问

面试官：「用户执行 `/review`，这是一个 `prompt` 型命令。它执行完以后，会话里发生了什么？」

候选人 A：「发了一个请求，模型返回了 code review。」

候选人 B：「把 review skill 的内容注入当前会话，设置了 `shouldQuery: true`，主循环继续推进，模型收到带有技能正文的消息后开始推理。」

哪个答案更准确？都不够准确。真正的答案需要看 `getMessagesForPromptSlashCommand` 的源码。

## getMessagesForPromptSlashCommand 做了什么

```typescript
// 1. 调用命令的 getPromptForCommand，拿到技能内容
const result = await command.getPromptForCommand(args, context)

// 2. 注册 hooks（如果技能有 hooks 配置）
if (command.hooks && hooksAllowedForThisSkill) {
  registerSkillHooks(context.setAppState, sessionId, command.hooks, command.name, ...)
}

// 3. 记录技能调用（用于压缩时保留）
addInvokedSkill(command.name, skillPath, skillContent, getAgentContext()?.agentId ?? null)

// 4. 解析额外工具许可
const additionalAllowedTools = parseToolListFromCLI(command.allowedTools ?? [])

// 5. 构建消息内容（技能正文 + 图片 + 前置 blocks）
const mainMessageContent = imageContentBlocks.length > 0 || precedingInputBlocks.length > 0
  ? [...imageContentBlocks, ...precedingInputBlocks, ...result]
  : result

// 6. 提取附件
const attachmentMessages = await toArray(getAttachmentMessages(...))

// 7. 组装最终消息列表
const messages = [
  createUserMessage({ content: mainMessageContent, isMeta: true }),  // ← 技能内容对用户隐藏
  ...attachmentMessages,
  createAttachmentMessage({ type: 'command_permissions', allowedTools: additionalAllowedTools, model: command.model }),
]

// 8. 返回，触发主循环继续
return {
  messages,
  shouldQuery: true,
  allowedTools: additionalAllowedTools,
  model: command.model,
  effort: command.effort,
  command
}
```

执行完这个函数，会话里发生了七件事，不是一件。

## 七件事意味着什么

**技能正文作为 isMeta 用户消息注入**

技能的 prompt 内容（「你是一个 code reviewer，请按以下标准…」）被包装成 `isMeta: true` 的用户消息。用户 transcript 里看不到这段指令，但模型在当前轮次会看到。

这保证了技能内容不会污染用户阅读的会话记录，但仍然在本轮影响模型的推理方向。

**hooks 被注册到当前会话**

如果技能的 frontmatter 里声明了 hooks：

```yaml
---
hooks:
  PostToolUse:
    - matcher: "Bash"
      hooks:
        - type: command
          command: "./check-style.sh"
---
```

这些 hooks 在技能被调用时注册到 session 级别，从此这个 hook 会在后续每次匹配到 `Bash` 工具调用后触发，直到 session 结束（或 `once: true` 触发一次后自动移除）。

**工具许可被扩展**

`parseToolListFromCLI(command.allowedTools ?? [])` 把技能声明的额外工具权限解析出来，返回的 `allowedTools` 会被传给本轮的 API 调用，临时扩展这轮推理能用的工具集。

**技能调用被记录**

`addInvokedSkill` 在 `bootstrap/state.ts` 里记录了「本次会话激活了哪些技能」。这在压缩（`/compact`）时用到——压缩算法会保留已激活技能的 frontier，确保压缩后技能仍然「在线」。

## 为什么说它更像「装入工作方式」而不是「触发一次动作」

一次 `prompt` 命令可能同时改变：
- 当前轮次的系统指令（技能正文）
- 当前轮次的可用工具集（allowedTools）
- 当前 session 的 hooks 集合（持续到 session 结束）
- 后续压缩的行为（技能调用记录）
- 本轮的模型选择（command.model）
- 本轮的推理深度（command.effort）

这些变化有的只影响本轮，有的影响整个 session。这不是「触发一次动作」，而是在当前会话的运行制度里装入了一套新的工作条件。

---

*面试指导：被问到「skill 系统和 prompt 模板有什么区别」时，从「hooks 注册」和「allowedTools 扩展」这两个不是模板系统能做到的能力切入，说明 skill 改变的是会话制度，不只是发给模型的文本。*
