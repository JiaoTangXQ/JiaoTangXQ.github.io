# 第十三卷回收索引：前四卷里的不同来源最后收编成同一种 `Command` 语言

## 实现链
内建命令、技能目录、插件和 MCP 最后都被翻成同一种 `Command` 形状，靠的是名字、类型、加载方式、可见性和执行上下文这一套共同身份，再由 `findCommand()`、`hasCommand()`、格式化函数和 slash 处理器统一消费。

## 普通做法
更普通的做法，是给不同来源保留不同结构，最后再在入口层写很多 `if/else` 去适配。

## 为什么不用
Claude Code 不这么做，因为入口层不想知道“你来自哪里”才能决定怎么显示、怎么追踪、怎么过权限。

## 代价
代价是抽象会让最简单的命令也显得规格很重，但这换来的是来源还能继续扩张。
