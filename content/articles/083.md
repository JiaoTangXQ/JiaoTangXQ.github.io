# 第十三卷回收索引：前四卷里的清场补救与分级续写

## 实现链

这页具体回收的判断是“前四卷—清场补救与分级续写”。
这页回收的不是一个抽象评价，而是 `QueryEngine`、`query()`、`StreamingToolExecutor`、`query/stopHooks.ts` 这条回合链上的真实制度。代码不是让模型先完整输出、再把工具当成收尾动作统一处理；它会在工具调用、停止钩子、`tool_result` 回写、`transition` 决定是否继续这些节点上分段收口。换句话说，Claude Code 的“一轮”不是一句话，而是一串被制度看住的状态转换。

## 普通做法

更普通的 agent 写法，会把一轮对话理解成线性的三步：用户输入、模型回答、必要时跑一下工具。这样写起来更顺，主循环也更短，因为工具、继续理由和中断收尾都被当成旁支。

## 为什么不用

Claude Code 没走这条路，主要是因为它要处理的不是“能不能答一句话”，而是“行动开始后怎么不中途失控”。一旦工具会并发、会被打断、会失败、会留下需要继续推进的状态，主循环就不能只靠一个简单 for/while 收尾，而必须把继续理由、结案理由和清理动作编进运行时。

## 代价

这套做法的好处是恢复性强，哪里停、为什么停、下一步为什么还能继续，代码里都有落点。代价也很明显：回合逻辑被拆散在多个文件和多个阶段里，新手第一次读会觉得系统在“绕”，而不是像普通聊天循环那样一眼看穿。
