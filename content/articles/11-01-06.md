# 指标在进程退出时统一落盘说明观测不该反拖主循环

`StatsProvider` 不是每记一笔指标就同步写盘，而是在进程退出时统一刷一次。这个决定很务实：观测应该尽量便宜，不能反过来拖慢主循环。

指标是复盘材料，不该变成运行时负担。Claude Code 宁可晚一点结账，也不让记账打断干活。

## 实现链

`StatsProvider` 会在 `process.on('exit', flush)` 里统一冲刷指标，并通过 `saveCurrentProjectConfig` 把统计结果落到项目配置，而不是每记一条就同步写盘。指标在退出时集中收口，说明观测层不该反拖主循环。

## 普通做法

更直觉的做法，是每次状态变化都顺手把统计写出去，或者哪个事件产生了数据就在哪个 handler 里立即持久化。

## 为什么不用

Claude Code 不愿意把观测写盘塞进每次交互，是因为这会把监控成本直接叠到主路径上。统计是辅助现实，不该为了记账把输入、渲染和主循环都拖慢。

## 代价

代价是退出前这次集中 flush 必须可靠，否则统计可能丢最后一截；但整体上，它把观测成本从热路径挪到了收尾时刻。
