# 连群发都被要求克制，说明团队沟通不是默认聊天室

团队当然需要广播，但 Claude Code 明确把广播当成少用能力，而不是默认姿势。

## 实现链

`teammatePromptAddendum.ts` 规定：`SendMessage` 可以 `to: "*"`, 但要 “sparingly”。这不是文案修饰，而是制度设计。系统允许全员广播，却先在系统提示里提醒它属于高成本动作。

## 普通做法

普通做法会把团队通信做成聊天室模型。所有消息默认大家都看见，最多再额外支持私聊。

## 为什么不用

Claude Code 不把团队当聊天室，是因为广播太便宜时，团队信息熵会迅速失控。每个成员都能顺手把半成品 thoughts 丢给所有人，最后没人能分清哪些是必须处理的正式信号。

所以它把广播保留下来，但把默认姿势仍然放在定向消息上。

## 代价

代价是协调者需要更认真地决定消息路由，写起来没那么随意。但这换来了更低的噪音。
