# 颜色和 pane 先保护的是人的团队心智模型

团队界面的颜色和 pane 在 Claude Code 里不是皮肤，而是组织认知工具。

## 实现链

`teammateLayoutManager.ts` 用 `assignTeammateColor()` 为每个 teammate 分配稳定颜色，pane backend 再把这个颜色用于边框和标题。tmux 布局里 leader 固定在左侧，teammate 在右侧扩展。也就是说，视觉布局直接编码了“谁是 lead、谁是谁”。

## 普通做法

普通做法会把颜色和布局当美化层，顶多做得好看点。

## 为什么不用

Claude Code 不把它当装饰，是因为多 agent 团队最怕用户和开发者同时失去方位感。颜色和空间位置越稳定，人脑越容易记住“蓝色那个是 researcher，左侧那个是 lead”。

## 代价

代价是视觉系统要和团队生命周期耦合起来，清理、复用、重建都得顾及身份连续性。
