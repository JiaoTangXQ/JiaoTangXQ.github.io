# 粘顶提示只按 idx 去重，说明估算位置会漂但身份不该跟着乱跳

## 实现链
StickyTracker 的 lastIdx 只按消息 idx 去重，不把 estimate 一起算进去。因为 estimate 会随着滚动每帧变化，真正该保持稳定的是这条 prompt 的身份，不是它的临时估值。

## 普通做法
普通做法可能会把位置估计也放进去重键里。那样只要 scrollTop 变一点，sticky prompt 就会不断重发。

## 为什么不用
这里承认估算会漂，但不该让身份跟着乱跳。只按 idx 去重，才能让同一条 prompt 在滚动里保持“还是它”。

## 代价
代价是显示内容要接受估算和身份分离这件事，调试时也更绕。好处是 sticky header 不会因为微小的位移抖成一串新节点。
