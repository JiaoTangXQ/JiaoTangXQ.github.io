# 第十三卷回收索引：任务世界里 `outputFile` 把后台过程留成外部痕迹，劳动能被重新接上

## 实现链
任务世界另一条线，不在“认不认领”，而在“过程能不能被重新接上”。

这条线真正逼出来的，是第十三卷里关于长期工作台观的另一半判断。  
成熟工作台不只接最终结果，也要给后台过程留下可重新找到的外部痕迹。

这页实际回收的实现链是：
- `05-03-02` 把 `outputFile` 讲成后台工作的外部痕迹，这会继续支撑 `13-07-02`，说明系统不只要知道任务做过什么，还要给这份劳动留下随时能重新接上的过程依附点。

一份后台劳动之所以能被重新接上，不是因为最后会弹出一句“完成了”。  
而是因为 Claude Code 先把过程留成 `outputFile` 这种外部痕迹，让劳动本身还有回来的路。

## 普通做法
更普通的做法，是把后台动作当成松散的异步句柄，跑起来就算开始，结束了再临时补状态和通知。

## 为什么不用
Claude Code 不这么做，是因为没有统一任务对象，就没有稳定的生命周期、输出、通知和回看链，前后台切换也会变成一串彼此不认账的临时处理。

## 代价
代价是任务状态字段、输出文件、通知协议和前后台转场逻辑都会变厚，维护者要记住更多接缝。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
