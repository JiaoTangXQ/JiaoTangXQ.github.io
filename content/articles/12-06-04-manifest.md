# manifest 内容没变就不重写，说明接入链也要尽量少惊动用户和浏览器

安装 manifest 时，系统不会每次都粗暴重写文件，而是先读旧内容，只有真的变了才写入。这种克制非常重要，因为接入链一旦太爱“自我修复”，就会不断惊动浏览器、惊动磁盘、惊动用户对系统稳定性的信任。

很多工程问题不是因为做不到，而是因为太爱乱动。Claude Code 在这里反而选择“能不碰就不碰”，这正是成熟接入链该有的手感。

## 实现链
`installChromeNativeHostManifest()` 和 `createWrapperScript()` 都先读旧内容，只有内容变了才重写文件。这里不是小优化，而是在减少不必要的浏览器侧扰动。

## 普通做法
普通做法是每次启动都覆盖写一遍，反正文件不大。

## 为什么不用
Claude Code 不这么做，因为 manifest 改写后往往还要引发 reconnect 或系统侧重新感知。没有变化时还去重写，只会制造额外噪音。

## 代价
代价是代码要多做一次读取和比较，接入逻辑看起来也更繁琐。
