# needs-auth缓存说明平台总线也要记住谁暂时还没准备好

## 实现链
- `client.ts` 会把需要重新认证的 server 记进 `mcp-needs-auth-cache.json`，并给它一个 15 分钟 TTL。之后装配连接时，客户端可以直接把这个 server 当成 `needs-auth`，而不是每次都重新撞一遍失败流程。
- 代码表达得很明确：平台总线不仅要记住“谁已经连上”，还要记住“谁现在还没准备好”。

## 普通做法
- 普通做法是每次启动都重新连，连不上就再报一次错，让用户自己忍受反复弹出的认证失败。
- 这种方案实现简单，因为不需要本地缓存额外状态。

## 为什么不用
- Claude Code 没这么做，是因为 MCP server 的认证失败往往不是一秒后就会自己好。持续重试只会制造噪音、浪费时间，还会把 UI 搞成反复闪烁的失败现场。
- 从代码看，它宁可临时保留一份“暂时别碰它”的记忆，也不愿每次都重新撞墙。

## 代价
- 代价是缓存天然有陈旧风险，用户如果刚修好认证，短时间内可能还会看到 `needs-auth` 状态。
- 不过作者用 TTL 把这个代价限制在可接受范围里，这种权衡总体是合理的。
