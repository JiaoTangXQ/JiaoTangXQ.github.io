# 真正的优雅不是连得更远，而是连远了以后系统还像一个整体

Claude Code 在外延执行这块真正漂亮的地方，不是能连到桌面、浏览器和桥，而是这些东西连进来以后，系统仍然像一个整体。`mcpServer.ts` 里，浏览器桥不是只管发消息，它还要带上 permission mode、持久设备身份、safe analytics keys 和统一的 `sideQuery` 约束；`setup.ts` 里，扩展自动启用、wrapper、manifest、重连页也都不是各写各的临时脚本，而是在配合主系统维持一个可继续工作的现场。

更普通的工程会把“连得更远”本身当成成功，于是只要桥打通了、工具能调了，就算大功告成。Claude Code 明显不是这个标准。它还在乎断线文案是不是说明真实原因、配对身份是不是会持久写回、日志是不是只放行安全白名单、桥上推理是不是最后只把 text blocks 翻回宿主能接住的语言。

这就是为什么这里值得叫“优雅”。不是因为技术炫，而是因为它在边界扩张之后还坚持保持整体性。代价也很清楚：所有外延能力都要额外承担适配、收口和解释成本。只是如果不这么做，所谓外延执行最后只会变成一堆互不相认的外挂。

## 实现链
`createChromeContext()` 同时处理 socket、bridge、logger、配对、权限和推理；`wrapper.tsx` 又把 computer use 的锁、应用授权、截图尺寸和中断统一回 AppState。不同外延能力虽然来自不同模块，但都被接回同一个整体。

## 普通做法
普通扩张往往是每连一种新世界，就多一小套独立机制。

## 为什么不用
Claude Code 不愿意把系统长成一串外挂，因为外挂越多，恢复链和制度边界就越容易断。

## 代价
代价是中间接缝件很多，优雅感主要来自收口，而不是来自代码更少。
