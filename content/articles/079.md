# 第十三卷回收索引：连续性世界里的外置记忆与长期沉淀

## 实现链
这页对应的实现链，主要落在 `microcompact`、`autocompact`、`compact.ts`、`SessionMemory`、`auto-memory`、`stopHooks` 这条时间制度链上。它真正解释的是：长会话为什么不能只靠“记更多”或者“超长再总结”，而要靠分层整理和回合后插槽。

## 普通做法
更普通的做法，是上下文太长了再整体摘要，或者直接截断旧历史，把连续性问题留给用户自己兜底。

## 为什么不用
Claude Code 没这么做，因为它要保住的是继续工作，而不是只让 token 勉强够用。整理动作如果抢进主回合，反而会打断当前劳动。

## 代价
代价是整理逻辑会分散在多个阶段，时机判断更难，连续性相关代码也明显更重。
