# 远端消息先学会说本地语言，现有界面才能继续复用

## 实现链
- `sdkMessageAdapter.ts` 会把远端 `SDKMessage` 转成本地 `Message`、`StreamEvent` 或系统消息，`useRemoteSession.ts` 和 `useAssistantHistory.ts` 都复用这套转换。
- 这意味着远端消息不是直接闯入 UI，而是先学会说本地消息语言，再进入已有渲染链。

## 普通做法
- 普通做法是远端场景单独做一套消息类型和渲染组件，本地界面继续用本地那套。
- 这样每条线都更贴近自己的后端协议。

## 为什么不用
- Claude Code 没再造第二套消息体系，是因为它想继续复用本地的消息渲染、搜索、分隔线和工具结果处理。
- 如果远端坚持讲自己的语言，本地 UI 就得变成双语系统，复杂度会一路扩散。

## 代价
- 代价是 adapter 必须做一些压缩和取舍，某些远端语义会被翻成更普通的本地形状。
- 但这比复制一整套 UI 基础设施更划算。
