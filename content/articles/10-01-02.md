# 好系统不是硬记一切，而是承认必须分层留存

## 实现链

这一页重点看“好系统不是硬记一切，而是承认必须分层留存”。对应源码主要是 `src/query.ts`、`src/services/compact/autoCompact.ts`。
这一章真正落在 `query.ts` 对消息窗口、token 使用和回合推进的长期处理上。Claude Code 先承认长会话一定会越做越重，再从代码上分出“当前窗口”“压缩后历史”“外置记忆”这几层留存方式，而不是假装一切都能原样塞回模型。

## 普通做法

更直觉的做法，是尽量保留所有历史，等上下文真的爆了再想办法。这样最符合“别丢信息”的本能。

## 为什么不用

Claude Code 没这么做，因为长会话越成功，积累就越多；成功本身会制造上下文压力。它必须先承认分层留存是常态，而不是把上下文爆炸当成偶然事故。

## 代价

这种现实主义做法更稳，但代价是系统必须公开承认“不是所有历史都会一直留在主窗口里”，用户也要接受连续性被分层管理。
