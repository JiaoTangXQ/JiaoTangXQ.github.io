# 第十三卷回收索引：前四卷里的并发安全先守住世界一致性

## 实现链
`partitionToolCalls()` 和 `StreamingToolExecutor.canExecuteTool()` 会先看 `inputSchema.safeParse()`，再按 `isConcurrencySafe()` 把工具拆成只读 batch 和独占 batch；并发工具的 `contextModifiers` 也会先排队，等这一批结束后再按顺序应用。

## 普通做法
更普通的做法，要么一律串行，要么只要是多个 `tool_use` 就一律并发。

## 为什么不用
Claude Code 不这么做，因为读文件、搜代码这类近似只读动作，和 Bash、编辑、权限修改这种强副作用动作，本来就不该放在同一种并发规则里。

## 代价
代价是工具作者必须认真声明并发安全性，一旦判断错，bug 会比纯串行更隐蔽。
