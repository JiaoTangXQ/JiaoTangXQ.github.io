# 第十三卷回收索引：前四卷里的真实边界与层级秩序

## 实现链
这页对应的实现链，主要落在 `main.tsx`、`QueryEngine.ts`、`query.ts`、`Tool.ts`、工具调度、恢复链和前置制度装配上。它真正回收的是：前四卷里那些启动、主循环、接缝和恢复判断，为什么要作为一套内核主干被重新钉住。

## 普通做法
更普通的做法，是把前几卷只当“流程介绍”，等细节复杂了再临时补一些例外说明。

## 为什么不用
Claude Code 的前段主干不是顺序图而已，它本来就承载了分层、权限、恢复和统一语言这些基础判断。如果不重新收口，后面的很多世界都会失去共同地基。

## 代价
代价是会出现更多回收页、更多导航页，读者也需要接受“前段主干”会被多次回看。
