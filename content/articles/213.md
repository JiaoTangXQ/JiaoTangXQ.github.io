# 第十三卷回收索引：界面世界里界面先被承认为运行时一部分

## 实现链
界面世界里先要钉住的，不是某个组件的样子，  
而是界面能不能被当成运行时的一部分。

这条线真正逼出来的，是第十一卷里“UI 不是外壳，而是运行时本体”这条判断。

这页实际回收的实现链是：
- `11-01-02` 把界面解释成运行时的一部分，这会在 `13-02-01` 里再次出现，说明 UI 不是外壳，而是运行时本体的一部分。

界面如果还被当成纯展示层，后面的状态就只能悬空。  
先承认它是运行时的一部分，后面的落屏才有基础。

## 普通做法
更普通的做法，是把界面当展示壳，状态散落在各个组件或服务里，最后再临时拼起来给人看。

## 为什么不用
Claude Code 不这么做，是因为跨子系统事实如果没有同一个落屏世界，界面很快就会出现各说各话的假真相，长会话尤其容易失控。

## 代价
代价是 AppState 会更大，显示层也得承担一部分制度、接缝和注意力管理逻辑。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
