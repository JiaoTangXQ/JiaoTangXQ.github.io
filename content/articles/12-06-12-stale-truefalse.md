# 只缓存正向检测，宁可保留 stale true 也不缓存 false，说明错判代价被认真比较过

扩展安装检测这件事，Claude Code 没有简单做成“扫到什么就都缓存”。它只缓存正向检测，不缓存负向结果，因为一个错的 `false` 可能把自动启用永久毒死；一个过时的 `true` 最多也只是多尝试连一次。这里已经不是“会不会缓存”的问题，而是在认真比较两种错判各自有多贵。

这种对代价的精算，正是成熟工程的味道。系统不是追求绝对正确，而是在不完美现实里选伤害更小的错。

## 实现链
`isChromeExtensionInstalled_CACHED_MAY_BE_STALE()` 的注释把取舍讲得很清楚：只持久化正向检测，不缓存 `false`。因为 stale `true` 最多带来一次静默连接尝试，而 stale `false` 可能把自动启用永远毒死。

## 普通做法
普通缓存会把 true/false 都记下来，下一次直接用。

## 为什么不用
Claude Code 不缓存 `false`，是因为配置文件可能在多台机器、远端环境和 Homespace 之间共享，负例比正例更危险。

## 代价
代价是它必须接受“缓存真值偶尔过时”的现实，偶发多做一次连接尝试。
