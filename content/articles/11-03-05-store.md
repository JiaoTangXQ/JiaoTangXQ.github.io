# store 只在第一次挂载时创建说明现实源头不能跟着重渲染换壳

这里的 `store` 不是每次渲染顺手重建，而是借 `useState` 只在第一次挂载时创建。这样整段会话里，订阅源头、写入入口和状态身份都保持稳定。

这很重要。对外部 store 来说，现实源头一旦跟着重渲染换壳，所有订阅都会开始发抖。Claude Code 先把源头钉死，再让界面围着它转。

## 实现链

`AppStateProvider` 用的是 `useState(() => createStore(...))`，store 只在第一次挂载时创建，后面重渲染不会重建。现实源头一旦建立，就不能随着组件重渲染换壳。

## 普通做法

更随手的写法，是在 render 里直接 `createStore()`，或者把 store 依赖 props 每次重新算一遍。

## 为什么不用

Claude Code 不让 store 跟着 render 重建，是因为订阅、外部 watcher 和副作用总闸门都挂在这份 store 上。源头一换壳，整套订阅关系就会抖。

## 代价

代价是初始参数只在第一次真正生效，后面不能靠改 props 重建世界；但换来的是 store 引用稳定，订阅者心智也稳定。
