# 第十三卷回收索引：连续性世界里 postSamplingHooks 把整理正式安放进回合后插槽

## 实现链
连续性世界里真正重要的，不只是“晚一点整理”，  
而是系统把整理放进了一个正式、统一的回合后插槽。

这条线真正逼出来的，是第十三卷里关于制度先于便利的判断。

这页实际回收的实现链是：
- `10-09-02` 把 `postSamplingHooks` 写成统一的回合后插槽，这会在 `13-04-01` 里继续成立，说明整理不是临时补丁，而是正式时机。

好的连续性设计，不是给整理找一个随手空隙。  
它是先把 `postSamplingHooks` 变成统一插槽，再让整理稳定落位。

## 普通做法
更普通的做法，是一发现上下文变长就立刻整理，或者把当前状态和长期记忆统统塞回主会话，省掉额外的插槽和外置存储。

## 为什么不用
Claude Code 不这么做，是因为整理太早会打断主回合，记忆不分层会把短期噪音和长期知识混在一起，最后让连续性系统同时失去可读性和可恢复性。

## 代价
代价是需要额外维护 compact boundary、回合后插槽、外置 memory 和记忆回灌链，系统会更厚。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
