# 第十三卷回收索引：界面世界里的状态栏也被 hook 编排，说明制度已经住进显示层

## 实现链
界面世界里更细的一层，不是“有状态栏”，而是状态栏本身已经能被编排。

这条线真正逼出来的，是第十三卷里关于制度进入显示层的判断。

这页实际回收的实现链是：
- `11-05-02` 说明 `StatusLine` 连自己的文本都允许通过 hook 命令生成，这会在 `13-04-01` 里回收到“制度已经进入显示层，而不只是结果展示”。

状态栏也被 hook 编排，说明它不再只是装饰。  
它已经是制度进入显示层以后，最先被看见的那一层摘要。

## 普通做法
更普通的做法，是把界面当展示壳，状态散落在各个组件或服务里，最后再临时拼起来给人看。

## 为什么不用
Claude Code 不这么做，是因为跨子系统事实如果没有同一个落屏世界，界面很快就会出现各说各话的假真相，长会话尤其容易失控。

## 代价
代价是 AppState 会更大，显示层也得承担一部分制度、接缝和注意力管理逻辑。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
