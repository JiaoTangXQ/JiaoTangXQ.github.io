# Claude Code 界面与状态地图

Claude Code 的终端界面不是外壳，它是运行时把内部事实翻译给人的器官。

## 实现链

状态核心在 `state/AppState*` 和相关 selectors。界面主体在 `components/*`，上下文型配套在 `context/*`，比如通知、overlay、mailbox。很多组件并不是自己管一份局部 state，而是从 `AppState` 里订阅切片，再配合 selectors 派生出“当前该显示什么”。

## 普通做法

普通 CLI UI 会把终端当成打印窗口，状态很浅，界面更多是输出而不是运行时一部分。

## 为什么不用

Claude Code 不可能这么轻，因为它要显示工具进度、权限对话、后台任务、team view、消息流和输入驾驶舱。没有统一状态仓库和显式订阅纪律，UI 很快就会和执行现实脱节。

## 代价

代价是状态和界面之间的耦合明显增加，读代码时不能把 UI 简单视为“最后再看”的皮层。
