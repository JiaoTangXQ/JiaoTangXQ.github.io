# 外部元数据也能回灌本地状态说明现实是双向同步的

状态世界还承认一件事：现实不是单向的。`externalMetadataToAppState()` 允许外部会话元数据重新灌回本地，把权限模式和 `ultraplan` 标记写回共同现实。

这说明本地界面不是自说自话。外面如果已经改写了当前世界，本地也要服从这份更新。好的工作台，内外必须说同一种现实语言。

## 实现链

`externalMetadataToAppState()` 会把外部元数据重新翻译回本地状态，比如把外部 `permission_mode` 还原成 `toolPermissionContext.mode`。AppState 不是只单向喂给外界，它也承认外部现实会回灌本地。

## 普通做法

更普通的实现会把 UI store 当成单向终点：本地改完再往外发，至于外面怎么变，组件自己再补一层读取。

## 为什么不用

Claude Code 不把状态层只当单向出口，是因为远端、桥接和外部代理真的会改写本地现实。要是没有回灌链，本地 UI 只会一直盯着过期事实。

## 代价

代价是同步方向从单向变成双向，状态翻译层必须更小心；但这样至少能保住“外面真的变了，本地也要承认”。
