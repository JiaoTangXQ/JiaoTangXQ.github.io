# bridge 遥测只放行白名单字符串，说明边界扩张连日志都先收口

浏览器桥一旦接到真实页面，连日志都会变危险。Claude Code 在 `trackEvent()` 里没有贪心地把所有字符串都往分析系统里送，而是只允许极少数白名单字段通过，连 `status` 这样的键名都先改成更安全的 `bridge_status`。这是一种非常成熟的自觉：系统知道日志也可能泄露现场。

这类设计平时几乎没人夸，但它恰恰最能说明工程品味。真正把边界扩出去的系统，不能只给工具加护栏，还得给遥测、调试和运营侧的旁路都加护栏。

## 实现链
`SAFE_BRIDGE_STRING_KEYS` 只允许 `bridge_status`、`error_type`、`tool_name` 这类白名单字符串进入 `trackEvent()`；其他只收布尔和数字。连浏览器桥上的遥测也先收口，不让页面内容和用户数据顺手流进日志。

## 普通做法
普通做法是把整份 metadata 原样打进 telemetry，方便排查。

## 为什么不用
Claude Code 不敢这么贪，因为 bridge 层最容易碰到页面文本、站点数据和用户输入，观测便利不能直接压过隐私边界。

## 代价
代价是排障信息会少一截，某些问题只能靠更克制的信号间接判断。
