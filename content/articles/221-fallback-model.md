# 第十三卷回收索引：前四卷里的 fallback model 先清理现场再换路

## 实现链
前四卷里有些恢复动作，看起来只是“换一条路继续”，  
但它们真正先做的，其实是把现场清干净。

这条线真正逼出来的，是第十三卷里关于恢复能力不能白拿的判断。

这页实际回收的实现链是：
- `04-07-01` 把 fallback model 讲成先清理现场再换路，这会继续支撑 `13-05-02`，说明恢复能力从来不是白拿的。

真正成熟的 fallback model，不是先找下一条路。  
它先把现场清理掉，再让系统换路时不会把旧问题一并带走。

## 普通做法
更普通的做法，是把启动、输入、主循环里的这些小判断留在顺手逻辑里，不单独立规矩。

## 为什么不用
Claude Code 不这么做，是因为工作台一旦要长期运行，这些看似前期的小处理其实都是后面秩序能否站稳的根，不能等出事了再补。

## 代价
代价是前期代码和解释都会显得更重，也更早暴露出长期工程的野心。 对单页回收附录来说，额外成本是页数更多、重复解释也会增加。
