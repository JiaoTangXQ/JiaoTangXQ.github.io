# Claude Code 消息显示地图

消息显示层解决的不是“把字符串打印出来”，而是“怎样把复杂执行现场稳定翻译成可读对话”。

## 实现链

消息来源包括用户消息、assistant 消息、工具使用消息、系统消息、权限消息和任务通知。显示层要接住这些不同类型，再在 `components/Messages*`、输入通知区和长 transcript 组件里做分流、折叠、高亮和上下文提示。很多权限请求和任务通知虽然长得像消息，背后其实来自别的制度层。

## 普通做法

普通做法会把消息显示当成一个线性聊天列表：谁来一条就 append 一条。

## 为什么不用

Claude Code 不能这样简化，因为它展示的不只是聊天，还包括工具、权限、后台任务、compact 边界和 teammate 现场。如果只做线性 append，用户看到的会是一锅杂糅日志。

## 代价

代价是消息显示层必须理解更多消息类型，组件会比普通聊天界面复杂很多。
