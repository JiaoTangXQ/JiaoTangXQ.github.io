# 重连时宁可清空任务和工具进度，也不愿让远端状态永远漂错

## 实现链
- `useRemoteSession.ts` 在断线、重连和某些恢复路径里会清掉本地的工具进度、正在进行中的 tool use 集合和远端后台任务计数。
- 作者显然接受一个保守原则：不知道现在准不准时，先把可能漂掉的进度清空，不继续假装它们还可信。

## 普通做法
- 普通做法是把上次看到的进度继续保留在屏幕上，等远端回来再接着跑。
- 这样用户感知上更平滑，不会突然一片空白。

## 为什么不用
- Claude Code 没这么乐观，是因为远端重连后未必还能保证工具 ID、任务集合和进度流与本地缓存完全一致。
- 一旦旧进度留在界面上，本地工作台就会开始讲假故事。它宁可先清空，再等新事实回来。

## 代价
- 代价是用户会看到状态回退，体验上不如持续动画顺滑。
- 但对需要可信状态的工作台来说，暂时空白比永久漂错更可接受。
