# 模式提示放在任务标签前，说明制度现实比热闹更值得先保住

底部区有个很见功力的小决定：`mode` 提示被故意放在任务 pill 前面，免得后面那些很长的标签把当前制度状态挤出屏幕。也就是说，界面先保住“我现在处在哪种安全姿态”，再去保住“旁边还有多少热闹东西”。

这类顺序感特别重要。真正决定你接下来能不能放心行动的，不是有多少任务在跑，而是你现在到底开着哪种模式。

## 实现链

footer 组合顺序里，`modePart` 会被故意放在任务 pill 和其他 parts 前面，注释还写明是为了防止长任务标签把 mode indicator 挤掉。制度现实先于热闹信息，被直接编码进渲染顺序。

## 普通做法

更随手的实现通常按拼接顺序展示，谁先算出来谁先上，不太会给信息层级额外立规矩。

## 为什么不用

Claude Code 不愿意把 mode 当成普通附属标签，是因为 permission mode 会直接改写你下一步能不能做事，比“现在有几个后台任务”更基础。它宁可先保制度，再保热闹。

## 代价

代价是摘要层不只是展示，还在做信息优先级排序；这让底部实现更像编排器，不再是单纯字符串拼接。
