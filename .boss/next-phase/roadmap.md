## 三轨路线图

### Track A: SEO & 传播力

目标：让搜索引擎和社交平台正确展示焦糖星球的内容。

| ID | 任务 | 描述 | 收益 | 复杂度 | 波次 |
|----|------|------|------|--------|------|
| A1 | OG / Twitter Card 元数据 | 构建时为每篇文章生成独立 `<meta property="og:*">` 标签，包含标题、摘要、封面图 | 社交分享时有精美预览卡片 | 中 | 1 |
| A2 | robots.txt + sitemap.xml | 构建脚本自动生成，包含所有文章 URL | 搜索引擎可发现全部内容 | 低 | 1 |
| A3 | 文章页预渲染 | 构建时生成每篇文章的静态 HTML shell（含正文、meta），SPA hydrate | Google 可索引文章全文 | 高 | 2 |
| A4 | 自定义域名 | CNAME 配置 + DNS 指向 | 专业品牌形象 | 低 | 3 |

**核心技术挑战：**
- SPA 架构下 OG 标签需要构建时注入，不能靠客户端 JS
- 方案选项：(a) vite-plugin-prerender 构建时渲染每篇文章页 (b) 构建脚本直接生成带 meta 的 HTML 文件 (c) Cloudflare Workers 等边缘注入
- 推荐方案 (b)：构建脚本在 `npm run build:data` 阶段为每篇文章生成 `/article/<slug>/index.html`，包含完整 meta 标签 + SPA 引导脚本

---

### Track B: 视觉震撼升级

目标：把焦糖星球的视觉体验提升到「让人忍不住截图分享」的水平。

| ID | 任务 | 描述 | 收益 | 复杂度 | 波次 |
|----|------|------|------|--------|------|
| B1 | 宇宙缩放过渡 | 点击星球 → 相机俯冲推近 → 星球放大 → 溶解过渡到文章页；返回时反向缩回宇宙 | **体验分水岭**，最核心的沉浸升级 | 高 | 1 |
| B2 | 搜索飞行效果 | 选中搜索结果 → 相机平滑飞向目标节点 → 到达后展开 | 搜索体验质感 | 中 | 3 |
| B3 | 流星 + 星尘粒子 | GPU 粒子系统，流星随机划过，星尘随相机视差漂移 | 宇宙活力感 + 微交互 | 中 | 2 |
| B4 | 多层 FBM 星云 | Fractal Brownian Motion 叠加多层 simplex noise，边缘溶解效果 | 深空层次感大幅提升 | 中 | 2 |
| B5 | 引力场暗示 | 相关节点间微弱光线/粒子流，距离越近越明显 | 知识关联可视化 | 高 | 3 |

**核心技术挑战：**
- B1 缩放过渡需要 Three.js 相机动画 + DOM 过渡 + 路由切换的精确协调
- 方案：相机动画用 spring physics（已有弹簧缓动基础），DOM 层用 CSS transition + AnimatePresence
- B3 流星粒子适合 Points + BufferGeometry + vertex shader 控制生命周期
- B5 引力场可用 Lines + shader 实现，alpha 随距离衰减

---

### Track C: 规模化基建

目标：为 50-200 篇文章的规模做好技术准备。

| ID | 任务 | 描述 | 收益 | 复杂度 | 波次 |
|----|------|------|------|--------|------|
| C1 | MiniSearch 升级 | 替换当前子串匹配为 MiniSearch，支持模糊搜索 + 字段权重 + 相关性排序 | 搜索体验质的飞跃 | 低 | 1 |
| C2 | 内容校验脚本 | 构建时检查 frontmatter 必填字段、slug 唯一性、日期格式、主题合法性 | 避免内容增长后的隐性 bug | 低 | 1 |
| C3 | Instanced Rendering | 将独立 mesh 节点替换为 InstancedMesh + InstancedBufferAttribute | 200+ 节点不掉帧 | 中 | 2 |
| C4 | FPS 监控 | 开发模式 Stats.js + 性能基准测试脚本 | 优化有数据支撑 | 低 | 2 |
| C5 | AI 辅助内容管线 | Claude API 自动生成摘要、推荐主题标签、语义相似度聚类 | 内容生产效率 × 3 | 中 | 3 |

**核心技术挑战：**
- C3 instanced rendering 需要将 per-node shader uniforms 迁移为 instance attributes（颜色、大小、发光强度等）
- C5 AI 管线建议在构建脚本中集成，非实时调用，构建时批量处理

## 执行节奏

### 第一波 — 快速见效

并行启动，每个都能独立完成，完成后立即带来可感知的提升。

```
Track A: A1 (OG 卡片) + A2 (sitemap)
Track B: B1 (宇宙缩放过渡)
Track C: C1 (MiniSearch) + C2 (内容校验)
```

**完成标志：**
- 分享链接到微信/Twitter 时显示封面预览卡片
- 点击星球有电影级缩放过渡，返回时缩回宇宙
- 搜索输入 "typ" 能模糊匹配到 "TypeScript"
- 构建时自动校验 18 篇文章的 frontmatter

### 第二波 — 深度打磨

在第一波基础上深入，需要更多 shader 和架构工作。

```
Track A: A3 (文章预渲染)
Track B: B3 (流星粒子) + B4 (FBM 星云)
Track C: C3 (Instanced rendering) + C4 (FPS 监控)
```

**完成标志：**
- Google 搜索 "焦糖星球" 能直接看到文章内容摘要
- 宇宙中有流星随机划过，星云层次感丰富
- 可以承载 200 个节点不掉帧，有数据监控支撑

### 第三波 — 锦上添花

前两波完成后的增强，非核心但能提升品质上限。

```
Track A: A4 (自定义域名)
Track B: B2 (搜索飞行) + B5 (引力场暗示)
Track C: C5 (AI 内容管线)
```

**完成标志：**
- 自定义域名上线
- 搜索和节点间都有精美的飞行/连线动画
- 新文章只需写正文，摘要和标签自动生成

## 风险与决策点

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| B1 缩放过渡涉及相机/DOM/路由三层协调 | 实现复杂度高，可能影响现有手势系统 | 先做原型验证，保留回退到当前侧滑卡片的能力 |
| A3 预渲染改变构建产物结构 | 可能破坏现有 SPA 路由 | 渐进式实施，先生成 HTML 再逐步 hydrate |
| C3 instanced rendering 重构节点 shader | 现有 per-node uniforms 需要全部迁移 | 保留旧实现作 fallback，A/B 对比性能 |
| 三轨并行可能分散精力 | 每条轨道进度都慢 | 每次聚焦一条轨道完成一个完整任务，不要三个同时半成品 |
